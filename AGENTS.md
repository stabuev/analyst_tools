# Инструкции проекта

Этот репозиторий содержит открытый практический курс **«Инструменты аналитика»**.
Рабочий язык материалов и общения о курсе — русский.

## Начало новой сессии

Перед содержательной работой:

1. Прочитай `README.md`.
2. Прочитай `docs/PROJECT_STATUS.md`; затем проверь `git status`, потому что рабочее
   дерево новее статусного документа.
3. Для изменения программы прочитай `curriculum.json` и `docs/course-design.md`.
4. Для разработки урока прочитай `LESSON_TEMPLATE.md`, README соответствующей фазы и
   ближайший завершенный урок как пример.
5. Для обсуждения состава библиотек и исходной программы прочитай
   `docs/research-baseline.md`.

Не полагайся на историю чата как на единственный источник решений.

## Цель и эталон

Итоговый продукт состоит из двух равноправных частей:

- самодостаточных уроков в `phases/`;
- standalone static-сайта в `site/`, который показывает программу и ведет на материалы
  уроков в GitHub.

Структурный эталон — `ai-engineering-from-scratch`: фазы, небольшие уроки, единый учебный
цикл, проверяемые артефакты и статический сайт. Не копируй предметное содержание или
оформление механически; сохраняй идеологию организации.

Локальный эталонный репозиторий, если он доступен:
`/Users/stabuev/Documents/GitHub/ai-engineering-from-scratch`.

## Источники правды

- `curriculum.json` — фазы, уроки, статусы, маршруты и зависимости.
- `LESSON_TEMPLATE.md` — обязательный контракт завершенного урока.
- `docs/course-design.md` — педагогические принципы.
- `docs/data-universe.md` — общая учебная предметная область.
- `docs/PROJECT_STATUS.md` — текущий handoff и следующий шаг.

Не редактируй вручную:

- `ROADMAP.md`;
- `phases/*/README.md`;
- `outputs/index.json`;
- `site/data.js`.

Они генерируются скриптами из источников правды.

## Разработка урока

Разрабатывай уроки последовательно, а не массовыми пустыми заготовками. Завершенный урок
содержит:

- `docs/ru.md` по циклу
  `Problem -> Concept -> Build It -> Use It -> Break It -> Verify It -> Ship It`;
- проверяемую практику: исполняемый пример с behavioral tests либо направляемый
  артефакт с отдельной рубрикой;
- `quiz.json` с минимум двумя входными и тремя итоговыми вопросами;
- именованный артефакт в `outputs/` и `artifact.json`;
- `lesson.json`;
- от трех до пяти аннотированных развивающих ссылок, преимущественно на официальную
  документацию и первичные источники.

Статус `complete` ставится только после выполнения контракта и полного прогона проверок.
Код не добавляется ради единообразия: он обязателен только когда программный или
вычислительный механизм входит в результат обучения.

## Сайт

`site/` обязан работать без backend и без JavaScript build toolchain на любом static
hosting, включая GitHub Pages в подпапке репозитория.

- Используй относительные пути к локальным assets.
- Данные сайта генерируются через `scripts/render_site.py`.
- Готовые уроки должны иметь корректные GitHub-ссылки.
- После заметного изменения интерфейса проверь desktop и mobile через локальный HTTP-сервер.

## Исследование

Актуальные сведения о библиотеках, API и рекомендуемых материалах проверяй по официальной
документации или первичным источникам. Новая библиотека добавляется только вместе с
задачей, которую она решает лучше уже присутствующих средств.

## Команды проверки

```bash
python3 scripts/validate_course.py
python3 scripts/render_curriculum.py --check
python3 scripts/render_outputs.py --check
python3 scripts/render_site.py --check
python3 -m unittest discover -s tests
python3 scripts/run_lesson_tests.py
```

После изменения `curriculum.json` или материалов:

```bash
python3 scripts/render_curriculum.py
python3 scripts/render_outputs.py
python3 scripts/render_site.py
```

## Git

Не коммить и не отправляй изменения в remote без явной команды пользователя. Не
перезаписывай пользовательские изменения в грязном рабочем дереве. Перед handoff обновляй
`docs/PROJECT_STATUS.md`, если изменились готовые этапы, следующий урок или архитектурные
решения.
