# Быстрый старт

Эта инструкция помогает дойти от чистой машины до первого запущенного урока. Курс не
требует глобальной установки всех библиотек: зависимости и подходящий Python
восстанавливает `uv` из `uv.lock`.

## Поддерживаемое окружение

- **macOS и Linux:** используйте Bash или Zsh.
- **Windows:** используйте WSL 2 с Ubuntu. Команды терминальной фазы написаны для Unix
  shell; PowerShell и `cmd.exe` не являются поддерживаемой учебной средой.
- **Python:** проект требует Python 3.11 или новее. `uv` может скачать совместимый
  интерпретатор автоматически.
- **Git:** нужен для клонирования курса и упражнений фаз 00–01.

Если вы впервые работаете с терминалом, сначала установите инструменты по официальным
инструкциям:

- [Git: установка](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git) —
  выберите раздел своей операционной системы;
- [uv: установка](https://docs.astral.sh/uv/getting-started/installation/) — используйте
  официальный installer и затем откройте новый терминал;
- [Microsoft: установка WSL](https://learn.microsoft.com/windows/wsl/install) — только
  для Windows.

## Проверьте инструменты

```bash
git --version
uv --version
```

Обе команды должны напечатать версию и завершиться без ошибки. Отдельная команда
`python` на этом шаге необязательна: проектный Python будет запускаться через `uv run`.

## Клонируйте и восстановите курс

```bash
git clone https://github.com/stabuev/analyst_tools.git
cd analyst_tools
uv sync --locked --dev
uv run --locked python scripts/validate_course.py
```

Ожидаемый последний вывод:

```text
Course structure is valid.
```

Проверьте, что запускается первый пример:

```bash
uv run --locked python \
  phases/00-entry-and-tools/01-profession-map/code/main.py
```

## Где выполнять упражнения

Не изменяйте эталонный файл до первой самостоятельной попытки. Создайте локальную рабочую
папку, которая уже исключена из Git:

```bash
mkdir -p work/00-02-diagnostic
touch work/00-02-diagnostic/solution.py
```

Перенесите в `solution.py` только условие из документации и начните с собственного
решения. Для обычного урока создавайте файлы в `work/<phase>-<lesson>/`. Папка `work/` не
попадёт в commit. Готовые `code/` и `outputs/` остаются reference implementation:
открывайте их после собственной попытки или когда нужно локализовать непонятный шаг.

## Типовые ошибки первого запуска

### `command not found: uv`

Откройте новый терминал после установки и снова выполните `uv --version`. Если команда
не появилась, вернитесь к разделу PATH в официальной инструкции uv.

### `No pyproject.toml found`

Проверьте текущий каталог:

```bash
pwd
ls pyproject.toml uv.lock
```

Команды курса с `uv run --locked` запускаются из корня репозитория, если в конкретном
уроке явно не сказано иначе.

### `uv.lock needs to be updated`

Не запускайте `uv lock` только ради обхода ошибки. Сначала проверьте, что вы на ветке
`main`, не изменили `pyproject.toml` и получили актуальную версию репозитория.

### Windows-команда не понимает путь или shell-синтаксис

Убедитесь, что репозиторий открыт внутри WSL, а команда выполняется в Bash. Путь вида
`C:\\Users\\...` и prompt PowerShell означают, что вы находитесь вне поддерживаемой среды.
