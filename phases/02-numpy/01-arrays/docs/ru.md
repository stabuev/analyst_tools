# ndarray и модель массива

> `ndarray` хранит однородные данные в прямоугольной многомерной структуре. Чтобы уверенно работать с NumPy, сначала научитесь читать контракт массива: `ndim`, `shape`, `size` и `dtype`.

**Тип:** Learn
**Треки:** Core
**Пререквизит:** `01-reproducible-project/09-continuous-integration`
**Время:** ~60 минут

## Цели обучения

После урока вы сможете:

- объяснить, чем модель `ndarray` отличается от обычного списка Python;
- определить размерность, форму, количество элементов и тип данных массива;
- создать массив через `np.array`, `np.zeros`, `np.ones`, `np.arange` и `np.linspace`;
- заранее обнаружить рваную вложенную последовательность;
- выпустить CLI-инспектор, который проверяет контракт числового массива.

## Проблема

Допустим, аналитик получил наблюдения по двум магазинам за три дня:

```python
sales = [
    [12, 15, 9],
    [10, 11, 14],
]
```

Вложенный список выглядит как таблица, но сам Python не гарантирует этого. Во второй
строке может оказаться меньше значений:

```python
ragged_sales = [
    [12, 15, 9],
    [10, 11],
]
```

Или среди чисел внезапно появится строка:

```python
mixed_sales = [
    [12, 15, 9],
    [10, "нет данных", 14],
]
```

Для списка обе структуры допустимы. Список Python является универсальным изменяемым
контейнером: его элементы могут иметь разные типы, вложенные списки могут иметь разную
длину, а сам контейнер может расти.

Это удобно для общего программирования, но численным алгоритмам нужен более строгий
контракт. Операция над таблицей должна заранее знать:

- сколько у данных осей;
- какова длина каждой оси;
- сколько всего элементов;
- как числа представлены в памяти.

Такой контракт дает `numpy.ndarray`.

## Концепция

Импортируем NumPy по общепринятому короткому имени:

```python
import numpy as np
```

Создадим массив из исходного списка:

```python
sales_array = np.array(sales)
print(sales_array)
```

```text
[[12 15  9]
 [10 11 14]]
```

У массива есть четыре базовых атрибута:

```python
print(sales_array.ndim)
print(sales_array.shape)
print(sales_array.size)
print(sales_array.dtype)
```

Пример результата:

```text
2
(2, 3)
6
int64
```

Точное имя целочисленного `dtype` может зависеть от платформы. Смысл атрибутов от этого
не меняется.

| Атрибут | Вопрос | Значение для примера |
|---|---|---|
| `ndim` | Сколько осей у массива? | `2` |
| `shape` | Какова длина каждой оси? | `(2, 3)` |
| `size` | Сколько всего элементов? | `6` |
| `dtype` | Как представлен каждый элемент? | целочисленный тип |

Для любого обычного массива выполняются два полезных инварианта:

```python
sales_array.ndim == len(sales_array.shape)
sales_array.size == np.prod(sales_array.shape)
```

В нашем примере:

```text
ndim = len((2, 3)) = 2
size = 2 * 3 = 6
```

### Оси и форма

Форма читается слева направо:

- скаляр имеет форму `()`;
- вектор из трех элементов имеет форму `(3,)`;
- матрица из двух строк и трех столбцов имеет форму `(2, 3)`;
- набор из четырех матриц формы `(2, 3)` имеет форму `(4, 2, 3)`.

Запятая в `(3,)` важна: это кортеж с одним элементом, а не число в скобках.

В NumPy оси нумеруются с нуля. Для массива формы `(2, 3)`:

- ось `0` имеет длину `2`;
- ось `1` имеет длину `3`.

Подробные операции по осям появятся в следующем уроке. Сейчас достаточно научиться
читать форму как часть контракта данных.

### Однородный тип

У списка Python нет одного общего типа элементов:

```python
python_values = [1, 2.5, "3"]
```

У `ndarray` есть один `dtype` для всех элементов. При создании массива NumPy пытается
найти общее представление:

```python
values = np.array([1, 2.5])
print(values.dtype)
print(values)
```

Обычно целое `1` будет преобразовано в число с плавающей точкой, чтобы оба элемента
имели один `dtype`:

```text
float64
[1.  2.5]
```

Если добавить строку, NumPy может выбрать строковый `dtype`. Это формально корректный
массив, но уже не числовой. Поэтому прикладной контракт часто должен быть строже
возможностей самой библиотеки.

### Список и массив решают разные задачи

```python
python_list = [1, 2, 3]
numpy_array = np.array([1, 2, 3])

print(python_list * 2)
print(numpy_array * 2)
```

```text
[1, 2, 3, 1, 2, 3]
[2 4 6]
```

Для списка оператор `*` повторяет контейнер. Для массива он по умолчанию применяется
поэлементно. Это не значит, что массив всегда лучше списка. Выбирайте модель по задаче:

| Свойство | `list` | `ndarray` |
|---|---|---|
| Типы элементов | могут различаться | представлены одним `dtype` |
| Вложенная структура | может быть рваной | имеет одну прямоугольную `shape` |
| Размер контейнера | может расти | число элементов фиксировано |
| Численные операторы | часто управляют контейнером | обычно работают поэлементно |

Значения внутри существующего массива менять можно. Но добавление элементов создает
новый массив, потому что исходная прямоугольная область памяти не растет как список.

## Соберите это

Прежде чем вызывать NumPy, построим упрощенную проверку формы самостоятельно. Она
поможет увидеть, откуда берется `shape`.

Скаляр не добавляет новую ось, поэтому его форма равна пустому кортежу:

```python
def infer_shape(value):
    if isinstance(value, (int, float)) and not isinstance(value, bool):
        return ()
```

Последовательность добавляет ось длиной `len(value)`:

```python
def infer_shape(value):
    if isinstance(value, list):
        if not value:
            return (0,)

        child_shapes = [infer_shape(item) for item in value]
        expected = child_shapes[0]

        if any(shape != expected for shape in child_shapes[1:]):
            raise ValueError("ragged nested sequence")

        return (len(value), *expected)

    if isinstance(value, (int, float)) and not isinstance(value, bool):
        return ()

    raise TypeError("expected a real number or a nested list")
```

Проверим несколько случаев:

```python
assert infer_shape(7) == ()
assert infer_shape([7, 8, 9]) == (3,)
assert infer_shape([[1, 2, 3], [4, 5, 6]]) == (2, 3)
```

Для матрицы алгоритм рассуждает так:

1. Каждый скаляр имеет форму `()`.
2. Каждая строка из трех скаляров имеет форму `(3,)`.
3. Две строки одинаковой формы образуют массив формы `(2, 3)`.

Рваная структура не проходит проверку:

```python
infer_shape([[1, 2], [3]])
```

```text
ValueError: ragged nested sequence
```

Учебный артефакт расширяет этот алгоритм: сообщает путь до проблемного элемента,
принимает JSON, проверяет только вещественные числа и сопоставляет ручной результат с
`array.shape`.

## Используйте это

### Создание из существующих данных

Используйте `np.array`, когда значения уже находятся в последовательности Python:

```python
sales = np.array([[12, 15, 9], [10, 11, 14]])
```

При необходимости `dtype` можно задать явно:

```python
sales = np.array([[12, 15, 9], [10, 11, 14]], dtype=np.float32)
```

Явный `dtype` полезен, когда формат чисел является частью контракта. Подробно выбор
типов, точность и переполнение будут разобраны в отдельном уроке.

### Заполнение известной формы

Когда важна форма, а значения должны начинаться с нулей или единиц:

```python
zeros = np.zeros((2, 3))
ones = np.ones((2, 3))
```

По умолчанию эти функции обычно создают `float64`:

```python
integer_zeros = np.zeros((2, 3), dtype=np.int64)
```

Функция `np.empty((2, 3))` тоже выделяет массив нужной формы, но не заполняет его
предсказуемыми значениями. Она предназначена для случаев, когда вы сразу перезапишете
каждый элемент. Для исходных данных урока безопаснее использовать `zeros` или `ones`.

### Числовая последовательность с шагом

`np.arange` похож на `range`, но возвращает массив:

```python
even_numbers = np.arange(0, 10, 2)
print(even_numbers)
```

```text
[0 2 4 6 8]
```

Аргументы означают начало, невключенную правую границу и шаг.

Для нецелого шага точное количество элементов `arange` может быть неочевидным из-за
арифметики с плавающей точкой. Если важнее количество точек, используйте `linspace`.

### Фиксированное количество точек

`np.linspace` делит интервал на заданное число точек:

```python
grid = np.linspace(0, 1, num=5)
print(grid)
```

```text
[0.   0.25 0.5  0.75 1.  ]
```

Правая граница включена по умолчанию. Здесь контракт читается как «создать ровно пять
точек от нуля до единицы».

## Сломайте это

### Ошибка 1. Принять рваную структуру за матрицу

```bash
uv run --locked python phases/02-numpy/01-arrays/outputs/array_inspector.py \
  --values '[[1, 2], [3]]'
```

Ожидаемый результат:

```text
array-inspector: ragged nested sequence at values[1]: expected child shape (2,), got (1,)
```

Программа завершится с кодом `2` и не создаст двусмысленный массив.

### Ошибка 2. Незаметно превратить числа в строки

Сам NumPy может найти общий строковый `dtype`:

```python
mixed = np.array([1, "2", 3])
print(mixed.dtype)
```

Но инспектор моделирует числовой аналитический контракт и отклоняет строку:

```bash
uv run --locked python phases/02-numpy/01-arrays/outputs/array_inspector.py \
  --values '[1, "2", 3]'
```

Не используйте успешное создание `ndarray` как единственную проверку качества данных.
Проверяйте еще и допустимую предметную область.

### Ошибка 3. Перенести семантику списка на массив

```python
assert [1, 2, 3] * 2 == [1, 2, 3, 1, 2, 3]
assert (np.array([1, 2, 3]) * 2 == np.array([2, 4, 6])).all()
```

Один и тот же оператор выполняет разные действия, потому что типы реализуют разные
модели данных.

### Ошибка 4. Считать `shape` количеством элементов

Для формы `(2, 3)`:

```python
array = np.zeros((2, 3))

assert array.shape == (2, 3)
assert array.size == 6
```

`shape` является кортежем длин осей. `size` является произведением этих длин.

### Ошибка 5. Читать значения из `np.empty`

```python
scratch = np.empty((2, 3))
print(scratch)
```

Напечатанные числа не являются исходными данными. Это содержимое выделенной памяти,
которое нужно полностью перезаписать перед использованием.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| `ndarray` | «Это быстрый список Python» | Однородный многомерный массив с фиксированным числом элементов и прямоугольной формой |
| Ось (`axis`) | «Это обязательно строка или столбец» | Одно из направлений массива; оси нумеруются от `0` до `ndim - 1` |
| `ndim` | «Количество элементов» | Количество осей массива |
| `shape` | «Общий размер массива» | Кортеж длин всех осей |
| `size` | «Размер в байтах» | Общее количество элементов, равное произведению длин осей |
| `dtype` | «Тип отдельного случайного значения» | Общее представление элементов массива |
| Рваная структура | «Матрица с пропуском» | Вложенные последовательности, у которых дочерние формы различаются |
