## Проблема

В предыдущей фазе вы подготовили воспроизводимое окружение, тесты и автоматические
проверки. Теперь в этом окружении появляется первый инструмент для самих вычислений —
NumPy.

Допустим, аналитик получил количество заказов в двух магазинах за три дня:

```python
order_counts = [
    [12, 15, 9],
    [10, 11, 14],
]
```

По расположению значений мы предполагаем следующий смысл:

```text
строки  -> магазины
столбцы -> дни
```

Но вложенный список не хранит эти договоренности. Python разрешит добавить строку другой
длины:

```python
ragged_counts = [
    [12, 15, 9],
    [10, 11],
]
```

Он также разрешит смешать числа и текст:

```python
mixed_counts = [
    [12, 15, 9],
    [10, "нет данных", 14],
]
```

Наконец, оператор может означать для списка не то, что ожидает аналитик:

```python
print([1, 2, 3] * 2)
```

```text
[1, 2, 3, 1, 2, 3]
```

Список повторился. Значения внутри него не умножились.

Список Python не плох и не ошибочен. Это универсальный контейнер, который может расти и
хранить разные объекты. Но для одинаковой операции над большим количеством однородных
чисел удобнее более строгая модель данных.

Перед расчетом нужно ответить на четыре вопроса:

1. Все ли элементы описывают один вид значения?
2. Можно ли расположить их в прямоугольной структуре?
3. Что означает каждая ось этой структуры?
4. Совпала ли фактическая форма с ожидаемой?

Если ответы определены, данные являются хорошим кандидатом для `numpy.ndarray`.

## Концепция

### Где аналитику встречается NumPy

NumPy предоставляет многомерный массив и операции над ним. Эта модель особенно полезна
для:

- числовых временных рядов;
- матриц признаков для моделей;
- результатов симуляций;
- изображений и сигналов;
- промежуточных численных расчетов;
- обмена массивами между библиотеками Python.

NumPy важен не только как самостоятельная библиотека. Модель массива используется во
многих инструментах научного Python и машинного обучения. Понимание `shape` и `dtype`
понадобится позже в pandas, SciPy и scikit-learn.

При этом `ndarray` не является универсальной заменой любой структуре данных. Если в одной
строке находятся идентификатор, дата, категория и сумма, перед нами разнородная бизнес-
таблица. Для нее обычно удобнее DataFrame или SQL-таблица с отдельным типом и именем у
каждого столбца.

### Три свойства модели массива

Обычный массив NumPy принимает три основных ограничения.

1. **Один `dtype`.** Все элементы интерпретируются по одному правилу представления.
2. **Прямоугольная форма.** На одном уровне вложенности дочерние части имеют одинаковую
   форму.
3. **Размер не растет как у списка.** Добавление значений обычно создает новый массив.

Эти ограничения позволяют выполнять одну операцию сразу над многими элементами и дают
библиотеке предсказуемую структуру данных.

Импортируем NumPy по общепринятому короткому имени:

```python
import numpy as np
```

Создадим массив:

```python
orders = np.array(order_counts)
print(orders)
```

```text
[[12 15  9]
 [10 11 14]]
```

Теперь умножение применяется к каждому элементу:

```python
print(orders * 2)
```

```text
[[24 30 18]
 [20 22 28]]
```

Это первая причина использовать массив: код выражает численную операцию над всем набором
значений, а не управление контейнером.

### Четыре свойства конкретного массива

```python
print(orders.ndim)
print(orders.shape)
print(orders.size)
print(orders.dtype)
```

Пример результата:

```text
2
(2, 3)
6
int64
```

Точное целочисленное имя `dtype` может зависеть от платформы. Смысл атрибутов от этого не
меняется.

| Атрибут | Какой вопрос задаем | Значение в примере |
|---|---|---|
| `ndim` | Сколько у массива осей? | `2` |
| `shape` | Какова длина каждой оси по порядку? | `(2, 3)` |
| `size` | Сколько всего элементов? | `6` |
| `dtype` | Как NumPy интерпретирует каждый элемент? | целочисленный тип |

Для массива выполняются два полезных соотношения:

```python
orders.ndim == len(orders.shape)
orders.size == np.prod(orders.shape)
```

В нашем примере две длины в `shape` означают две оси, а произведение `2 * 3` дает шесть
элементов.

### Форма без смысла недостаточна

`shape == (2, 3)` сообщает длины осей, но не объясняет предметную область. Контракт
исходных данных следует записать так:

```text
(store, day) = (2, 3)
```

Тогда чтение формы становится однозначным:

- ось `0`, `store`: два магазина;
- ось `1`, `day`: три дня.

Другой массив той же формы мог бы означать два рекламных канала и три метрики. NumPy не
хранит эти названия внутри обычного `ndarray`: смысл осей задает аналитик.

В следующем уроке мы научимся предсказывать, как операции переставляют, добавляют и
удаляют оси. Здесь достаточно научиться называть оси исходного массива.

### Скаляр, вектор и многомерный массив

| Значения | `ndim` | `shape` | Предметное чтение |
|---|---:|---|---|
| `np.array(7)` | `0` | `()` | одно значение без осей |
| `np.array([7, 8, 9])` | `1` | `(3,)` | три наблюдения |
| `np.array([[1, 2, 3], [4, 5, 6]])` | `2` | `(2, 3)` | две строки по три значения |
| `np.zeros((4, 2, 3))` | `3` | `(4, 2, 3)` | четыре слоя формы `(2, 3)` |

Python-число `7` и нульмерный массив `np.array(7)` — разные объекты:

```python
python_number = 7
array_scalar = np.array(7)

assert not isinstance(python_number, np.ndarray)
assert isinstance(array_scalar, np.ndarray)
assert array_scalar.shape == ()
assert array_scalar.size == 1
```

Запятая в `(3,)` важна: это кортеж из одного элемента. Выражение `(3)` является просто
числом `3` в скобках.

### Один dtype не означает «только числа»

У каждого массива есть один `dtype`, но он может быть числовым, логическим, строковым,
объектным или более сложным. Поэтому успешное создание `ndarray` еще не доказывает, что
массив подходит для численного расчета.

```python
mixed = np.array([1, "2", 3])

print(mixed)
print(mixed.dtype)
```

NumPy выберет строковый `dtype`, способный представить все три значения. Это корректный
`ndarray`, но не тот числовой массив, который ожидал аналитик.

Если смешать целые и дробные числа, NumPy обычно выберет общий вещественный тип:

```python
values = np.array([1, 2.5])

print(values)
print(values.dtype)
```

```text
[1.  2.5]
float64
```

Здесь достаточно заметить факт приведения к одному `dtype`. Диапазоны, память, пропуски и
осознанный выбор разрядности рассматриваются в отдельном уроке.

### Список, массив и таблица

| Свойство | `list` | `ndarray` | DataFrame или SQL-таблица |
|---|---|---|---|
| Основная роль | универсальный контейнер | однородные многомерные данные | именованные разнородные столбцы |
| Типы | элементы могут различаться | один `dtype` на массив | отдельный тип у каждого столбца |
| Форма | может быть рваной | прямоугольная | строки и именованные столбцы |
| Рост | может расти на месте | не растет как список | строки добавляются средствами таблицы |
| Численные операции | часто требуют цикла | обычно поэлементные | обычно по столбцам |

Выбирайте структуру по смыслу данных, а не по тому, какой импорт уже написан в файле.

## Соберите это

Сначала предскажем контракт массива без NumPy. Нам не нужен универсальный рекурсивный
алгоритм: для первого шага достаточно явно проверить двумерный рабочий пример.

### Шаг 1. Назовите оси

```python
order_counts = [
    [12, 15, 9],
    [10, 11, 14],
]

axis_names = ("store", "day")
```

Не переходя к коду, запишите ожидание:

```text
ndim = 2
shape = (2 stores, 3 days)
size = 2 * 3 = 6
dtype = integer
```

### Шаг 2. Проверьте прямоугольность исходных списков

```python
row_lengths = [len(row) for row in order_counts]

assert len(set(row_lengths)) == 1
```

Если длины строк одинаковы, можно получить ожидаемую форму:

```python
expected_shape = (len(order_counts), row_lengths[0])
expected_size = expected_shape[0] * expected_shape[1]

assert expected_shape == (2, 3)
assert expected_size == 6
```

Это уменьшенная модель механизма: форма двумерного массива складывается из числа строк и
длины каждой строки. Обобщать проверку на любую глубину сейчас не требуется.

### Шаг 3. Создайте массив и сравните с ожиданием

```python
import numpy as np

orders = np.asarray(order_counts)

assert orders.ndim == 2
assert orders.shape == expected_shape
assert orders.size == expected_size
assert np.issubdtype(orders.dtype, np.number)
```

`np.asarray` принимает как Python-последовательность, так и готовый массив. Если вход уже
является подходящим `ndarray`, функция может использовать его без лишнего копирования.
Подробные правила копий и представлений появятся в уроке об индексации.

### Шаг 4. Добавьте предметный паспорт

Артефакт урока выполняет ту же проверку и связывает длины с названиями осей:

```python
report = describe_array(
    order_counts,
    axes=("store", "day"),
)
```

Ключевая часть результата:

```python
{
    "ndim": 2,
    "shape": (2, 3),
    "size": 6,
    "dtype": "int64",
    "is_numeric": True,
    "axes": {
        "store": 2,
        "day": 3,
    },
}
```

Такой отчет не заменяет знание предметной области. Функция не может догадаться, что первая
ось означает магазин: это имя передает аналитик.

## Используйте это

### Создавайте массив по намерению

Не нужно запоминать длинный перечень конструкторов. Сначала сформулируйте, откуда должны
взяться значения.

| Намерение | Подходящий вызов | Пример результата |
|---|---|---|
| Преобразовать существующие значения | `np.asarray(values)` | массив из списка или другого array-like объекта |
| Создать известную форму с заполнителем | `np.zeros(shape)` или `np.full(shape, value)` | заготовка известной формы |
| Создать целочисленный интервал с шагом | `np.arange(start, stop, step)` | значения до невключенной правой границы |
| Получить точное количество точек интервала | `np.linspace(start, stop, num)` | ровно `num` точек, обычно с обеими границами |

Примеры:

```python
orders = np.asarray([[12, 15, 9], [10, 11, 14]])
empty_week = np.zeros((2, 7), dtype=int)
target = np.full((2, 7), 20)
even_hours = np.arange(0, 24, 2)
thresholds = np.linspace(0, 1, num=5)
```

Для `arange` естественным контрактом является шаг. Правая граница не включается:

```python
assert np.array_equal(np.arange(0, 6, 2), np.array([0, 2, 4]))
```

Для `linspace` естественным контрактом является количество точек:

```python
assert np.array_equal(
    np.linspace(0, 1, num=5),
    np.array([0.0, 0.25, 0.5, 0.75, 1.0]),
)
```

При дробном шаге количество значений `arange` может быть неочевидным из-за арифметики с
плавающей точкой. Если требование сформулировано как «ровно пять точек», выбирайте
`linspace`.

## Сломайте это

### Ошибка 1. Рваная вложенная структура

```python
ragged = [[1, 2], [3]]
np.asarray(ragged)
```

Современный NumPy завершит создание массива с `ValueError`: у второй оси нет единой
длины. Артефакт переводит это в сообщение о необходимости проверить длины вложенных
последовательностей.

Если явно запросить `dtype=object`, можно получить одномерный массив из двух Python-
объектов. Это не превращает рваные строки в обычный двумерный числовой массив.

### Ошибка 2. Успешное создание не гарантирует числовой смысл

```python
report = describe_array([1, "2", 3], axes=("observation",))

assert report["is_numeric"] is False
```

Все значения получили общий строковый `dtype`. Функция `require_numeric_array()` отклонит
такой вход вместо неявного продолжения расчета.

### Ошибка 3. Перепутать порядок осей

```python
orders = np.asarray([[12, 15, 9], [10, 11, 14]])
```

Одинаковый массив можно ошибочно подписать как `(day, store)`. Код не узнает, что длина
`2` относится к магазинам, а длина `3` — к дням. Поэтому ожидаемые названия и длины осей
нужно получить из контракта источника, а не из внешнего вида напечатанного массива.

### Ошибка 4. Передать не то количество названий

```python
describe_array(orders, axes=("store",))
```

У двумерного массива две оси, поэтому паспорт отклонит одно имя. Эта проверка не доказывает
правильность смысла, но обнаруживает неполный контракт.

### Ошибка 5. Использовать массив для разнородной бизнес-строки

```python
record = [101, "2026-07-17", "Москва", 1490.50]
```

Элементы описывают разные сущности: идентификатор, дату, город и сумму. Принудительно
сводить их к одному `dtype` неудобно и опасно. Здесь нужна именованная табличная модель,
которую мы начнем изучать в фазе pandas.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| `ndarray` | «Это быстрый список» | Многомерный контейнер с одной формой и одним `dtype` |
| Ось (`axis`) | «Это обязательно строка или столбец» | Одно направление массива с предметным смыслом и определенной длиной |
| `ndim` | «Количество элементов» | Количество осей массива |
| `shape` | «Общий размер» | Упорядоченный кортеж длин осей |
| `size` | «Размер в байтах» | Общее количество элементов массива |
| `dtype` | «Тип одного случайного значения» | Единое правило интерпретации элементов массива |
| Нульмерный массив | «Обычное Python-число» | `ndarray` формы `()` с одним элементом и без осей |
| Рваная структура | «Матрица с пропуском» | Вложенные последовательности с несовпадающими дочерними формами |
| Array-like | «Только список» | Объект, который NumPy умеет преобразовать в массив |
