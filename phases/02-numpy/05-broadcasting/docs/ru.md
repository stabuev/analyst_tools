# Broadcasting без магии

> Broadcasting сравнивает формы справа налево: длины совместимы, если равны или одна из них равна `1`.

**Тип:** Build
**Треки:** Core
**Пререквизит:** `02-numpy/04-indexing-and-masks`
**Время:** ~60 минут
**Результат:** проверяет совместимость форм и использует broadcasting осознанно.

## Цели обучения

- вручную вывести результирующую broadcast-форму;
- отличить совместимость формы от правильности смысла;
- нормализовать столбцы матрицы без Python-циклов;
- обнаружить нулевой масштаб и неверно ориентированный параметр;
- оценить риск большого промежуточного массива.

## Проблема

Матрица признаков имеет форму `(100, 3)`: сто наблюдений и три признака. Чтобы вычесть
среднее каждого столбца, нужен вектор формы `(3,)`.

```python
normalized = (features - means) / scales
```

NumPy не создаёт вручную сто копий `means`. Он сопоставляет формы и применяет значения
вектора к каждой строке.

Но совместимая форма может иметь неверный смысл. Вектор среднего по строкам формы `(100,)`
не совместим с `(100, 3)`, а форма `(100, 1)` уже совместима, хотя нормализует другую
семантическую ось.

## Концепция

Формы выравниваются по правому краю:

```text
matrix: (100, 3)
vector:      (3,)
result: (100, 3)
```

Для каждой пары осей допустимо:

1. длины равны;
2. одна длина равна `1`;
3. ось отсутствует и считается равной `1`.

Несовместимый пример:

```text
matrix: (100, 3)
vector:    (100,)
```

Последние длины `3` и `100` конфликтуют.

Совместимый, но другой по смыслу пример:

```text
matrix: (100, 3)
column: (100, 1)
result: (100, 3)
```

Здесь одно значение применяется ко всем трём столбцам каждой строки.

## Соберите это

### Ручное объединение двух форм

```python
def broadcast_shape(left, right):
    width = max(len(left), len(right))
    left = (1,) * (width - len(left)) + left
    right = (1,) * (width - len(right)) + right

    result = []
    for a, b in zip(left, right):
        if a == b or a == 1 or b == 1:
            result.append(max(a, b))
        else:
            raise ValueError("incompatible shapes")
    return tuple(result)
```

```python
assert broadcast_shape((5, 1), (1, 6)) == (5, 6)
assert broadcast_shape((100, 3), (3,)) == (100, 3)
```

Алгоритм не растягивает данные. Он только предсказывает логическую форму операции.

### Нормализация столбцов

Для каждого признака:

```text
z = (x - center) / scale
```

Ручная версия с циклами:

```python
result = []
for row in features:
    normalized_row = []
    for value, center, scale in zip(row, centers, scales):
        normalized_row.append((value - center) / scale)
    result.append(normalized_row)
```

Форма `centers` и `scales` должна соответствовать числу признаков, а масштаб не должен
быть нулевым.

## Используйте это

```python
import numpy as np

features = np.array([
    [1.0, 10.0],
    [3.0, 14.0],
    [5.0, 18.0],
])

centers = features.mean(axis=0)
scales = features.std(axis=0)
normalized = (features - centers) / scales
```

Формы:

```text
features:   (3, 2)
centers:       (2,)
scales:        (2,)
normalized: (3, 2)
```

Проверим результат:

```python
np.testing.assert_allclose(normalized.mean(axis=0), [0, 0], atol=1e-12)
np.testing.assert_allclose(normalized.std(axis=0), [1, 1], atol=1e-12)
```

Официальный предиктор NumPy:

```python
assert np.broadcast_shapes((3, 2), (2,), (2,)) == (3, 2)
```

## Сломайте это

### Несовместимые формы

```python
np.zeros((4, 3)) + np.zeros(4)
```

Последние оси имеют длины `3` и `4`, поэтому операция отклоняется.

### Неверная семантика

Форма `(3, 1)` совместима с матрицей `(3, 2)`, но означает параметр на строку. Артефакт
нормализации принимает только `(2,)` или `(1, 2)`, потому что его контракт задаёт
параметры на столбец.

### Постоянный признак

Если стандартное отклонение равно нулю:

```text
(x - mean) / 0
```

нормализация не определена. Артефакт сообщает ошибку вместо выдачи `NaN` или `inf`.

### Большой промежуточный результат

Broadcasting обычно не копирует малый операнд, но сама операция может создать огромный
результат. Вычитание массивов форм `(50_000, 1, 100)` и `(1, 50_000, 100)` логически
создаёт форму `(50_000, 50_000, 100)`. Корректная форма не означает приемлемую память.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Broadcasting | «Физическое копирование малого массива» | Правила виртуального согласования форм |
| Trailing axis | «Первая ось» | Ось в правой части shape |
| Singleton axis | «Пустая ось» | Ось длины `1`, которую можно расширить |
| Совместимость | «Операция верна по смыслу» | Только возможность согласовать формы |
| Стандартизация | «Любое деление массива» | Вычитание центра и деление на ненулевой масштаб по выбранной оси |
