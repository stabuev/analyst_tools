# Shape, axes и размерность

> `shape` — это упорядоченный контракт осей, а не просто кортеж чисел. До вычисления назовите изменяемую ось и запишите, какие оси и длины должны остаться в результате.

**Тип:** Build
**Треки:** Core
**Пререквизит:** `02-numpy/01-arrays`
**Время:** ~75 минут
**Результат:** по предметным именам осей предсказывает форму и смысл результата после удаления, сохранения, перестановки, объединения или добавления осей и закрепляет ожидание проверкой.

## Цели обучения

После урока вы сможете:

- продолжить паспорт массива из первого урока и читать `shape` вместе с предметными именами осей;
- объяснить, почему `axis` — это позиция в `shape`, а не универсальное обозначение строки или столбца;
- до запуска NumPy предсказать форму результата редукции, `transpose`, `reshape` и `expand_dims`;
- нормализовать отрицательный номер оси и обнаружить несуществующую или повторную ось;
- отличить перестановку осей от перегруппировки элементов и не использовать `reshape` вместо `transpose`;
- заметить ситуацию, когда правильный `shape` не гарантирует правильный предметный смысл;
- зафиксировать ожидаемую форму именованной проверкой рядом с аналитическим расчетом.

## Проблема

В первом уроке мы создали массив заказов:

```python
order_counts = [
    [12, 15, 9],
    [10, 11, 14],
]
```

Его паспорт был записан не просто как `(2, 3)`, а вместе со смыслом:

```text
(store, day) = (2, 3)
```

Такой паспорт отвечает на вопрос «что лежит в массиве». Теперь нужно научиться отвечать
на следующий вопрос: **что станет с осями после операции**.

Допустим, по каждому магазину и дню у нас есть уже не одно, а два числовых показателя:
количество заказов и выручка. Получается трехмерный массив:

```text
(store, day, metric) = (2, 3, 2)
```

Задача звучит так: получить итог за все дни отдельно для каждого магазина и каждого
показателя.

До кода ожидаемый контракт результата можно записать словами:

```text
было:      (store, day, metric)
свернули:          day
осталось:  (store,      metric)
shape:     (2, 2)
```

Теперь сравним две синтаксически правильные операции:

```python
daily_metrics.sum(axis=0).shape  # (3, 2)
daily_metrics.sum(axis=1).shape  # (2, 2)
```

Первая складывает магазины и сохраняет дни. Вторая складывает дни и сохраняет магазины.
Обе возвращают числа без исключения, но только вторая отвечает поставленной задаче.

Это опасный класс аналитических ошибок:

- код запускается;
- результат имеет правдоподобный размер;
- числа могут выглядеть разумно;
- смысл расчета уже неверен.

Поэтому номер `axis` нельзя выбирать по памяти. Перед операцией нужно назвать ось, которую
мы меняем, предсказать оставшиеся оси и только затем писать вызов NumPy.

## Концепция

### Что мы переносим из первого урока

Для конкретного массива уже известны четыре свойства:

| Свойство | Вопрос | Пример |
|---|---|---|
| `ndim` | Сколько осей? | `3` |
| `shape` | Какова длина каждой оси по порядку? | `(2, 3, 2)` |
| `size` | Сколько всего элементов? | `12` |
| `dtype` | Как интерпретируется каждый элемент? | например, `int64` |

В этом уроке мы концентрируемся на первых трех. У исходного массива:

```text
ndim = len(shape) = 3
size = 2 * 3 * 2 = 12
```

Но голых чисел недостаточно. Рядом с каждой длиной должно стоять предметное имя:

| Номер оси | Имя оси | Длина | Что меняется вдоль оси |
|---:|---|---:|---|
| `0` | `store` | `2` | магазин |
| `1` | `day` | `3` | день |
| `2` | `metric` | `2` | показатель |

Эта таблица — исходный контракт всех дальнейших преобразований.

### Ось — это позиция в упорядоченной форме

В двумерном массиве ось `0` часто называют строками, а ось `1` — столбцами. Такое
объяснение удобно только для первого знакомства и быстро перестает работать.

В трехмерном массиве нет единственной естественной пары «строка — столбец». Есть три
направления, и каждое определяется позицией в `shape`:

```text
shape[0] -> длина оси 0 -> store
shape[1] -> длина оси 1 -> day
shape[2] -> длина оси 2 -> metric
```

Поэтому `axis=1` означает не «по строкам» и не «по столбцам». Он означает: операция
применяется вдоль второй позиции текущего контракта, то есть вдоль `day`.

После перестановки осей тот же номер может означать уже другую сущность. Номер принадлежит
текущему порядку осей, а не предметной области навсегда.

### Отрицательные номера считаются с конца

NumPy позволяет ссылаться на оси с конца:

```text
(store, day, metric)
    0     1      2
   -3    -2     -1
```

Для массива с `ndim == 3`:

| Запись | Нормализованный номер | Имя |
|---|---:|---|
| `axis=0` | `0` | `store` |
| `axis=-3` | `0` | `store` |
| `axis=1` | `1` | `day` |
| `axis=-2` | `1` | `day` |
| `axis=2` | `2` | `metric` |
| `axis=-1` | `2` | `metric` |

Для отрицательного номера действует правило:

```text
normalized_axis = axis + ndim
```

Но оно применяется только к допустимому диапазону от `-ndim` до `ndim - 1`. Для трех
осей номера `3` и `-4` уже недопустимы.

Отрицательная запись полезна, когда контракт гарантирует смысл последней оси. Например,
`axis=-1` часто читается как «последняя ось», но не как «ось метрик вообще»: после
перестановки последней может оказаться другая ось.

### Пять действий над контрактом осей

Большинство операций этого урока можно понять без значений массива. Достаточно преобразовать
список имен осей и их длин.

| Действие | Что происходит с осями | Что происходит с `size` |
|---|---|---:|
| Поэлементная операция | все оси сохраняются | сохраняется |
| Редукция | выбранная ось удаляется или получает длину `1` | обычно уменьшается |
| `transpose` | оси переставляются | сохраняется |
| `reshape` | старые оси перегруппировываются в новые | сохраняется |
| `expand_dims` | вставляется новая ось длины `1` | сохраняется |

Эта таблица не описывает все NumPy. Индексация и маски будут отдельным уроком,
broadcasting — отдельным уроком, а выбор правильной статистической агрегации — уроком об
агрегатах. Здесь задача уже: научиться отслеживать форму.

### Редукция удаляет ось

Редукция заменяет множество значений вдоль выбранной оси одним результатом: суммой,
средним, минимумом или другой сводкой.

Для суммы по дням:

```text
source axes:  (store, day, metric)
reduce axis:          day
result axes:  (store,      metric)

source shape: (2, 3, 2)
result shape: (2,    2)
```

По умолчанию выбранная ось исчезает, поэтому `ndim` уменьшается с `3` до `2`.

Если указать `keepdims=True`, ось остается на прежнем месте, но ее длина становится
равной `1`:

```text
source:               (store=2, day=3, metric=2)
reduce day, keepdims:  (store=2, day=1, metric=2)
```

Длина `1` не означает, что в исходных данных был один день. Она означает, что три дня
были сведены к одному агрегированному положению. Такое сохранение позиции пригодится для
совместимых операций с исходным массивом; подробно это появится в уроке о broadcasting.

Если `axis=None`, NumPy сворачивает все оси. Результат имеет форму `()` — это нульмерный
массив или скалярное значение, а не одномерный массив формы `(1,)`.

### `transpose` переставляет оси

`transpose` не объединяет и не удаляет оси. Он задает, в каком порядке старые оси должны
стать новыми:

```text
source axes:  (store, day, metric)
permutation:     2     0      1
result axes:  (metric, store, day)

source shape: (2, 3, 2)
result shape: (2, 2, 3)
```

Кортеж `(2, 0, 1)` читается так:

1. новой осью `0` станет старая ось `2` (`metric`);
2. новой осью `1` станет старая ось `0` (`store`);
3. новой осью `2` станет старая ось `1` (`day`).

Каждый старый номер должен встретиться ровно один раз. Повтор `(0, 0, 1)` не является
перестановкой: одна ось продублирована, другая потеряна.

### `reshape` перегруппировывает элементы

`reshape` меняет количество и длины осей, но сохраняет число элементов:

```text
prod(source_shape) == prod(target_shape)
```

Например, массив `(store=2, day=3, metric=2)` содержит `12` элементов. Его можно
представить как матрицу `(store_day=6, metric=2)`:

```text
(2 * 3) * 2 = 6 * 2 = 12
```

Это полезно, когда следующая функция ожидает двумерную матрицу вида
`(observation, feature)`. Но `reshape` не знает, что первые две оси означают магазин и
день. Новое имя `store_day` и соответствие каждой строки исходным координатам обязан
зафиксировать аналитик.

Одна длина `-1` просит NumPy вычислить ее:

```text
(2, 3, 2) -> (-1, 2)
12 / 2 = 6
result: (6, 2)
```

Использовать два `-1` нельзя: из общего размера невозможно однозначно вывести обе длины.

Важно: `reshape` не является перестановкой. Совпадение числа элементов еще не означает,
что значения оказались на предметно правильных местах.

### `expand_dims` добавляет ось длины один

Иногда следующий расчет ожидает дополнительную ось: например, несколько сценариев для
одного и того же набора метрик. Пока сценарий один, можно добавить ось длины `1`:

```text
source:  (store=2, day=3, metric=2)
result:  (store=2, day=3, metric=2, scenario=1)
```

Число элементов не меняется, потому что умножение на `1` не меняет произведение длин.
Но позиция новой оси важна. Формы `(1, 2, 3, 2)` и `(2, 3, 2, 1)` содержат одинаковое
число элементов, однако первая означает `(scenario, store, day, metric)`, а вторая —
`(store, day, metric, scenario)`.

### Правильная форма еще не доказывает правильный смысл

Проверка `shape` обнаруживает много ошибок, но не все. В нашем примере оси `store` и
`metric` обе имеют длину `2`:

```text
(store, day, metric) = (2, 3, 2)
```

Если случайно переставить их:

```text
(metric, day, store) = (2, 3, 2)
```

числовой `shape` вообще не изменится. Простая проверка `array.shape == (2, 3, 2)` пропустит
ошибку.

Отсюда граница проверки формы:

- `shape` проверяет количество и длины осей;
- имена осей фиксируют предполагаемый смысл;
- контракт источника, идентификаторы и контрольные значения подтверждают, что смысл
  действительно не перепутан.

### Четыре шага предсказания формы

Перед любой операцией используйте один и тот же короткий алгоритм.

1. **Запишите исходный контракт:** `(store=2, day=3, metric=2)`.
2. **Назовите действие словами:** «свернуть `day`», «поставить `metric` первой»,
   «объединить `store` и `day` в `store_day`».
3. **Преобразуйте имена осей:** удалите, сохраните, переставьте, объедините или добавьте
   нужное имя.
4. **Подставьте длины и поставьте assertion:** только после этого запускайте NumPy.

Такой порядок важнее запоминания отдельных методов. Он переносится на массивы любой
размерности и на библиотеки, которые используют ту же модель форм.

## Соберите это

Сначала построим уменьшенную модель формы без NumPy. Она не вычисляет суммы и не хранит
большой массив: она только преобразует контракт осей.

### Шаг 1. Составьте реестр осей

```python
source_shape = (2, 3, 2)
source_axis_names = ("store", "day", "metric")

assert len(source_shape) == len(source_axis_names)
assert len(set(source_axis_names)) == len(source_axis_names)

axis_registry = [
    {"axis": index, "name": name, "length": length}
    for index, (name, length) in enumerate(
        zip(source_axis_names, source_shape)
    )
]
```

Результат:

```python
[
    {"axis": 0, "name": "store", "length": 2},
    {"axis": 1, "name": "day", "length": 3},
    {"axis": 2, "name": "metric", "length": 2},
]
```

Уникальность имен не является требованием самого NumPy. Это требование нашего учебного
контракта: два одинаковых имени сделали бы дальнейшее рассуждение неоднозначным.

### Шаг 2. Нормализуйте номер оси

```python
def normalize_axis(axis, ndim):
    normalized = axis + ndim if axis < 0 else axis
    if normalized < 0 or normalized >= ndim:
        raise ValueError(
            f"axis {axis} is out of bounds for ndim={ndim}"
        )
    return normalized
```

Проверим крайние случаи:

```python
assert normalize_axis(0, 3) == 0
assert normalize_axis(-1, 3) == 2
assert normalize_axis(-3, 3) == 0
```

Оси `3` и `-4` должны завершаться ошибкой. Молчаливо исправлять их нельзя: это почти
всегда означает, что вызывающий код неверно понял размерность входа.

### Шаг 3. Предскажите редукцию

Сначала удалим выбранную ось из кортежа длин:

```python
def reduction_shape(shape, axis, keepdims=False):
    normalized = normalize_axis(axis, len(shape))
    if keepdims:
        return tuple(
            1 if index == normalized else length
            for index, length in enumerate(shape)
        )
    return tuple(
        length
        for index, length in enumerate(shape)
        if index != normalized
    )
```

Та же операция над именами:

```python
def reduction_axis_names(axis_names, axis, keepdims=False):
    normalized = normalize_axis(axis, len(axis_names))
    if keepdims:
        return tuple(axis_names)
    return tuple(
        name
        for index, name in enumerate(axis_names)
        if index != normalized
    )
```

Теперь числовое и предметное ожидания согласованы:

```python
assert reduction_shape(source_shape, 1) == (2, 2)
assert reduction_axis_names(source_axis_names, 1) == (
    "store",
    "metric",
)

assert reduction_shape(source_shape, 1, keepdims=True) == (2, 1, 2)
assert reduction_axis_names(
    source_axis_names,
    1,
    keepdims=True,
) == ("store", "day", "metric")
```

### Шаг 4. Предскажите перестановку

Для перестановки достаточно собрать длины и имена в новом порядке:

```python
permutation = (2, 0, 1)

assert sorted(permutation) == list(range(len(source_shape)))

transposed_shape = tuple(source_shape[index] for index in permutation)
transposed_names = tuple(source_axis_names[index] for index in permutation)

assert transposed_shape == (2, 2, 3)
assert transposed_names == ("metric", "store", "day")
```

Здесь `2` в начале не является новой длиной. Это номер старой оси, которую нужно поставить
первой.

### Шаг 5. Проверьте допустимость `reshape`

```python
from math import prod


def can_reshape(source, target):
    return prod(source) == prod(target)
```

```python
assert can_reshape((2, 3, 2), (6, 2))
assert not can_reshape((2, 3, 2), (5, 2))
```

После числовой проверки отдельно задайте новый предметный контракт:

```text
source: (store=2, day=3, metric=2)
target: (store_day=6, metric=2)
```

Функция `prod` способна проверить количество элементов, но не может придумать имя
`store_day` и доказать, что выбранный порядок значений соответствует этому имени.

### Шаг 6. Вставьте новую ось

Для вставки в конец:

```python
expanded_shape = (*source_shape, 1)
expanded_names = (*source_axis_names, "scenario")

assert expanded_shape == (2, 3, 2, 1)
assert expanded_names == ("store", "day", "metric", "scenario")
```

В общем случае кортеж нужно разрезать в нормализованной позиции и вставить туда длину
`1` и новое имя. Именно это делает артефакт урока.

## Используйте это

Теперь применим правила к реальному `ndarray`. Данные продолжают пример первого урока:

```python
import numpy as np

daily_metrics = np.array(
    [
        [[12, 1200], [15, 1500], [9, 900]],
        [[10, 1000], [11, 1100], [14, 1400]],
    ]
)

assert daily_metrics.shape == (2, 3, 2)
```

В каждой паре первое число — количество заказов, второе — выручка. Для реального проекта
названия и единицы измерения должны храниться в отдельном контракте; обычный `ndarray` их
не сохраняет.

### Сверните ось дней

```python
store_totals = daily_metrics.sum(axis=1)

assert store_totals.shape == (2, 2)
```

Предметный контракт результата:

```text
(store, metric) = (2, 2)
```

Мы пока не изучаем тонкости агрегирования разных метрик. Здесь важно, что NumPy применил
сумму отдельно для каждой оставшейся комбинации `store` и `metric`.

Если позиция оси нужна дальше:

```python
store_totals_kept = daily_metrics.sum(axis=1, keepdims=True)

assert store_totals_kept.shape == (2, 1, 2)
```

Контракт теперь `(store, summarized_day, metric)`. Имя можно оставить `day`, если в
документации явно сказано, что длина `1` представляет агрегат всех дней, а не один
исходный день.

### Поставьте метрики первой осью

```python
metric_first = np.transpose(daily_metrics, (2, 0, 1))

assert metric_first.shape == (2, 2, 3)
```

```text
(metric, store, day) = (2, 2, 3)
```

`np.transpose` изменил порядок доступа к тем же значениям. Он не сложил магазины и не
объединил дни.

Для двумерного массива сокращение `.T` часто удобно, но в учебном трехмерном примере
явная перестановка лучше: она показывает, откуда взялась каждая новая позиция.

### Соберите матрицу наблюдений и признаков

```python
store_day_rows = daily_metrics.reshape(-1, 2)

assert store_day_rows.shape == (6, 2)
```

```text
(store_day, metric) = (6, 2)
```

Такая форма удобна для функции, ожидающей матрицу `(observation, feature)`. Но после
объединения двух осей одной формы мало: если нужно вернуться к конкретному магазину и дню,
сохраните их идентификаторы или явное правило соответствия строк исходным координатам.

### Добавьте ось сценария

```python
with_scenario = np.expand_dims(daily_metrics, axis=-1)

assert with_scenario.shape == (2, 3, 2, 1)
```

Эквивалентная короткая запись:

```python
with_scenario = daily_metrics[..., np.newaxis]
```

Для первого знакомства `np.expand_dims` яснее: позиция новой оси названа отдельным
аргументом. Запись с `np.newaxis` станет естественнее после урока об индексации.

## Сломайте это

### Ошибка 1. Выбрать существующую, но неверную ось

```python
wrong = daily_metrics.sum(axis=0)

assert wrong.shape == (3, 2)
```

Операция не сломалась. Она получила итог по магазинам для каждого дня и каждой метрики:

```text
(day, metric) = (3, 2)
```

Если задача требовала итоги по дням для каждого магазина, расчет неверен. Лучший барьер —
до вызова записать ожидаемые имена результата `(store, metric)`, а после вызова проверить
`shape == (2, 2)`.

### Ошибка 2. Переставить две оси одинаковой длины

```python
swapped = np.transpose(daily_metrics, (2, 1, 0))

assert swapped.shape == daily_metrics.shape == (2, 3, 2)
```

Проверка формы прошла, хотя контракт изменился с `(store, day, metric)` на
`(metric, day, store)`. Поэтому критичные интерфейсы дополняют shape-проверку именами,
идентификаторами, контрольным наблюдением или более богатой схемой данных.

### Ошибка 3. Использовать `reshape` вместо `transpose`

```python
matrix = np.array(
    [
        [1, 2, 3],
        [4, 5, 6],
    ]
)

print(matrix.reshape(3, 2))
print(matrix.T)
```

Результаты различаются:

```text
reshape:       transpose:
[[1, 2],       [[1, 4],
 [3, 4],        [2, 5],
 [5, 6]]        [3, 6]]
```

Обе формы равны `(3, 2)`, но только `transpose` меняет местами исходные оси. `reshape`
читает элементы в выбранном порядке и группирует их по новым длинам.

### Ошибка 4. Изменить количество элементов через `reshape`

```python
daily_metrics.reshape(5, 2)
```

Исходный массив содержит `12` элементов, целевая форма требует `10`. NumPy остановит
операцию. Артефакт обнаруживает это по произведению длин без создания массива.

### Ошибка 5. Без разбора удалить все оси длины один

```python
values = np.zeros((1, 20, 1))
collapsed = values.squeeze()

assert collapsed.shape == (20,)
```

Без аргумента `squeeze()` удалил обе оси длины `1`. Одна могла означать единственный
сценарий, другая — единственную метрику. Если удалить нужно только конкретную ось,
указывайте ее явно и сначала записывайте ожидаемый результат.

### Ошибка 6. Случайно свернуть весь массив

```python
total = daily_metrics.sum()

assert np.shape(total) == ()
```

Отсутствующий `axis` означает редукцию по всем осям. Это подходит для одного общего итога,
но уничтожает разрезы по магазину, дню и метрике.

### Ошибка 7. Добавить ось не в ту позицию

```python
first = np.expand_dims(daily_metrics, axis=0)
last = np.expand_dims(daily_metrics, axis=-1)

assert first.shape == (1, 2, 3, 2)
assert last.shape == (2, 3, 2, 1)
```

Обе операции добавили длину `1`, но создали разные контракты. Эта разница станет особенно
важной, когда массивы начнут автоматически согласовывать формы при broadcasting.

```python
store_totals = daily_metrics.sum(axis=1)

    store_totals,
    (2, 2),
    name="store_totals",
    axis_names=("store", "metric"),

```

Сообщение об ошибке содержит роль массива, фактическую форму и ожидаемые длины с
предметными именами. Такой assertion особенно полезен на границе функций: он превращает
неявное предположение автора в исполняемый контракт.

Артефакт не доказывает, что имя `store` соответствует именно магазинам. Источник истины
для смысла — спецификация данных и контрольные наблюдения. Инструмент проверяет, что
числовая форма согласована с заявленным контрактом и что смысл не потерян незаметно во
время преобразования.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Ось (`axis`) | «Ось `0` всегда строки» | Позиция направления в текущем упорядоченном контракте массива |
| Имя оси | «NumPy хранит его внутри обычного массива» | Предметная метка, которую аналитик ведет рядом с `ndarray` |
| `ndim` | «Длина самой большой оси» | Количество осей, равное длине `shape` |
| `shape` | «Просто размер массива» | Упорядоченный кортеж длин всех осей |
| Отрицательная ось | «Особый вид оси» | Номер, отсчитанный от конца текущего `shape` |
| Редукция | «Любое уменьшение формы» | Сведение значений вдоль выбранных осей к агрегированным результатам |
| `keepdims` | «Не выполнять редукцию» | Сохранить позицию редуцированной оси с длиной `1` |
| `reshape` | «Переставить оси» | Перегруппировать то же число элементов по новым длинам |
| `transpose` | «Переписать значения в обратном порядке» | Переставить существующие оси по указанной перестановке |
| Единичная ось | «Ось без данных» | Ось длины `1`, которая все равно входит в контракт формы |
| Shape-контракт | «Проверка количества элементов» | Ожидаемые имена, порядок и длины осей на границе вычисления |
