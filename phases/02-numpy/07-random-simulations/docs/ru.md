# Случайность и воспроизводимые симуляции

> Симуляция воспроизводима, когда seed, параметры, генератор и версия среды являются явной частью эксперимента.

**Тип:** Build
**Треки:** Core
**Пререквизит:** `02-numpy/06-broadcasting`
**Время:** ~60 минут
**Результат:** использует Generator и воспроизводит симуляцию по seed.

## Цели обучения

- объяснить различие случайности и псевдослучайного потока;
- использовать локальный `np.random.Generator`;
- воспроизводить результат по seed;
- симулировать распределение выборочного среднего;
- сверять эмпирическую стандартную ошибку с теорией.

## Проблема

Аналитик оценивает разброс среднего чека с помощью симуляции. При каждом запуске числа
меняются, а коллега не может повторить результат. Попытка вызвать `np.random.seed(42)`
внутри функции исправляет один запуск, но изменяет глобальное состояние всего процесса.

Надёжный эксперимент должен:

- владеть собственным генератором;
- сохранять seed и параметры;
- иметь контрольный теоретический результат;
- ограничивать объём выделяемой памяти.

## Концепция

Псевдослучайный генератор является детерминированным алгоритмом. Начальное состояние
задаётся seed:

```text
same algorithm + same seed + same calls -> same stream
```

Это не делает значения предсказуемыми для статистической задачи, но позволяет повторить
эксперимент.

### Локальное состояние

```python
rng = np.random.default_rng(42)
```

`rng` является объектом `Generator`. Он не требует изменения общего состояния модуля.
Функция может принимать готовый generator или создавать его из явного seed.

### Распределение выборочного среднего

Если наблюдения независимы и имеют стандартное отклонение `sigma`, стандартная ошибка
среднего выборки размера `n` равна:

```text
SE = sigma / sqrt(n)
```

Симуляция должна приблизить:

- среднее выборочных средних к среднему популяции;
- стандартное отклонение выборочных средних к `SE`.

## Соберите это

Уменьшенная модель псевдослучайного генератора:

```python
def lcg(seed):
    state = seed
    while True:
        state = (1_664_525 * state + 1_013_904_223) % 2**32
        yield state / 2**32
```

```python
first = lcg(42)
second = lcg(42)

assert [next(first) for _ in range(3)] == [
    next(second) for _ in range(3)
]
```

Этот пример не предназначен для анализа. Он показывает механизм: seed задаёт состояние,
а каждый вызов переводит его в следующее состояние.

### Ручная симуляция

```python
sample_means = []
for _ in range(repetitions):
    sample = draw_sample(sample_size)
    sample_means.append(sum(sample) / sample_size)
```

Затем сравните стандартное отклонение `sample_means` с `sigma / sqrt(n)`.

## Используйте это

```python
import numpy as np

rng = np.random.default_rng(42)
samples = rng.normal(
    loc=100,
    scale=15,
    size=(10_000, 25),
)
sample_means = samples.mean(axis=1)
```

Формы:

```text
samples:      (repetitions=10000, sample_size=25)
sample_means: (10000,)
```

Контроль:

```python
theoretical_se = 15 / np.sqrt(25)
empirical_se = sample_means.std(ddof=1)

np.testing.assert_allclose(empirical_se, theoretical_se, rtol=0.03)
```

Запуск артефакта:

```bash
uv run --locked python phases/02-numpy/07-random-simulations/outputs/sampling_distribution.py \
  --mean 100 \
  --std 15 \
  --sample-size 25 \
  --repetitions 10000 \
  --seed 42
```

JSON содержит параметры, первые пять средних, квантили, эмпирическую и теоретическую
стандартные ошибки.

## Сломайте это

### Скрытый seed

```python
def simulate():
    rng = np.random.default_rng()
```

Такой вызов подходит для неповторяемого интерактивного исследования, но артефакт нельзя
воспроизвести без сохранённого seed.

### Глобальное состояние

```python
np.random.seed(42)
```

Legacy API меняет общий поток и создаёт зависимость между удалёнными частями программы.
Для нового кода используйте `Generator`.

### Один seed на каждую итерацию

```python
for _ in range(100):
    rng = np.random.default_rng(42)
    draw = rng.normal()
```

Цикл сто раз получает одно и то же первое число. Создавайте generator один раз на поток.

### Неограниченная матрица

`repetitions * sample_size` определяет число генерируемых значений. Артефакт ограничивает
его десятью миллионами, чтобы ошибочный аргумент не попытался выделить гигантский массив.

### Считать seed вечной гарантией

Seed воспроизводит поток при согласованном генераторе и окружении. Если точные значения
важны для регрессионного теста, фиксируйте зависимости через `uv.lock` и храните
небольшой ожидаемый контрольный результат.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| PRNG | «Источник физической случайности» | Детерминированный алгоритм псевдослучайной последовательности |
| Seed | «Параметр распределения» | Начальное состояние генератора |
| `Generator` | «Глобальная настройка NumPy» | Локальный объект случайного потока |
| Репликация | «Повтор того же числа» | Один повтор статистического эксперимента |
| Стандартная ошибка | «Стандартное отклонение исходных наблюдений» | Разброс оценки между повторными выборками |
