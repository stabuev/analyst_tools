## Проблема

CSV с заказами открылся и даже показывает семь строк. Это ещё не доказывает, что одна
строка соответствует одному заказу. Индекс `0..6` создан pandas автоматически и не
является бизнес-ключом. Если `order_id` повторяется, сумма выручки может быть завышена,
хотя код не выдаст ошибку.

## Концепция

`DataFrame` имеет две оси: индекс строк и метки столбцов. Один столбец — `Series` со
своим индексом. pandas выравнивает объекты по меткам, а не только по позиции. Поэтому
арифметика двух `Series` с разными индексами создаёт объединение меток и пропуски там,
где пары нет.

Grain отвечает на вопрос «что означает одна строка». Для `orders` это один заказ, а
проверяемый ключ — `order_id`. `RangeIndex` удобен технически, но ничего не говорит о
семантике данных.

В pandas 3 Copy-on-Write включён постоянно: объект, полученный выборкой, ведёт себя как
отдельный объект при записи. Изменять исходную таблицу следует явным присваиванием через
неё саму, а не надеждой на побочный эффект промежуточного объекта.

## Соберите это

Сначала проверьте grain без pandas: выпишите значения `order_id`, посчитайте частоты и
отдельно отметьте пустые ключи. Контракт проходит только при двух условиях:

```text
duplicate_key_rows == 0
null_key_rows == 0
```

Затем вручную сопоставьте две последовательности по меткам:

```text
price:    order-a=100, order-b=200
discount: order-b=10,  order-c=20
```

Результат определён только для `order-b` и равен `190`.

## Сломайте это

Добавьте второй `O1001` или пустой `order_id`. Форма таблицы останется допустимой,
`RangeIndex` останется уникальным, но grain-контракт должен стать красным. Затем сложите
две `Series` с переставленными индексами и сравните результат с позиционным сложением
списков.

## Проверьте это

- Число строк и столбцов совпадает с ожидаемым.
- Все заявленные ключи существуют.
- Ключ не содержит пропусков и дубликатов.
- Выравнивание объясняется метками результата.
- Изменение выборки не используется как способ изменить исходный `DataFrame`.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Grain | Число строк | Смысл одной строки и допустимый ключ |
| Index | Всегда бизнес-ключ | Метки строк, которые могут быть техническими |
| Alignment | Операция по позиции | Сопоставление значений по меткам |
| Copy-on-Write | Любая выборка изменяет источник | Запись в производный объект отделяет его данные |
