# Выбор строк и столбцов

> Хороший фильтр — это именованное логическое условие с явной политикой пропусков.

**Тип:** Build
**Треки:** Core
**Пререквизиты:** 03/02
**Время:** ~75 минут
**Результат:** составляет булеву маску и изменяет строки одним присваиванием через `loc`.

## Цели обучения

- Различать выбор по меткам и позициям.
- Правильно комбинировать условия `&`, `|` и `~`.
- Определять, что означает `NA` в маске.
- Избегать chained assignment.

## Проблема

Нужно выбрать оплаченные заказы дороже 70 единиц. Статусы записаны как `paid`, `PAID` и
` paid `, а одна сумма отсутствует. Выражение без нормализации потеряет строки; выражение
без скобок может изменить приоритет операций; цепочка `df[mask]["flag"] = True` не
является способом изменить исходную таблицу.

## Концепция

Маска должна иметь тот же индекс, что и таблица. Каждая позиция отвечает на вопрос,
проходит ли соответствующая строка условие. Nullable boolean допускает третье состояние
`NA`; аналитик обязан решить, исключать такую строку или считать неопределённость ошибкой.

`loc` выбирает по меткам и подходит для одного явного присваивания:

```python
result.loc[mask, "review_status"] = "review"
```

`iloc` выбирает по позиции и не выражает бизнес-условие.

## Соберите это

Разбейте фильтр на независимые предикаты:

```text
normalized_status in allowed_statuses
normalized_currency in allowed_currencies
amount >= lower_bound
amount <= upper_bound
```

Сначала проверьте каждый предикат и число прошедших строк, затем соедините их AND.

## Используйте это

```python
status = orders["status"].astype("string").str.strip().str.lower()
amount = pd.to_numeric(orders["amount"], errors="coerce")
mask = status.eq("paid") & amount.ge(70)
selected = orders.loc[mask.fillna(False)].copy()
```

```bash
uv run --locked python outputs/safe_selection.py \
  ../data/tiny/orders.csv --status paid --min-amount 70
```

## Сломайте это

Уберите скобки вокруг сравнений, поменяйте нижнюю и верхнюю границу местами, передайте
маску от другой таблицы или оставьте `NA` без политики. Каждая ошибка должна быть либо
отклонена, либо отражена в ожидаемом количестве строк.

## Проверьте это

- Индекс маски совпадает с индексом таблицы.
- Границы включены или исключены осознанно.
- Нормализация строк происходит до сравнения.
- Пропуск суммы не проходит числовой фильтр молча как ноль.
- Возвращённая таблица не изменяет источник побочным эффектом.

1. Выберите RUB-заказы со статусами paid или refunded.
2. Добавьте политику, которая отклоняет любой пропуск в используемых полях.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Boolean mask | Список номеров строк | Логическая Series, выровненная по индексу |
| `loc` | Только чтение | Выбор и присваивание по меткам |
| Chained assignment | Короткая запись `loc` | Две независимые операции, не задающие явную цель записи |
| Missing policy | Всегда `fillna(False)` | Бизнес-решение об обработке неопределённого условия |
