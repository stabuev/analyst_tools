# Экспорт и передача результата

> Передаётся не только файл, но и контракт: grain, схема, проверки, источники и checksum.

**Тип:** Case
**Треки:** Core
**Пререквизиты:** 03/10
**Время:** ~105 минут
**Результат:** строит витрину из трёх грязных таблиц и экспортирует её с manifest.

## Цели обучения

- Интегрировать users, orders и order_items без изменения grain.
- Сохранять unmatched-сущности как наблюдаемый quality signal.
- Проверять согласованность сумм.
- Поставлять воспроизводимый артефакт с checksum.

## Концепция

Итоговый grain — один заказ. Поэтому:

1. users должен иметь уникальный `user_id`;
2. orders — уникальный `order_id`;
3. items агрегируется до `order_id`;
4. users присоединяется `many_to_one`;
5. число строк сверяется с orders.

Неизвестный `U999` не удаляется: строка заказа сохраняется, а `user_found=False` попадает
в quality checks. Manifest фиксирует grain, строки, столбцы, dtype, source checksums и
checksum выходного CSV.

## Соберите это

Нарисуйте grain каждого входа и целевого результата. Вручную вычислите item_total для
`O1001`: `1*800 + 2*200 = 1200`. Определите, какие поля должны быть nullable после left
join и какие unmatched-сущности допустимы.

## Используйте это

```bash
uv run --locked python outputs/mart_builder.py \
  --users ../data/tiny/users.csv \
  --orders ../data/tiny/orders.csv \
  --items ../data/tiny/order_items.csv \
  --output-dir mart
```

Результат:

```text
mart/order_mart.csv
mart/manifest.json
```

CSV удобен для передачи и ручной инспекции. Manifest делает конкретный файл
идентифицируемым и проверяемым. В следующих фазах формат хранения будет расширен Parquet,
Arrow и SQL-витринами.

## Сломайте это

Продублируйте заказ, добавьте неизвестную позицию, измените количество, удалите пользователя
и испортите timestamp. Проверьте, какие дефекты блокируют сборку, какие сохраняются в mart,
а какие становятся счётчиками quality checks.

## Проверьте это

- `order_id` уникален и непуст.
- Число строк mart равно числу orders.
- Item detail агрегирован до join.
- Unknown user сохранён и посчитан.
- Суммы orders и items сверены.
- SHA-256 manifest совпадает с фактическим CSV.

## Поставьте результат

`outputs/mart_builder.py` является интеграционным мини-проектом фазы. Его core-функция
работает с DataFrame, а CLI отвечает за I/O. Артефакт можно запустить без текста урока:
usage находится в `outputs/artifact.json`, manifest описывает результат.

## Упражнения

1. Добавьте параметр бизнес-timezone и локальную дату заказа.
2. Сделайте mismatch суммы блокирующим режимом.
3. Добавьте schema version и сравнение двух manifest перед передачей.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Mart | Любой финальный CSV | Таблица с определённым grain и назначением |
| Manifest | Человеческий README | Машинное описание конкретного артефакта |
| Checksum | Проверка смысла | Проверка идентичности байтов |
| Handoff | Отправка файла | Передача файла, контракта, проверок и инструкции |
