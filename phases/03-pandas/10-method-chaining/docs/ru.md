# Method chaining и читаемые пайплайны

> Цепочка хороша, когда каждый шаг имеет имя, контракт и один понятный уровень абстракции.

**Тип:** Build
**Треки:** Core
**Пререквизиты:** 03/09
**Время:** ~75 минут
**Результат:** собирает чистый pipeline из `pipe` и `assign` с проверками между стадиями.

## Цели обучения

- Писать чистые функции DataFrame -> DataFrame.
- Использовать `pipe` для доменных стадий.
- Использовать `assign` для связанных столбцов.
- Размещать инварианты внутри pipeline.

## Проблема

Notebook из десятков присваиваний зависит от порядка ячеек и позволяет использовать
полуобработанную таблицу. Слишком длинная цепочка без имён тоже плоха: в ней трудно найти,
на каком шаге изменился grain или dtype.

## Концепция

Pipeline — композиция чистых стадий. Каждая функция принимает таблицу, проверяет
preconditions и возвращает новый результат. `pipe` делает доменный шаг видимым,
`assign` создаёт столбцы без промежуточных мутаций.

Инварианты размещаются рядом с этапом, после которого должны стать истинными:
уникальный order_id проверяется до преобразований, корректность paid_amount — после его
создания.

## Соберите это

Опишите граф:

```text
raw -> validate grain -> normalize -> add UTC/calendar -> add paid metrics
    -> verify invariants -> sort -> result
```

Для каждой стрелки запишите входные столбцы, выходные столбцы и возможную ошибку.

## Используйте это

```python
result = (
    orders.copy()
    .pipe(validate_order_grain)
    .pipe(normalize_columns)
    .pipe(add_time_columns, timezone="Europe/Moscow")
    .pipe(add_paid_metrics)
    .pipe(check_invariants)
    .sort_values("order_id")
    .reset_index(drop=True)
)
```

## Сломайте это

Добавьте дубликат ключа, неверную дату и функцию, которая изменяет входной объект inplace.
Проверка должна завершаться на ближайшей стадии с доменным сообщением, а исходный DataFrame
должен оставаться без изменений.

## Проверьте это

- Все стадии можно тестировать отдельно.
- Исходная таблица не мутирует.
- Grain проверяется до вычислений.
- Ошибки времени локализованы во временной стадии.
- Финальные производные столбцы удовлетворяют инвариантам.

1. Добавьте стадию нормализации категорий.
2. Вставьте аудит числа строк после каждого этапа.
3. Сравните цепочку с процедурной версией и найдите места скрытого состояния.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Method chain | Максимально длинное выражение | Последовательность читаемых преобразований |
| Pure stage | Функция без файлов | Функция без скрытой мутации и состояния |
| `pipe` | Аналог shell pipe без аргументов | Передача DataFrame в доменную функцию |
| Checkpoint | Только сохранение файла | Проверка инварианта между стадиями |
