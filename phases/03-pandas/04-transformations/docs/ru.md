# Преобразования без apply по умолчанию

> Векторное выражение показывает операцию над столбцом целиком и обычно проще проверяется, чем построчная функция.

**Тип:** Build
**Треки:** Core
**Пререквизиты:** 03/03
**Время:** ~90 минут
**Результат:** строит производные признаки строковыми, числовыми и условными операциями.

## Цели обучения

- Нормализовать строки через `.str`.
- Преобразовывать числа один раз и переиспользовать результат.
- Использовать `where`, `mask`, `cut` и арифметику столбцов.
- Рассматривать `apply(axis=1)` как исключение, а не стартовый API.

## Проблема

Построчная функция кажется универсальной: взять строку, очистить статус, проверить
оплату и рассчитать сумму. Но она скрывает разные операции в одном Python-вызове,
затрудняет проверку dtype и часто работает медленнее. Кроме того, ручная ветка легко
расходится с логикой фильтра из предыдущего урока.

## Концепция

Векторное преобразование выражает одно правило для всего столбца. Нормализованный статус
строится через строковые методы, `is_paid` — сравнением, `paid_amount` — `where`.
Каждый промежуточный столбец имеет понятный dtype и инвариант.

`apply` оправдан, если операция действительно не раскладывается на доступные массивные
примитивы и измерение подтверждает приемлемую стоимость. Само наличие пользовательской
функции не является таким доказательством.

## Используйте это

```python
status = orders["status"].astype("string").str.strip().str.lower()
amount = pd.to_numeric(orders["amount"], errors="coerce").astype("Float64")
is_paid = status.eq("paid").astype("boolean")
paid_amount = amount.where(is_paid, 0)
```

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Vectorization | Всегда NumPy без pandas | Операция над массивом или столбцом целиком |
| `where` | Только фильтр строк | Выбор значения по условию с сохранением формы |
| `apply(axis=1)` | Обычный способ создать столбец | Построчный Python-callback с дополнительной стоимостью |
| Invariant | Пример результата | Условие, которое истинно для всех допустимых строк |
