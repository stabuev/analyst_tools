# GroupBy и единица анализа

> До агрегации назовите grain входа и grain результата.

**Тип:** Case
**Треки:** Core
**Пререквизиты:** 03/04
**Время:** ~105 минут
**Результат:** считает сегментные метрики и подтверждает их ручным контрольным расчётом.

## Цели обучения

- Разделять split, apply и combine в `groupby`.
- Объявлять ключи результата до расчёта.
- Использовать именованные агрегации.
- Проверять сумму на tiny-наборе вручную.

## Проблема

Нужно посчитать оплаченную выручку по валютам. Если исходная таблица уже содержит две
строки на заказ, `sum` честно сложит дубли. Если сгруппировать одновременно по валюте и
пользователю, grain результата изменится. Сам вызов `groupby` не знает, какой уровень
детализации нужен бизнесу.

## Концепция

Агрегация уменьшает детализацию. Входной grain здесь — один заказ; результат — одна строка
на комбинацию `group_by`. Ключи результата должны быть уникальны. Метрики имеют разные
правила: `nunique(order_id)`, сумма суммы и среднее по заказам нельзя заменять одним
универсальным `count`.

`dropna=False` делает группу с пропущенным ключом видимой. `observed=True` не создаёт
не встречавшиеся комбинации категорий.

## Соберите это

На списке словарей отберите paid-заказы, заведите словарь `currency -> total` и сложите
суммы. Для tiny-данных контроль:

```text
RUB=1200, KZT=5500, USD=49.99, EUR=75.50
```

Отдельно посчитайте число заказов и пользователей: эти показатели не обязаны совпадать.

## Используйте это

```python
result = (
    paid.groupby(["currency"], dropna=False, observed=True)
    .agg(
        paid_orders=("order_id", "nunique"),
        paid_amount=("amount", "sum"),
        average_order_value=("amount", "mean"),
    )
    .reset_index()
)
```

```bash
uv run --locked python outputs/segment_metrics.py \
  ../data/tiny/orders.csv --group-by currency
```

Проверьте, какая ошибка меняет grain входа, какая создаёт отдельную группу, а какая меняет
смысл метрики.

## Проверьте это

- `order_id` уникален до агрегации.
- Ключи результата уникальны после неё.
- Фильтр paid совпадает с предыдущим уроком.
- Ручная сумма tiny-набора совпадает с pandas.
- Название каждой метрики отражает функцию агрегации.

## Поставьте результат

`outputs/segment_metrics.py` принимает DataFrame и список группирующих столбцов. Функция
возвращает таблицу на объявленном grain и отклоняет дубли исходного заказа.

## Упражнения

1. Посчитайте метрики по `currency,user_id`.
2. Добавьте долю оплаченных заказов, сохранив корректный знаменатель.
3. Проверьте поведение при пропущенном ключе с `dropna=True` и `False`.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Group key | Просто столбец | Ключ grain результата |
| Aggregation | Любая функция | Операция, уменьшающая детализацию |
| `size` | Число уникальных сущностей | Число строк группы |
| Control total | Ещё одна метрика | Независимый ручной эталон |
