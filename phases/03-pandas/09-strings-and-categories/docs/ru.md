# Строки и категориальные типы

> Нормализация текста и ограничение словаря — разные этапы с разными failure modes.

**Тип:** Build
**Треки:** Core
**Пререквизиты:** 03/08
**Время:** ~75 минут
**Результат:** сводит строковые варианты к канону и не теряет неизвестные категории.

## Цели обучения

- Использовать nullable string и `.str`.
- Нормализовать пробелы, регистр и разделители.
- Сводить известные синонимы явной картой.
- Управлять неизвестными значениями при переходе к `category`.

## Проблема

`add-on`, `Addon` и ` add-on ` обозначают одну категорию, но дают три группы. Простое
приведение к `category` не исправит значения: оно лишь зафиксирует три разных метки.
Если позже появится новая категория, присваивание в закрытый словарь может превратить её
в пропуск или ошибку без понятной бизнес-политики.

## Концепция

Pipeline состоит из трёх шагов:

1. техническая нормализация строки;
2. сведение известных синонимов;
3. проверка против допустимого словаря.

Пропуск отличается от неизвестной непустой категории. Пропуск сохраняется как `pd.NA`,
неизвестное значение либо отклоняется, либо явно отображается в `other`.

`category` полезен для повторяющегося ограниченного словаря и фиксирует порядок категорий.
Для свободного текста он не является подходящим контрактом.

## Соберите это

На списке значений выполните `strip`, lowercase и замену пробелов/дефисов на `_`.
После этого примените словарь `addon -> add_on`. Сравните множество результата с
разрешёнными категориями.

## Используйте это

```python
normalized = (
    items["category"]
    .astype("string")
    .str.strip()
    .str.lower()
    .str.replace(r"[\s-]+", "_", regex=True)
    .replace({"addon": "add_on"})
)
items["category"] = normalized.astype(
    pd.CategoricalDtype(["add_on", "subscription", "service"])
)
```

```bash
uv run --locked python outputs/text_categories.py \
  ../data/tiny/users.csv --items ../data/tiny/order_items.csv
```

- Все известные варианты получают один канон.
- Синонимы задаются явной картой.
- Неизвестные значения видимы.
- Пропуски сохраняются отдельно.
- Категориальный словарь перечислен в отчёте.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Canonical value | Самый частый вариант | Единственное договорённое представление |
| Alias | Ошибка, которую удаляют | Известный вариант, отображаемый в канон |
| Category vocabulary | Все строки столбца | Явный набор допустимых значений |
| Unknown | То же, что missing | Непустое значение вне словаря |
