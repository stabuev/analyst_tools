# Excel как источник: листы, диапазоны и формулы

> Excel-книга является интерфейсом для человека; таблица внутри нее должна получить отдельный машинный контракт.

**Тип:** Case  
**Треки:** Core  
**Пререквизиты:** 05/01  
**Время:** ~60 минут  
**Результат:** выбирает лист и диапазон, распознает служебное оформление и преобразует
рабочую книгу в явную табличную схему.

## Цели обучения

- Отличать книгу, лист и извлекаемый табличный диапазон.
- Обнаруживать merged cells, hidden columns и служебные строки.
- Разделять текст формулы и сохраненное вычисленное значение.
- Читать Excel через pandas только после структурной проверки.

## Проблема

Финансовый отчет открывается красивой титульной страницей. Данные находятся на втором
листе, заголовок начинается с четвертой строки, а справа есть формулы. После ручной правки
автор добавляет строку «сформировано повторно», и привычный `read_excel(path)` создает
столбцы `Unnamed`.

Ошибка здесь не в Excel как таковом. Код смешал три решения: какой лист является
источником, где начинается таблица и можно ли доверять вычисленным ячейкам.

## Концепция

Спецификация извлечения фиксирует:

| Поле | Решение |
|---|---|
| `sheet` | точное имя рабочего листа |
| `range` | прямоугольник исходной таблицы |
| `header_row` | строка имен столбцов |
| `columns` | ожидаемый порядок и набор полей |
| `formula_policy` | допустимы ли формулы внутри данных |

Merged cells, цвета и скрытые столбцы относятся к оформлению, но их нужно видеть:
поставщик может спрятать служебный идентификатор или объединить заголовок над таблицей.

`data_only=False` возвращает формулу вроде `=D5*F5`. `data_only=True` возвращает последнее
значение, сохраненное Excel или другим расчетным приложением. openpyxl не вычисляет
формулы, поэтому отсутствие cached value нельзя заменять нулем.

## Соберите это

Сначала откройте книгу как структуру:

```python
book = load_workbook(path, data_only=False)
print(book.sheetnames)
sheet = book["Заказы"]
```

Затем зафиксируйте диапазон `A4:F9`: первая строка является header, следующие пять строк
имеют grain «одна строка на заказ». Столбец `G` содержит формулы и намеренно не входит в
извлечение.

Для каждой ячейки диапазона проверьте координату и значение. Отдельно соберите:

```python
merged = [str(value) for value in sheet.merged_cells.ranges]
formulas = [
    cell.coordinate
    for row in sheet.iter_rows()
    for cell in row
    if isinstance(cell.value, str) and cell.value.startswith("=")
]
```

Запустите наблюдаемый пример:

```bash
uv run --locked python code/main.py
```

## Используйте это

После preflight-проверки pandas получает те же решения явно:

```python
frame = pd.read_excel(
    path,
    sheet_name="Заказы",
    header=3,
    usecols="A:F",
    nrows=5,
    engine="openpyxl",
    keep_default_na=False,
)
```

`header=3` использует нулевую индексацию, поэтому соответствует строке Excel 4. Аудитор
сравнивает результат pandas со спецификацией, а не считает успешное чтение доказательством
корректности.

## Проверьте это

Контрольные факты:

- книга содержит листы `Инструкция`, `Заказы`, `Справочник`;
- рабочий header находится в строке 4;
- диапазон `A4:F9` содержит пять заказов и шесть столбцов;
- `A1:G1` является merged range;
- `H` скрыт, а формулы находятся в `G5:G9`;
- сдвинутая книга проваливает header check;
- `data_only=True` не вычисляет формулу.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Workbook | Одна таблица | Контейнер листов, свойств и оформления |
| Sheet | Всегда данные | Отдельная поверхность книги, в том числе служебная |
| Range | Все непустые ячейки | Явно выбранный прямоугольник источника |
| Cached value | Результат openpyxl | Значение формулы, ранее сохраненное расчетным приложением |
| Merged cell | Повторенное значение | Значение только в верхней левой ячейке объединения |
