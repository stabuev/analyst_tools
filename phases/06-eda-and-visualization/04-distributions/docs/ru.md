# Распределения и выбросы

> Показывайте форму и хвосты данных; одно среднее не является описанием распределения.

**Тип:** Case  
**Треки:** Core  
**Пререквизиты:** 06/03  
**Время:** ~90 минут  
**Результат:** сравнивает форму распределений через histogram, ECDF и robust summaries,
обосновывает bins и scale и показывает выбросы без автоматического удаления.

## Цели обучения

- Строить histogram с документированной bin policy.
- Читать ECDF без зависимости от интервалов.
- Использовать median, IQR и fences вместе с исходным хвостом.
- Разделять невозможные значения и редкие допустимые наблюдения.

## Проблема

Среднее onboarding может вырасти из-за общего сдвига, длинного правого хвоста или одной
ошибки `-1`. Одинаковое среднее также возможно у распределений с разной формой.
Автоматическое удаление всех «выбросов» делает картинку аккуратнее, но может убрать
реальную проблему Android.

## Концепция

Histogram группирует значения, поэтому вывод зависит от edges. Правило
Freedman-Diaconis использует IQR и размер выборки:

```text
bin width = 2 * IQR / cube_root(n)
```

ECDF для каждого `x` показывает долю наблюдений `<= x` и не требует bins. Median и IQR
устойчивее среднего к правому хвосту, но не заменяют сам график.

Fence `Q3 + 1.5 * IQR` является диагностическим правилом, а не командой удалить строку.

## Соберите это

Реализуйте median и ECDF на списках:

```bash
uv run --locked python code/main.py
```

Последняя точка ECDF обязана равняться `(maximum, 1.0)`. Экстремум остается в
отсортированной выборке.

## Используйте это

```bash
uv run --locked python outputs/distribution_panel.py \
  --input ../data/tiny/user_journeys.csv \
  --column onboarding_seconds \
  --output-dir distribution-output
```

Артефакт отдельно сообщает отрицательные значения, но строит панель по допустимым.
Валидный `3600` остается в histogram, ECDF и summary. JSON фиксирует edges, scale,
квантили, число значений выше fence и checksum фигуры.

## Сломайте это

1. Сравните histogram с разными edges.
2. Удалите все значения выше fence и пересчитайте median и maximum.
3. Включите log scale при наличии нуля: артефакт должен остановиться.
4. Смешайте отрицательный дефект с допустимым хвостом.

## Проверьте это

- `-1` отделен как invalid;
- `3600` остается maximum;
- edges покрывают полный диапазон;
- ECDF заканчивается в `1`;
- оба Axes используют одну исходную выборку.

```bash
uv run --locked python -m unittest discover -s tests
```

## Упражнения

1. Добавьте общие edges для сравнения платформ.
2. Реализуйте winsorized summary, не меняя исходную фигуру.
3. Добавьте log scale только после явной проверки нулей.

## Ключевые термины

| Термин | Распространенное заблуждение | Точное значение |
|---|---|---|
| Histogram | Объективная форма | Форма, зависящая от bin edges |
| ECDF | Сглаженная линия | Эмпирическая накопленная доля |
| IQR | Полный диапазон | Разность Q3 и Q1 |
| Outlier fence | Правило удаления | Диагностический порог |
| Heavy tail | Ошибка данных | Медленно убывающий хвост допустимых значений |
