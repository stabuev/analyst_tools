01

По какому признаку надежнее всего различать аналитические роли?

Названия ролей меняются между компаниями, а повторяющиеся решения и артефакты описывают реальную работу.

02

Что в первую очередь отличает analytics engineer от продуктового аналитика?

Главное различие находится в ответственности и поставляемом слое данных, а не в формальном владении языком.

Карта профессии аналитика

Выбирайте маршрут по типу решений и артефактов, а не по модному названию должности.

Тип: Case
Треки: Core
Пререквизиты: Нет
Время: ~45 минут
Результат: различает основные аналитические роли и выбирает обоснованный маршрут курса

Цели обучения

  • Различать роли по ежедневным вопросам, ответственности и итоговым артефактам.
  • Отделять обязательное аналитическое ядро от профессиональной специализации.
  • Выбирать маршрут по желаемой работе, не выдавая интерес за уже освоенный навык.
  • Объяснять, почему соседние роли пересекаются, но не являются взаимозаменяемыми.

Проблема

Названия аналитических ролей нестабильны. В одной компании product analyst проектирует эксперименты, в другой строит дашборды, а в третьей пишет production SQL. Data analyst может означать и начинающего специалиста по отчетности, и сильного исследователя. Если выбирать обучение только по названию вакансии, легко потратить месяцы на инструменты, которые не приближают к желаемой работе.

Надежнее смотреть на три вещи: какой вопрос человек получает на входе, какое решение принимается по результату и какой артефакт должен остаться после работы. Библиотеки идут после этого выбора.

Концепция

Все маршруты курса опираются на общее ядро: воспроизводимую среду, NumPy, pandas, SQL, визуализацию и проверки. Специализация начинается там, где меняется основной тип решения.

Маршрут Главный вопрос Типичный артефакт Характерная ошибка
Базовый аналитик Что произошло и что делать дальше? Проверенный отчет или регулярный расчет Красивый результат без контроля данных
Продуктовый аналитик Почему изменилась метрика и сработало ли изменение? Система метрик, исследование, эксперимент Причинный вывод из наблюдаемой корреляции
Аналитик временных рядов Что произойдёт с метрикой и насколько прогноз надёжен? Forecast package, backtesting и anomaly policy Утечка будущего и сравнение без сезонного baseline
Analytics Engineer Как сделать надежный слой данных для других аналитиков? Витрина, тесты, lineage, контракт Быстрый SQL без grain и владельца
ML-аналитик Можно ли предсказывать исход и использовать прогноз в решении? Baseline, pipeline, model card Leakage и оптимизация метрики без стоимости ошибки

Это не четыре уровня старшинства. Сильный analytics engineer не является «следующей ступенью» продуктового аналитика, а ML не является обязательным финалом аналитической карьеры.

Соберите это

Сначала постройте решение без рекомендательной системы.

Шаг 1: выпишите желаемые результаты работы

Возьмите пять будущих задач и для каждой заполните:

Задача Кто принимает решение Итоговый артефакт Как проверить качество
Почему снизилась активация? Product manager Исследование и рекомендация Метрика воспроизводится, альтернативы проверены
Подготовить слой заказов Аналитики команды Витрина и контракт Ключи, свежесть и суммы проходят тесты

Не записывайте библиотеки. «Использовать CatBoost» не является бизнес-результатом.

Шаг 2: присвойте задачам маршруты

Для каждой строки выберите маршрут по артефакту:

  • отчет, исследование, автоматизированный расчет: базовый;
  • метрики, поведение, эксперимент, причинный вопрос: продуктовый;
  • прогноз временной метрики, сезонность, интервалы и аномалии: временные ряды;
  • витрина, контракт, lineage, производительность: analytics engineering;
  • прогноз, ранжирование, классификация, model card: ML.

Подсчитайте голоса. Ничья означает пересечение интересов, а не ошибку. В этом случае выберите основной маршрут по ближайшему рабочему результату, а вторую ветку оставьте дополнительной.

Используйте это

Артефакт outputs/route_advisor.py автоматизирует ту же прозрачную схему. Он задает пять вопросов, показывает баллы всех маршрутов и не скрывает ничью.

Интерактивный запуск:

bash
python3 outputs/route_advisor.py

Воспроизводимый запуск:

bash
python3 outputs/route_advisor.py \
  --answers product,product,basic,product,data \
  --json

CLI не оценивает текущие знания. Для placement-диагностики используется /find-your-level: интерес определяет маршрут, а знания определяют стартовую фазу.

Сломайте это

Проверьте четыре неудачных сценария:

  1. Выберите ML только потому, что хотите изучать сложные модели, но укажите итоговым результатом регулярную витрину.
  2. Дайте по одному голосу всем маршрутам и посмотрите, как CLI показывает неопределенность.
  3. Передайте неизвестный код ответа и убедитесь, что программа останавливается с понятной ошибкой.
  4. Попробуйте пропустить Core, хотя желаемый маршрут использует pandas, SQL и тесты.

Главная поломка здесь методологическая: рекомендатор не должен путать желаемую роль с готовностью начать ее продвинутые фазы.

Проверьте это

Хорошая рекомендация отвечает на вопросы:

  • Совпадает ли маршрут с желаемыми артефактами?
  • Показаны ли альтернативы при близких баллах?
  • Отделен ли выбор специализации от входной диагностики?
  • Включает ли маршрут все пререквизиты?
  • Можно ли объяснить рекомендацию без ссылки на «алгоритм так решил»?

Тесты в tests/test_main.py проверяют устойчивость выбора, ничью и ошибочный ввод.

Поставьте результат

Урок поставляет CLI outputs/route_advisor.py. Он использует только стандартную библиотеку, работает интерактивно и принимает ответы через аргумент командной строки. Его можно встроить в onboarding курса или использовать как основу для формы выбора маршрута.

Manifest и инструкция находятся в outputs/artifact.json.

Упражнения

  1. Заполните ручную таблицу для трех задач из своей текущей работы.
  2. Сравните маршруты временных рядов и ML на задаче прогноза churn: какой route выбирается для агрегированной метрики, а какой — для score отдельного пользователя?
  3. Измените scoring: один вопрос может давать баллы двум маршрутам. Проверьте, не стала ли рекомендация менее объяснимой.

import importlib.util
from pathlib import Path
from unittest import TestCase


def load_advisor():
    path = Path(__file__).resolve().parents[1] / "outputs" / "route_advisor.py"
    spec = importlib.util.spec_from_file_location("route_advisor_test", path)
    if spec is None or spec.loader is None:
        raise RuntimeError("Cannot load route advisor")
    module = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(module)
    return module


class RouteAdvisorTest(TestCase):
    @classmethod
    def setUpClass(cls) -> None:
        cls.advisor = load_advisor()

    def test_data_answers_recommend_analytics_engineer(self) -> None:
        result = self.advisor.build_recommendation(["data"] * 5)
        self.assertEqual(result["recommended"], "Analytics Engineer")
        self.assertEqual(result["scores"]["Analytics Engineer"], 5)

    def test_forecast_answers_recommend_time_series_route(self) -> None:
        result = self.advisor.build_recommendation(["forecast"] * 5)
        self.assertEqual(result["recommended"], "Аналитик временных рядов")
        self.assertEqual(result["path"], "00-09 -> 14 -> 17 -> 18")

    def test_tie_is_visible(self) -> None:
        result = self.advisor.build_recommendation(
            ["product", "data", "product", "data", "basic"]
        )
        self.assertEqual(
            result["tied_routes"],
            ["Продуктовый аналитик", "Analytics Engineer"],
        )

    def test_invalid_answer_is_rejected(self) -> None:
        with self.assertRaisesRegex(ValueError, "Unknown answer"):
            self.advisor.build_recommendation(
                ["product", "data", "unknown", "ml", "basic"]
            )

    def test_answer_count_is_checked(self) -> None:
        with self.assertRaisesRegex(ValueError, "Expected 5 answers"):
            self.advisor.build_recommendation(["product"])

from __future__ import annotations

import argparse
import json
from collections import Counter
from collections.abc import Sequence

ROUTE_ORDER = (
    "Базовый аналитик",
    "Продуктовый аналитик",
    "Аналитик временных рядов",
    "Analytics Engineer",
    "ML-аналитик",
)

ANSWER_TO_ROUTE = {
    "basic": "Базовый аналитик",
    "product": "Продуктовый аналитик",
    "forecast": "Аналитик временных рядов",
    "data": "Analytics Engineer",
    "ml": "ML-аналитик",
}

ROUTE_INFO = {
    "Базовый аналитик": {
        "path": "00-10 -> 17 -> 18",
        "focus": "исследования, проверенные расчеты и доставка выводов",
    },
    "Продуктовый аналитик": {
        "path": "00-10 -> 13 -> 17 -> 18",
        "focus": "метрики, эксперименты и причинные продуктовые решения",
    },
    "Аналитик временных рядов": {
        "path": "00-09 -> 14 -> 17 -> 18",
        "focus": "временные метрики, backtesting, интервалы прогноза и anomaly policy",
    },
    "Analytics Engineer": {
        "path": "00-07 -> 11-12 -> 17 -> 18",
        "focus": "надежные модели данных, тесты и производительные пайплайны",
    },
    "ML-аналитик": {
        "path": "00-07 -> 09 -> 12 -> 15-18",
        "focus": "честные predictive baselines, интерпретация и доставка моделей",
    },
}

QUESTIONS = (
    "Какой результат вы чаще хотите поставлять?",
    "Какой тип ошибки вам интереснее предотвращать?",
    "Какой вопрос вы хотите получать от заказчика?",
    "Какой артефакт вы хотели бы защищать на ревью?",
    "Какой рабочий день кажется наиболее привлекательным?",
)

OPTION_LABELS = {
    "basic": "отчет, расчет или понятная рекомендация",
    "product": "метрика, эксперимент или исследование поведения",
    "forecast": "временной прогноз, backtesting или мониторинг аномалий",
    "data": "витрина, контракт, lineage или тест данных",
    "ml": "прогноз, pipeline или model card",
}


def score_answers(answers: Sequence[str]) -> dict[str, int]:
    if len(answers) != len(QUESTIONS):
        raise ValueError(f"Expected {len(QUESTIONS)} answers, got {len(answers)}")
    unknown = [answer for answer in answers if answer not in ANSWER_TO_ROUTE]
    if unknown:
        raise ValueError(f"Unknown answer: {unknown[0]}")
    counts = Counter(ANSWER_TO_ROUTE[answer] for answer in answers)
    return {route: counts.get(route, 0) for route in ROUTE_ORDER}


def build_recommendation(answers: Sequence[str]) -> dict[str, object]:
    scores = score_answers(answers)
    maximum = max(scores.values())
    tied_routes = [route for route in ROUTE_ORDER if scores[route] == maximum]
    recommended = tied_routes[0]
    return {
        "recommended": recommended,
        "tied_routes": tied_routes,
        "scores": scores,
        "path": ROUTE_INFO[recommended]["path"],
        "focus": ROUTE_INFO[recommended]["focus"],
    }


def format_recommendation(result: dict[str, object]) -> str:
    lines = [
        f"Рекомендованный маршрут: {result['recommended']}",
        f"Фазы: {result['path']}",
        f"Фокус: {result['focus']}",
        "Баллы:",
    ]
    scores = result["scores"]
    if not isinstance(scores, dict):
        raise TypeError("scores must be a dictionary")
    lines.extend(f"  {route}: {scores[route]}" for route in ROUTE_ORDER)
    tied_routes = result["tied_routes"]
    if isinstance(tied_routes, list) and len(tied_routes) > 1:
        lines.append("Ничья: " + ", ".join(tied_routes))
        lines.append("Выберите основной маршрут по ближайшему рабочему артефакту.")
    return "\n".join(lines)


def ask_answers() -> list[str]:
    answers: list[str] = []
    option_text = ", ".join(f"{key}={label}" for key, label in OPTION_LABELS.items())
    for question in QUESTIONS:
        print(f"\n{question}")
        print(option_text)
        answers.append(input("> ").strip().lower())
    return answers


def parse_args() -> argparse.Namespace:
    parser = argparse.ArgumentParser(description="Recommend an Analyst Tools course route")
    parser.add_argument(
        "--answers",
        help="Five comma-separated answers: basic, product, forecast, data, or ml",
    )
    parser.add_argument("--json", action="store_true", help="Print machine-readable output")
    return parser.parse_args()


def main() -> None:
    args = parse_args()
    answers = (
        [answer.strip().lower() for answer in args.answers.split(",")]
        if args.answers
        else ask_answers()
    )
    result = build_recommendation(answers)
    if args.json:
        print(json.dumps(result, ensure_ascii=False, indent=2))
    else:
        print("\n" + format_recommendation(result))


if __name__ == "__main__":
    main()
