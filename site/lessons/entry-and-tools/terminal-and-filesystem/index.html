01

Что означает относительный путь data/orders.csv?

Относительный путь интерпретируется относительно текущего рабочего каталога, который можно увидеть командой pwd.

02

Что делает оператор | между двумя командами?

Pipe соединяет stdout команды слева со stdin команды справа, позволяя строить последовательность преобразований.

Терминал и файловая система

Терминал становится рабочим инструментом, когда путь, поток данных и ошибка остаются явными.

Тип: Build
Треки: Core
Пререквизиты: 02 — Диагностика Python и SQL
Время: ~60 минут
Результат: навигирует по проекту, управляет файлами и связывает команды через pipes

Цели обучения

  • Объяснять разницу между текущим каталогом, относительным и абсолютным путём.
  • Создавать, копировать, перемещать и удалять файлы в ограниченной рабочей области.
  • Соединять команды через стандартные потоки, pipes и перенаправления.
  • Безопасно обходить имена с пробелами и другими специальными символами.
  • Превращать одноразовую команду в воспроизводимый Bash-скрипт с проверкой ошибок.

Проблема

Аналитический проект быстро обрастает выгрузками, ноутбуками, отчётами и временными файлами. Пока файлов мало, можно открывать Finder или проводник и искать нужное глазами. Через месяц непонятно, какая выгрузка самая большая, где лежит результат расчёта и почему команда коллеги работает только из его домашнего каталога.

Терминал решает эту задачу не потому, что печатать команды «профессиональнее». Его преимущество — наблюдаемая последовательность операций. Команду можно повторить, проверить, положить в Git и запустить на другом каталоге. Но некавыченный путь или непроверенный rm превращают ту же скорость в риск потери данных.

В этом уроке вы построите аудит каталога: посчитаете файлы и байты, найдёте самые большие файлы и сгруппируете их по расширениям.

Концепция

Shell — одновременно интерпретатор команд и небольшой язык программирования. Он связывает утилиты через три стандартных потока:

text
stdin (0)  -> команда -> stdout (1)
                     \-> stderr (2)
  • stdin содержит входные данные.
  • stdout содержит обычный результат.
  • stderr содержит диагностику и ошибки.
  • | передаёт stdout команды слева в stdin команды справа.
  • > записывает stdout в файл с заменой содержимого.
  • >> дописывает stdout в конец файла.

Путь является аргументом команды, а не свойством файла.

Вид пути Пример Откуда начинается
Относительный data/orders.csv От текущего каталога процесса
Абсолютный /home/anna/project/data/orders.csv От корня файловой системы
Домашний ~/project После раскрытия ~ shell

Команда pwd показывает текущий каталог, cd меняет его для текущего shell, а ls показывает содержимое. Переменная PATH решает другую задачу: в ней shell ищет исполняемую команду, когда вы вводите python3 или git без пути к файлу.

Соберите это

Работайте в отдельной учебной папке. Не повторяйте команды удаления на каталоге с реальными данными.

Из корня репозитория перейдите в каталог урока:

bash
cd phases/00-entry-and-tools/03-terminal-and-filesystem

Шаг 1: создайте небольшое дерево проекта

bash
mkdir -p "terminal-lab/data/raw" "terminal-lab/reports"
printf 'order_id,amount\n101,120\n102,75\n' > "terminal-lab/data/raw/orders.csv"
printf '# План анализа\n' > "terminal-lab/reports/analysis plan.md"
printf 'temporary\n' > "terminal-lab/data/raw/to-delete.txt"

Проверьте своё положение и дерево:

bash
pwd
find "terminal-lab" -maxdepth 3 -print

Кавычки вокруг пути обязательны, когда имя содержит пробел. Полезнее выработать привычку кавычить подстановки всегда:

bash
project="terminal-lab"
cd "$project"

Без кавычек результат $project проходит word splitting и filename expansion. Сегодня значение простое, завтра переменная содержит client reports, и команда получает два аргумента вместо одного.

Шаг 2: выполните управляемые операции

Находясь внутри terminal-lab, создайте копию и переименуйте её:

bash
cp "data/raw/orders.csv" "data/raw/orders.backup.csv"
mv "data/raw/orders.backup.csv" "data/raw/orders.snapshot.csv"

Удалите только заранее созданный временный файл:

bash
rm -i "data/raw/to-delete.txt"

rm не переносит файл в корзину. Перед рекурсивным удалением сначала распечатайте путь и содержимое цели. Конструкцию rm -rf "$variable" нельзя запускать, пока не проверены пустое значение, текущий каталог и границы рабочей области.

Вернитесь в родительский каталог:

bash
cd ..

Шаг 3: соберите первый pipeline

Найдите обычные файлы и посчитайте байты каждого:

bash
find "terminal-lab" -type f -exec wc -c {} \;

Передайте строки в числовую сортировку:

bash
find "terminal-lab" -type f -exec wc -c {} \; | sort -nr

Сохраните первые строки отчёта:

bash
find "terminal-lab" -type f -exec wc -c {} \; |
  sort -nr |
  sed -n '1,5p' > "largest-files.txt"

Каждая команда выполняет одну операцию. Это и есть прозрачная модель будущего скрипта: источник записей, вычисление размера, сортировка и поставка результата.

Шаг 4: назовите ограничения одноразовой команды

Pipeline выше полезен для исследования, но ещё не является надёжным артефактом:

  • формат wc предназначен для человека и зависит от состава аргументов;
  • имя с переносом строки разрушит построчный отчёт;
  • ошибка ранней команды может быть скрыта успешной последней командой;
  • повторная запись отчёта внутрь каталога может изменить следующий аудит;
  • команда не проверяет аргументы и не объясняет способ использования.

Используйте это

Артефакт outputs/file_audit.sh превращает pipeline в Bash CLI.

Выведите отчёт в терминал:

bash
bash outputs/file_audit.sh --top 5 "terminal-lab"

Запишите его атомарно в файл:

bash
bash outputs/file_audit.sh \
  --top 10 \
  --output "terminal-lab/audit.md" \
  "terminal-lab"

Скрипт сначала строит отчёт во временном файле и только затем перемещает его в указанное место. Если audit.md уже находится внутри проверяемого каталога, он исключается из подсчёта. Поэтому повторный запуск на неизменившемся дереве даёт тот же результат.

Внутри используются четыре защитных решения:

  1. set -euo pipefail останавливает выполнение при ошибке, неизвестной переменной и неуспешной команде внутри pipeline.
  2. Все раскрытия путей заключены в двойные кавычки.
  3. find -print0 и read -d '' используют NUL как разделитель имён.
  4. LC_ALL=C фиксирует правила сортировки и преобразования регистра.

Запустите готовую демонстрацию:

bash
python3 code/main.py

Python здесь только создаёт временное дерево и запускает поставляемый Bash-артефакт.

Сломайте это

Проверьте характерные поломки в копии учебного каталога.

Пробел в имени

bash
touch "terminal-lab/data/raw/client export.csv"
for file in $(find "terminal-lab" -type f); do printf '%s\n' "$file"; done

Цикл разделит имя на части. Надёжный вариант не превращает вывод find в слова:

bash
while IFS= read -r -d '' file; do
  printf '%q\n' "$file"
done < <(find "terminal-lab" -type f -print0)

Перенос строки в имени

На Unix имя файла может содержать перевод строки:

bash
touch $'terminal-lab/data/raw/line\nbreak.csv'
bash outputs/file_audit.sh "terminal-lab"

Артефакт читает имя через NUL и печатает shell-escaped представление на одной строке.

Скрытая ошибка pipeline

Сравните статусы:

bash
bash -c 'false | true; echo "$?"'
bash -o pipefail -c 'false | true; echo "$?"'

Без pipefail успешная последняя команда маскирует ошибку слева.

Неверная рабочая директория

bash
cd "terminal-lab"
wc -c "terminal-lab/data/raw/orders.csv"

Команда ищет вложенный terminal-lab/terminal-lab. Исправление — понимать текущий каталог, а не добавлять ../ наугад:

bash
pwd
wc -c "data/raw/orders.csv"

Проверки подтверждают:

  • число файлов и сумма байтов совпадают с контрольным деревом;
  • --top ограничивает таблицу крупнейших файлов;
  • отчёт внутри проверяемого каталога не считает сам себя;
  • ошибочные аргументы завершаются кодом 2 и понятным сообщением.

Для ручной проверки выполните скрипт дважды с --output и сравните checksum:

bash
bash outputs/file_audit.sh --output "/tmp/audit-1.md" "terminal-lab"
bash outputs/file_audit.sh --output "/tmp/audit-2.md" "terminal-lab"
cmp -s "/tmp/audit-1.md" "/tmp/audit-2.md" && printf 'identical\n'

Одинаковый checksum подтверждает детерминированность только при неизменном дереве и одинаковом аргументе пути. Это не доказывает полноту аудита или корректность содержимого файлов.

Поставьте результат

Урок поставляет outputs/file_audit.sh — независимый Bash CLI без Python-зависимостей. Скопируйте его в аналитический проект и запускайте перед очисткой каталога, ревью большой выгрузки или настройкой .gitignore.

bash
bash outputs/file_audit.sh \
  --top 20 \
  --output "/tmp/project-audit.md" \
  "/path/to/project"

Manifest и готовая команда использования находятся в outputs/artifact.json. Скрипт считает обычные файлы, не следует по символическим ссылкам и исключает каталоги .git. Он измеряет размер и структуру дерева, но не определяет, можно ли безопасно удалить файл.

Упражнения

  1. Добавьте флаг --include-git и тест, подтверждающий оба режима.
  2. Добавьте таблицу каталогов первого уровня с количеством файлов, сохранив обработку пробелов в именах.
  3. Добавьте --max-bytes N, который завершает скрипт ненулевым кодом при превышении бюджета, и объясните, как использовать его в CI.

Проверьте себя

Ответьте после практики. Объяснение появится сразу после выбора.

01

Почему переменную пути обычно используют как "$path", а не как $path?

Двойные кавычки сохраняют результат подстановки как один аргумент и защищают пробелы и glob-символы в имени.

02

Почему надежный обход файлов использует find -print0 и read -d ''?

NUL не может входить в имя файла Unix, поэтому служит однозначным разделителем даже для необычных имен.

03

Зачем скрипту set -o pipefail?

Без pipefail статус pipeline обычно определяется последней командой, и ранняя ошибка может остаться незамеченной.

from __future__ import annotations

import os
import subprocess
from pathlib import Path
from tempfile import TemporaryDirectory
from unittest import TestCase


LESSON_ROOT = Path(__file__).resolve().parents[1]
AUDIT_SCRIPT = LESSON_ROOT / "outputs" / "file_audit.sh"


def run_audit(*arguments: str) -> subprocess.CompletedProcess[str]:
    return subprocess.run(
        ["bash", str(AUDIT_SCRIPT), *arguments],
        capture_output=True,
        text=True,
        check=False,
    )


class FileAuditTest(TestCase):
    def test_artifact_is_executable(self) -> None:
        self.assertTrue(os.access(AUDIT_SCRIPT, os.X_OK))

    def test_report_counts_files_and_excludes_git(self) -> None:
        with TemporaryDirectory() as directory:
            root = Path(directory) / "project with spaces"
            (root / "data").mkdir(parents=True)
            (root / "notes").mkdir()
            (root / ".git").mkdir()
            (root / "data" / "orders.csv").write_bytes(b"1234567890")
            (root / "notes" / "analysis plan.md").write_bytes(b"12345")
            (root / "README").write_bytes(b"123")
            (root / ".git" / "config").write_bytes(b"x" * 100)

            result = run_audit("--top", "2", str(root))

        self.assertEqual(result.returncode, 0, result.stderr)
        self.assertIn("- Files: 3", result.stdout)
        self.assertIn("- Bytes: 18", result.stdout)
        self.assertIn("data/orders.csv", result.stdout)
        self.assertIn(r"notes/analysis\ plan.md", result.stdout)
        self.assertNotIn(".git", result.stdout)
        self.assertIn("| `csv` | 1 |", result.stdout)
        self.assertIn("| `[none]` | 1 |", result.stdout)

    def test_output_inside_root_is_not_counted_on_repeat_runs(self) -> None:
        with TemporaryDirectory() as directory:
            root = Path(directory)
            (root / "sample.txt").write_text("sample", encoding="utf-8")
            output = root / "audit.md"

            first = run_audit("--output", str(output), str(root))
            first_report = output.read_text(encoding="utf-8")
            second = run_audit("--output", str(output), str(root))
            second_report = output.read_text(encoding="utf-8")

        self.assertEqual(first.returncode, 0, first.stderr)
        self.assertEqual(second.returncode, 0, second.stderr)
        self.assertEqual(first_report, second_report)
        self.assertIn("- Files: 1", second_report)

    def test_unusual_file_name_stays_on_one_report_line(self) -> None:
        with TemporaryDirectory() as directory:
            root = Path(directory)
            (root / "line\nbreak.txt").write_text("value", encoding="utf-8")

            result = run_audit(str(root))

        self.assertEqual(result.returncode, 0, result.stderr)
        self.assertIn(r"\n", result.stdout)
        self.assertNotIn("line\nbreak.txt", result.stdout)

    def test_invalid_arguments_return_usage_error(self) -> None:
        with TemporaryDirectory() as directory:
            invalid_top = run_audit("--top", "0", directory)
            directory_output = run_audit("--output", directory, directory)

        self.assertEqual(invalid_top.returncode, 2)
        self.assertIn("--top", invalid_top.stderr)
        self.assertEqual(directory_output.returncode, 2)
        self.assertIn("is a directory", directory_output.stderr)

from __future__ import annotations

import subprocess
from pathlib import Path
from tempfile import TemporaryDirectory


LESSON_ROOT = Path(__file__).resolve().parents[1]
AUDIT_SCRIPT = LESSON_ROOT / "outputs" / "file_audit.sh"


def build_demo_tree(root: Path) -> None:
    files = {
        "data/orders.csv": "order_id,amount\n101,120\n102,75\n",
        "data/customers.csv": "customer_id,segment\n1,new\n2,returning\n",
        "notes/analysis plan.md": "# Analysis plan\n\nCheck the grain first.\n",
        "README": "Demo analytical project\n",
        ".git/config": "[core]\nrepositoryformatversion = 0\n",
    }
    for relative, content in files.items():
        path = root / relative
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(content, encoding="utf-8")


def run_audit(root: Path, top: int = 5) -> str:
    result = subprocess.run(
        ["bash", str(AUDIT_SCRIPT), "--top", str(top), str(root)],
        check=True,
        capture_output=True,
        text=True,
    )
    return result.stdout


def main() -> None:
    with TemporaryDirectory() as directory:
        project = Path(directory) / "terminal lab"
        project.mkdir()
        build_demo_tree(project)
        print(run_audit(project))


if __name__ == "__main__":
    main()

outputs/file_audit.shПоказать
#!/usr/bin/env bash

set -euo pipefail
export LC_ALL=C

usage() {
    cat <<'EOF'
Usage:
  file_audit.sh [--top N] [--output FILE] DIRECTORY

Build a deterministic Markdown report for regular files under DIRECTORY.
The .git directory is excluded. If FILE is inside DIRECTORY, it is excluded too.

Options:
  --top N        Show N largest files (default: 10)
  --output FILE  Write the report atomically instead of printing it
  -h, --help     Show this help
EOF
}

fail() {
    printf 'file-audit: %s\n' "$1" >&2
    exit 2
}

top=10
output=
root_argument=

while (($# > 0)); do
    case "$1" in
        --top)
            (($# >= 2)) || fail "--top requires a positive integer"
            top=$2
            shift 2
            ;;
        --output)
            (($# >= 2)) || fail "--output requires a file path"
            output=$2
            shift 2
            ;;
        -h|--help)
            usage
            exit 0
            ;;
        --)
            shift
            (($# == 1)) || fail "expected exactly one directory after --"
            [[ -z "$root_argument" ]] || fail "expected exactly one directory"
            root_argument=$1
            shift
            ;;
        -*)
            fail "unknown option: $1"
            ;;
        *)
            [[ -z "$root_argument" ]] || fail "expected exactly one directory"
            root_argument=$1
            shift
            ;;
    esac
done

[[ "$top" =~ ^[1-9][0-9]*$ ]] || fail "--top must be a positive integer"
[[ -n "$root_argument" ]] || fail "directory is required"
[[ -d "$root_argument" ]] || fail "directory does not exist: $root_argument"

root=$(cd -- "$root_argument" && pwd -P)
output_absolute=

if [[ -n "$output" ]]; then
    [[ ! -d "$output" ]] || fail "output path is a directory: $output"
    output_directory=$(dirname -- "$output")
    output_name=$(basename -- "$output")
    [[ -d "$output_directory" ]] || fail "output directory does not exist: $output_directory"
    output_directory=$(cd -- "$output_directory" && pwd -P)
    output_absolute="$output_directory/$output_name"
fi

paths=$(mktemp "${TMPDIR:-/tmp}/file-audit-paths.XXXXXX")
records=$(mktemp "${TMPDIR:-/tmp}/file-audit-records.XXXXXX")
report=$(mktemp "${TMPDIR:-/tmp}/file-audit-report.XXXXXX")

cleanup() {
    rm -f -- "$paths" "$records" "$report"
}
trap cleanup EXIT

file_count=0
total_bytes=0

find "$root" -type d -name .git -prune -o -type f -print0 > "$paths"

while IFS= read -r -d '' file; do
    if [[ -n "$output_absolute" && "$file" == "$output_absolute" ]]; then
        continue
    fi

    bytes=$(wc -c < "$file")
    bytes=${bytes//[[:space:]]/}
    relative=${file#"$root"/}
    quoted_path=$(printf '%q' "$relative")
    name=${relative##*/}

    if [[ "$name" == *.* && "$name" != .* ]] || [[ "$name" == .*.* ]]; then
        extension=${name##*.}
    else
        extension="[none]"
    fi
    if [[ "$extension" != "[none]" ]]; then
        extension=$(printf '%s' "$extension" | tr '[:upper:]' '[:lower:]')
        if [[ ! "$extension" =~ ^[a-z0-9][a-z0-9_+-]*$ ]]; then
            extension="[other]"
        fi
    fi

    printf '%s\t%s\t%s\n' "$bytes" "$quoted_path" "$extension" >> "$records"
    file_count=$((file_count + 1))
    total_bytes=$((total_bytes + bytes))
done < "$paths"

root_label=$(printf '%q' "$root_argument")

{
    printf '# File audit\n\n'
    printf -- '- Root: `%s`\n' "$root_label"
    printf -- '- Files: %s\n' "$file_count"
    printf -- '- Bytes: %s\n' "$total_bytes"
    printf '\n## Largest files\n\n'
    printf '| Bytes | Path |\n'
    printf '|---:|---|\n'
    if ((file_count == 0)); then
        printf '| 0 | _No regular files_ |\n'
    else
        sort -t $'\t' -k1,1nr -k2,2 "$records" |
            awk -F '\t' -v limit="$top" \
                'NR <= limit { printf "| %s | `%s` |\n", $1, $2 }'
    fi
    printf '\n## Extensions\n\n'
    printf '| Extension | Files |\n'
    printf '|---|---:|\n'
    if ((file_count == 0)); then
        printf '| `[none]` | 0 |\n'
    else
        cut -f3 "$records" |
            sort |
            uniq -c |
            sort -k1,1nr -k2,2 |
            awk '{ printf "| `%s` | %s |\n", $2, $1 }'
    fi
} > "$report"

if [[ -n "$output_absolute" ]]; then
    mv -- "$report" "$output_absolute"
    printf 'file-audit: wrote %s\n' "$output_absolute" >&2
else
    cat "$report"
fi
