Проверка перед уроком

Ответьте до чтения: результат поможет заметить исходные предположения.

01

Массив имеет контракт (store, day, metric) = (4, 7, 3). Как следует прочитать число 7?

Shape читается по позициям: ось 0 — store длины 4, ось 1 — day длины 7, ось 2 — metric длины 3.

02

Из массива (store=4, day=7, metric=3) нужны недельные итоги отдельно для каждого магазина и показателя. Какой контракт результата следует записать до кода?

Недельный итог сворачивает ось day. Оси store и metric остаются в исходном порядке.

Shape, axes и размерность

shape — это упорядоченный контракт осей, а не просто кортеж чисел. До вычисления назовите изменяемую ось и запишите, какие оси и длины должны остаться в результате.

Тип: Build Треки: Core Пререквизит: 02-numpy/01-arrays Время: ~75 минут Результат: по предметным именам осей предсказывает форму и смысл результата после удаления, сохранения, перестановки, объединения или добавления осей и закрепляет ожидание проверкой.

Цели обучения

После урока вы сможете:

  • продолжить паспорт массива из первого урока и читать shape вместе с предметными именами осей;
  • объяснить, почему axis — это позиция в shape, а не универсальное обозначение строки или столбца;
  • до запуска NumPy предсказать форму результата редукции, transpose, reshape и expand_dims;
  • нормализовать отрицательный номер оси и обнаружить несуществующую или повторную ось;
  • отличить перестановку осей от перегруппировки элементов и не использовать reshape вместо transpose;
  • заметить ситуацию, когда правильный shape не гарантирует правильный предметный смысл;
  • зафиксировать ожидаемую форму именованной проверкой рядом с аналитическим расчетом.

Проблема

В первом уроке мы создали массив заказов:

python
order_counts = [
    [12, 15, 9],
    [10, 11, 14],
]

Его паспорт был записан не просто как (2, 3), а вместе со смыслом:

text
(store, day) = (2, 3)

Такой паспорт отвечает на вопрос «что лежит в массиве». Теперь нужно научиться отвечать на следующий вопрос: что станет с осями после операции.

Допустим, по каждому магазину и дню у нас есть уже не одно, а два числовых показателя: количество заказов и выручка. Получается трехмерный массив:

text
(store, day, metric) = (2, 3, 2)

Задача звучит так: получить итог за все дни отдельно для каждого магазина и каждого показателя.

До кода ожидаемый контракт результата можно записать словами:

text
было:      (store, day, metric)
свернули:          day
осталось:  (store,      metric)
shape:     (2, 2)

Теперь сравним две синтаксически правильные операции:

python
daily_metrics.sum(axis=0).shape  # (3, 2)
daily_metrics.sum(axis=1).shape  # (2, 2)

Первая складывает магазины и сохраняет дни. Вторая складывает дни и сохраняет магазины. Обе возвращают числа без исключения, но только вторая отвечает поставленной задаче.

Это опасный класс аналитических ошибок:

  • код запускается;
  • результат имеет правдоподобный размер;
  • числа могут выглядеть разумно;
  • смысл расчета уже неверен.

Поэтому номер axis нельзя выбирать по памяти. Перед операцией нужно назвать ось, которую мы меняем, предсказать оставшиеся оси и только затем писать вызов NumPy.

Концепция

Что мы переносим из первого урока

Для конкретного массива уже известны четыре свойства:

Свойство Вопрос Пример
ndim Сколько осей? 3
shape Какова длина каждой оси по порядку? (2, 3, 2)
size Сколько всего элементов? 12
dtype Как интерпретируется каждый элемент? например, int64

В этом уроке мы концентрируемся на первых трех. У исходного массива:

text
ndim = len(shape) = 3
size = 2 * 3 * 2 = 12

Но голых чисел недостаточно. Рядом с каждой длиной должно стоять предметное имя:

Номер оси Имя оси Длина Что меняется вдоль оси
0 store 2 магазин
1 day 3 день
2 metric 2 показатель

Эта таблица — исходный контракт всех дальнейших преобразований.

Ось — это позиция в упорядоченной форме

В двумерном массиве ось 0 часто называют строками, а ось 1 — столбцами. Такое объяснение удобно только для первого знакомства и быстро перестает работать.

В трехмерном массиве нет единственной естественной пары «строка — столбец». Есть три направления, и каждое определяется позицией в shape:

text
shape[0] -> длина оси 0 -> store
shape[1] -> длина оси 1 -> day
shape[2] -> длина оси 2 -> metric

Поэтому axis=1 означает не «по строкам» и не «по столбцам». Он означает: операция применяется вдоль второй позиции текущего контракта, то есть вдоль day.

После перестановки осей тот же номер может означать уже другую сущность. Номер принадлежит текущему порядку осей, а не предметной области навсегда.

Отрицательные номера считаются с конца

NumPy позволяет ссылаться на оси с конца:

text
(store, day, metric)
    0     1      2
   -3    -2     -1

Для массива с ndim == 3:

Запись Нормализованный номер Имя
axis=0 0 store
axis=-3 0 store
axis=1 1 day
axis=-2 1 day
axis=2 2 metric
axis=-1 2 metric

Для отрицательного номера действует правило:

text
normalized_axis = axis + ndim

Но оно применяется только к допустимому диапазону от -ndim до ndim - 1. Для трех осей номера 3 и -4 уже недопустимы.

Отрицательная запись полезна, когда контракт гарантирует смысл последней оси. Например, axis=-1 часто читается как «последняя ось», но не как «ось метрик вообще»: после перестановки последней может оказаться другая ось.

Пять действий над контрактом осей

Большинство операций этого урока можно понять без значений массива. Достаточно преобразовать список имен осей и их длин.

Действие Что происходит с осями Что происходит с size
Поэлементная операция все оси сохраняются сохраняется
Редукция выбранная ось удаляется или получает длину 1 обычно уменьшается
transpose оси переставляются сохраняется
reshape старые оси перегруппировываются в новые сохраняется
expand_dims вставляется новая ось длины 1 сохраняется

Эта таблица не описывает все NumPy. Индексация и маски будут отдельным уроком, broadcasting — отдельным уроком, а выбор правильной статистической агрегации — уроком об агрегатах. Здесь задача уже: научиться отслеживать форму.

Редукция удаляет ось

Редукция заменяет множество значений вдоль выбранной оси одним результатом: суммой, средним, минимумом или другой сводкой.

Для суммы по дням:

text
source axes:  (store, day, metric)
reduce axis:          day
result axes:  (store,      metric)

source shape: (2, 3, 2)
result shape: (2,    2)

По умолчанию выбранная ось исчезает, поэтому ndim уменьшается с 3 до 2.

Если указать keepdims=True, ось остается на прежнем месте, но ее длина становится равной 1:

text
source:               (store=2, day=3, metric=2)
reduce day, keepdims:  (store=2, day=1, metric=2)

Длина 1 не означает, что в исходных данных был один день. Она означает, что три дня были сведены к одному агрегированному положению. Такое сохранение позиции пригодится для совместимых операций с исходным массивом; подробно это появится в уроке о broadcasting.

Если axis=None, NumPy сворачивает все оси. Результат имеет форму () — это нульмерный массив или скалярное значение, а не одномерный массив формы (1,).

transpose переставляет оси

transpose не объединяет и не удаляет оси. Он задает, в каком порядке старые оси должны стать новыми:

text
source axes:  (store, day, metric)
permutation:     2     0      1
result axes:  (metric, store, day)

source shape: (2, 3, 2)
result shape: (2, 2, 3)

Кортеж (2, 0, 1) читается так:

  1. новой осью 0 станет старая ось 2 (metric);
  2. новой осью 1 станет старая ось 0 (store);
  3. новой осью 2 станет старая ось 1 (day).

Каждый старый номер должен встретиться ровно один раз. Повтор (0, 0, 1) не является перестановкой: одна ось продублирована, другая потеряна.

reshape перегруппировывает элементы

reshape меняет количество и длины осей, но сохраняет число элементов:

text
prod(source_shape) == prod(target_shape)

Например, массив (store=2, day=3, metric=2) содержит 12 элементов. Его можно представить как матрицу (store_day=6, metric=2):

text
(2 * 3) * 2 = 6 * 2 = 12

Это полезно, когда следующая функция ожидает двумерную матрицу вида (observation, feature). Но reshape не знает, что первые две оси означают магазин и день. Новое имя store_day и соответствие каждой строки исходным координатам обязан зафиксировать аналитик.

Одна длина -1 просит NumPy вычислить ее:

text
(2, 3, 2) -> (-1, 2)
12 / 2 = 6
result: (6, 2)

Использовать два -1 нельзя: из общего размера невозможно однозначно вывести обе длины.

Важно: reshape не является перестановкой. Совпадение числа элементов еще не означает, что значения оказались на предметно правильных местах.

expand_dims добавляет ось длины один

Иногда следующий расчет ожидает дополнительную ось: например, несколько сценариев для одного и того же набора метрик. Пока сценарий один, можно добавить ось длины 1:

text
source:  (store=2, day=3, metric=2)
result:  (store=2, day=3, metric=2, scenario=1)

Число элементов не меняется, потому что умножение на 1 не меняет произведение длин. Но позиция новой оси важна. Формы (1, 2, 3, 2) и (2, 3, 2, 1) содержат одинаковое число элементов, однако первая означает (scenario, store, day, metric), а вторая — (store, day, metric, scenario).

Правильная форма еще не доказывает правильный смысл

Проверка shape обнаруживает много ошибок, но не все. В нашем примере оси store и metric обе имеют длину 2:

text
(store, day, metric) = (2, 3, 2)

Если случайно переставить их:

text
(metric, day, store) = (2, 3, 2)

числовой shape вообще не изменится. Простая проверка array.shape == (2, 3, 2) пропустит ошибку.

Отсюда граница проверки формы:

  • shape проверяет количество и длины осей;
  • имена осей фиксируют предполагаемый смысл;
  • контракт источника, идентификаторы и контрольные значения подтверждают, что смысл действительно не перепутан.

Четыре шага предсказания формы

Перед любой операцией используйте один и тот же короткий алгоритм.

  1. Запишите исходный контракт: (store=2, day=3, metric=2).
  2. Назовите действие словами: «свернуть day», «поставить metric первой», «объединить store и day в store_day».
  3. Преобразуйте имена осей: удалите, сохраните, переставьте, объедините или добавьте нужное имя.
  4. Подставьте длины и поставьте assertion: только после этого запускайте NumPy.

Такой порядок важнее запоминания отдельных методов. Он переносится на массивы любой размерности и на библиотеки, которые используют ту же модель форм.

Соберите это

Сначала построим уменьшенную модель формы без NumPy. Она не вычисляет суммы и не хранит большой массив: она только преобразует контракт осей.

Шаг 1. Составьте реестр осей

python
source_shape = (2, 3, 2)
source_axis_names = ("store", "day", "metric")

assert len(source_shape) == len(source_axis_names)
assert len(set(source_axis_names)) == len(source_axis_names)

axis_registry = [
    {"axis": index, "name": name, "length": length}
    for index, (name, length) in enumerate(
        zip(source_axis_names, source_shape)
    )
]

Результат:

python
[
    {"axis": 0, "name": "store", "length": 2},
    {"axis": 1, "name": "day", "length": 3},
    {"axis": 2, "name": "metric", "length": 2},
]

Уникальность имен не является требованием самого NumPy. Это требование нашего учебного контракта: два одинаковых имени сделали бы дальнейшее рассуждение неоднозначным.

Шаг 2. Нормализуйте номер оси

python
def normalize_axis(axis, ndim):
    normalized = axis + ndim if axis < 0 else axis
    if normalized < 0 or normalized >= ndim:
        raise ValueError(
            f"axis {axis} is out of bounds for ndim={ndim}"
        )
    return normalized

Проверим крайние случаи:

python
assert normalize_axis(0, 3) == 0
assert normalize_axis(-1, 3) == 2
assert normalize_axis(-3, 3) == 0

Оси 3 и -4 должны завершаться ошибкой. Молчаливо исправлять их нельзя: это почти всегда означает, что вызывающий код неверно понял размерность входа.

Шаг 3. Предскажите редукцию

Сначала удалим выбранную ось из кортежа длин:

python
def reduction_shape(shape, axis, keepdims=False):
    normalized = normalize_axis(axis, len(shape))
    if keepdims:
        return tuple(
            1 if index == normalized else length
            for index, length in enumerate(shape)
        )
    return tuple(
        length
        for index, length in enumerate(shape)
        if index != normalized
    )

Та же операция над именами:

python
def reduction_axis_names(axis_names, axis, keepdims=False):
    normalized = normalize_axis(axis, len(axis_names))
    if keepdims:
        return tuple(axis_names)
    return tuple(
        name
        for index, name in enumerate(axis_names)
        if index != normalized
    )

Теперь числовое и предметное ожидания согласованы:

python
assert reduction_shape(source_shape, 1) == (2, 2)
assert reduction_axis_names(source_axis_names, 1) == (
    "store",
    "metric",
)

assert reduction_shape(source_shape, 1, keepdims=True) == (2, 1, 2)
assert reduction_axis_names(
    source_axis_names,
    1,
    keepdims=True,
) == ("store", "day", "metric")

Шаг 4. Предскажите перестановку

Для перестановки достаточно собрать длины и имена в новом порядке:

python
permutation = (2, 0, 1)

assert sorted(permutation) == list(range(len(source_shape)))

transposed_shape = tuple(source_shape[index] for index in permutation)
transposed_names = tuple(source_axis_names[index] for index in permutation)

assert transposed_shape == (2, 2, 3)
assert transposed_names == ("metric", "store", "day")

Здесь 2 в начале не является новой длиной. Это номер старой оси, которую нужно поставить первой.

Шаг 5. Проверьте допустимость reshape

python
from math import prod


def can_reshape(source, target):
    return prod(source) == prod(target)
python
assert can_reshape((2, 3, 2), (6, 2))
assert not can_reshape((2, 3, 2), (5, 2))

После числовой проверки отдельно задайте новый предметный контракт:

text
source: (store=2, day=3, metric=2)
target: (store_day=6, metric=2)

Функция prod способна проверить количество элементов, но не может придумать имя store_day и доказать, что выбранный порядок значений соответствует этому имени.

Шаг 6. Вставьте новую ось

Для вставки в конец:

python
expanded_shape = (*source_shape, 1)
expanded_names = (*source_axis_names, "scenario")

assert expanded_shape == (2, 3, 2, 1)
assert expanded_names == ("store", "day", "metric", "scenario")

В общем случае кортеж нужно разрезать в нормализованной позиции и вставить туда длину 1 и новое имя. Именно это делает артефакт урока.

Используйте это

Теперь применим правила к реальному ndarray. Данные продолжают пример первого урока:

python
import numpy as np

daily_metrics = np.array(
    [
        [[12, 1200], [15, 1500], [9, 900]],
        [[10, 1000], [11, 1100], [14, 1400]],
    ]
)

assert daily_metrics.shape == (2, 3, 2)

В каждой паре первое число — количество заказов, второе — выручка. Для реального проекта названия и единицы измерения должны храниться в отдельном контракте; обычный ndarray их не сохраняет.

Сверните ось дней

python
store_totals = daily_metrics.sum(axis=1)

assert store_totals.shape == (2, 2)

Предметный контракт результата:

text
(store, metric) = (2, 2)

Мы пока не изучаем тонкости агрегирования разных метрик. Здесь важно, что NumPy применил сумму отдельно для каждой оставшейся комбинации store и metric.

Если позиция оси нужна дальше:

python
store_totals_kept = daily_metrics.sum(axis=1, keepdims=True)

assert store_totals_kept.shape == (2, 1, 2)

Контракт теперь (store, summarized_day, metric). Имя можно оставить day, если в документации явно сказано, что длина 1 представляет агрегат всех дней, а не один исходный день.

Поставьте метрики первой осью

python
metric_first = np.transpose(daily_metrics, (2, 0, 1))

assert metric_first.shape == (2, 2, 3)
text
(metric, store, day) = (2, 2, 3)

np.transpose изменил порядок доступа к тем же значениям. Он не сложил магазины и не объединил дни.

Для двумерного массива сокращение .T часто удобно, но в учебном трехмерном примере явная перестановка лучше: она показывает, откуда взялась каждая новая позиция.

Соберите матрицу наблюдений и признаков

python
store_day_rows = daily_metrics.reshape(-1, 2)

assert store_day_rows.shape == (6, 2)
text
(store_day, metric) = (6, 2)

Такая форма удобна для функции, ожидающей матрицу (observation, feature). Но после объединения двух осей одной формы мало: если нужно вернуться к конкретному магазину и дню, сохраните их идентификаторы или явное правило соответствия строк исходным координатам.

Добавьте ось сценария

python
with_scenario = np.expand_dims(daily_metrics, axis=-1)

assert with_scenario.shape == (2, 3, 2, 1)

Эквивалентная короткая запись:

python
with_scenario = daily_metrics[..., np.newaxis]

Для первого знакомства np.expand_dims яснее: позиция новой оси названа отдельным аргументом. Запись с np.newaxis станет естественнее после урока об индексации.

CLI не создает массив из двенадцати или двенадцати миллионов элементов. Он работает с контрактом и показывает:

  • исходные номера, имена и длины осей;
  • нормализованную ось редукции;
  • имена и форму результата каждой операции;
  • предупреждение, если после reshape или добавления оси новый смысл не назван.

Все переданные операции рассчитываются независимо от исходной формы, а не цепочкой друг за другом. Это позволяет сравнить несколько возможных преобразований одного входа.

Сломайте это

Ошибка 1. Выбрать существующую, но неверную ось

python
wrong = daily_metrics.sum(axis=0)

assert wrong.shape == (3, 2)

Операция не сломалась. Она получила итог по магазинам для каждого дня и каждой метрики:

text
(day, metric) = (3, 2)

Если задача требовала итоги по дням для каждого магазина, расчет неверен. Лучший барьер — до вызова записать ожидаемые имена результата (store, metric), а после вызова проверить shape == (2, 2).

Ошибка 2. Переставить две оси одинаковой длины

python
swapped = np.transpose(daily_metrics, (2, 1, 0))

assert swapped.shape == daily_metrics.shape == (2, 3, 2)

Проверка формы прошла, хотя контракт изменился с (store, day, metric) на (metric, day, store). Поэтому критичные интерфейсы дополняют shape-проверку именами, идентификаторами, контрольным наблюдением или более богатой схемой данных.

Ошибка 3. Использовать reshape вместо transpose

python
matrix = np.array(
    [
        [1, 2, 3],
        [4, 5, 6],
    ]
)

print(matrix.reshape(3, 2))
print(matrix.T)

Результаты различаются:

text
reshape:       transpose:
[[1, 2],       [[1, 4],
 [3, 4],        [2, 5],
 [5, 6]]        [3, 6]]

Обе формы равны (3, 2), но только transpose меняет местами исходные оси. reshape читает элементы в выбранном порядке и группирует их по новым длинам.

Ошибка 4. Изменить количество элементов через reshape

python
daily_metrics.reshape(5, 2)

Исходный массив содержит 12 элементов, целевая форма требует 10. NumPy остановит операцию. Артефакт обнаруживает это по произведению длин без создания массива.

Ошибка 5. Без разбора удалить все оси длины один

python
values = np.zeros((1, 20, 1))
collapsed = values.squeeze()

assert collapsed.shape == (20,)

Без аргумента squeeze() удалил обе оси длины 1. Одна могла означать единственный сценарий, другая — единственную метрику. Если удалить нужно только конкретную ось, указывайте ее явно и сначала записывайте ожидаемый результат.

Ошибка 6. Случайно свернуть весь массив

python
total = daily_metrics.sum()

assert np.shape(total) == ()

Отсутствующий axis означает редукцию по всем осям. Это подходит для одного общего итога, но уничтожает разрезы по магазину, дню и метрике.

Ошибка 7. Добавить ось не в ту позицию

python
first = np.expand_dims(daily_metrics, axis=0)
last = np.expand_dims(daily_metrics, axis=-1)

assert first.shape == (1, 2, 3, 2)
assert last.shape == (2, 3, 2, 1)

Обе операции добавили длину 1, но создали разные контракты. Эта разница станет особенно важной, когда массивы начнут автоматически согласовывать формы при broadcasting.

Поставьте результат

Именованный артефакт урока:

text
phases/02-numpy/02-shape-and-axes/outputs/shape_contract.py

Он предоставляет функции:

  • normalize_axes() — проверяет и нормализует одну или несколько осей;
  • reduction_shape() — предсказывает форму редукции с учетом keepdims;
  • reshape_shape() — выводит одну длину -1 и запрещает изменение числа элементов;
  • transpose_shape() — проверяет полную перестановку осей;
  • expand_dims_shape() — вставляет ось длины 1 в допустимую позицию;
  • axis_descriptors() — связывает номера, имена и длины осей;
  • assert_shape() — останавливает расчет с именованным описанием ожидания;
  • build_report() — собирает машинно-читаемый отчет о возможных преобразованиях.

Пример проверки внутри аналитической функции:

python
store_totals = daily_metrics.sum(axis=1)

assert_shape(
    store_totals,
    (2, 2),
    name="store_totals",
    axis_names=("store", "metric"),
)

Сообщение об ошибке содержит роль массива, фактическую форму и ожидаемые длины с предметными именами. Такой assertion особенно полезен на границе функций: он превращает неявное предположение автора в исполняемый контракт.

Артефакт не доказывает, что имя store соответствует именно магазинам. Источник истины для смысла — спецификация данных и контрольные наблюдения. Инструмент проверяет, что числовая форма согласована с заявленным контрактом и что смысл не потерян незаметно во время преобразования.

Ключевые термины

Термин Распространенное заблуждение Точное значение
Ось (axis) «Ось 0 всегда строки» Позиция направления в текущем упорядоченном контракте массива
Имя оси «NumPy хранит его внутри обычного массива» Предметная метка, которую аналитик ведет рядом с ndarray
ndim «Длина самой большой оси» Количество осей, равное длине shape
shape «Просто размер массива» Упорядоченный кортеж длин всех осей
Отрицательная ось «Особый вид оси» Номер, отсчитанный от конца текущего shape
Редукция «Любое уменьшение формы» Сведение значений вдоль выбранных осей к агрегированным результатам
keepdims «Не выполнять редукцию» Сохранить позицию редуцированной оси с длиной 1
reshape «Переставить оси» Перегруппировать то же число элементов по новым длинам
transpose «Переписать значения в обратном порядке» Переставить существующие оси по указанной перестановке
Единичная ось «Ось без данных» Ось длины 1, которая все равно входит в контракт формы
Shape-контракт «Проверка количества элементов» Ожидаемые имена, порядок и длины осей на границе вычисления

Дополнительное чтение

  • Яндекс Образование: модули math и NumPy (RU) — начните с разделов «Изменение размерности массива», «Транспонирование и поворот массива» и «Агрегатные функции». На небольших двумерных примерах закрепите различие между изменением формы, перестановкой осей и редукцией.
  • NumPy quickstart: Shape manipulation (EN) — обязательное официальное продолжение урока: сравните reshape, resize, ravel и транспонирование, затем отдельно разберите пример с автоматически вычисляемой длиной -1.
  • Дмитрий Макаров: оси массива NumPy (RU) — наглядный разбор осей с диаграммами, суммированием по axis=0 и axis=1, несколькими и отрицательными осями. Найдите раздел «Оси массива» и используйте его, чтобы еще раз проговорить, какая ось исчезает, а какие остаются.
  • NumPy: sum (EN) — точный контракт реальной редукции: axis=None, один номер, кортеж и отрицательные оси, а также влияние keepdims на форму результата. Остальные параметры пока можно пропустить.
  • С. В. Лемешевский: транспонирование массивов и замена осей (RU) — откройте одноименный раздел в оглавлении: он объясняет переход от обычного .T к явной перестановке осей многомерного массива. Сопоставьте кортеж перестановки с именами осей из урока.
  • NumPy: reshape (EN) — справочник для проверки совместимости новой формы, одной выводимой длины -1, порядка чтения элементов и возможности получить view или копию. Особенно важен после упражнения, где reshape и transpose дают одинаковый shape, но разные значения.
  • NumPy: transpose (EN) — точные правила перестановки для N-мерного массива. Обратите внимание на смысл axes[i], допустимость отрицательных номеров и на то, что транспонирование одномерного массива не превращает его в вектор-столбец.
  • NumPy: expand_dims (EN) — правила вставки единичной оси в положительную или отрицательную позицию и примеры эквивалентности с np.newaxis. Сверяйте не только итоговый кортеж, но и предметное имя новой оси.
  • NumPy: squeeze (EN) — failure mode, обратный добавлению оси: страница показывает разницу между удалением всех единичных осей и одной явно указанной, а также появление формы () после удаления последних осей.
  • xarray: Terminology (EN) — необязательное углубление о границе ndarray: NumPy хранит позиции и длины, а xarray добавляет имена измерений и координаты. Читать достаточно разделы DataArray, Dimension и Coordinate; устанавливать новую библиотеку для этого урока не нужно.

Ответьте после практики. Объяснение появится сразу после выбора.

01

У массива x контракт (store, day, metric) = (4, 7, 3). Что означает x.sum(axis=-1)?

Для ndim=3 ось -1 нормализуется в 2 — последнюю ось metric. После редукции остаются store и day.

02

Какова форма x.sum(axis=1, keepdims=True), если x.shape == (4, 7, 3), и что означает длина 1?

keepdims не отменяет редукцию. Он оставляет выбранную ось на прежней позиции с длиной 1.

03

Нужно изменить контракт (store, day, metric) на (metric, store, day). Какая перестановка осей выражает это намерение?

Новой первой осью становится старая 2 (metric), затем старая 0 (store), затем старая 1 (day).

04

Матрица формы (2, 3) должна поменять местами исходные оси. Почему недостаточно вызвать reshape(3, 2)?

Обе операции могут дать форму (3, 2), но только transpose переносит старую ось 1 на место 0 и старую ось 0 на место 1.

05

После перестановки контракт изменился с (store, day, metric) = (2, 7, 2) на (metric, day, store) = (2, 7, 2). Почему shape assertion не обнаружил ошибку?

Числовой кортеж остался тем же, хотя смысл позиций изменился. Форма — необходимая, но не достаточная часть контракта.

06

Исходный контракт равен (store, day, metric) = (2, 7, 3). Как добавить последней ось scenario для одного сценария?

axis=-1 при expand_dims вставляет новую ось в последнюю позицию; ее длина равна 1, а исходные оси сохраняются.

from __future__ import annotations

import importlib.util
import json
import subprocess
import sys
import unittest
from pathlib import Path

import numpy as np

ROOT = Path(__file__).resolve().parents[1]
ARTIFACT = ROOT / "outputs" / "shape_contract.py"
MAIN = ROOT / "code" / "main.py"
SPEC = importlib.util.spec_from_file_location("shape_contract", ARTIFACT)
if SPEC is None or SPEC.loader is None:
    raise RuntimeError(f"cannot load {ARTIFACT}")
SHAPES = importlib.util.module_from_spec(SPEC)
SPEC.loader.exec_module(SHAPES)


class ShapeContractTest(unittest.TestCase):
    def test_negative_axes_are_normalized(self) -> None:
        self.assertEqual(SHAPES.normalize_axes(-1, 3), (2,))
        self.assertEqual(SHAPES.normalize_axes((0, -1), 3), (0, 2))

    def test_invalid_axes_are_rejected(self) -> None:
        for axis in (True, 3, -4, (0, -3)):
            with self.subTest(axis=axis), self.assertRaises(SHAPES.ShapeContractError):
                SHAPES.normalize_axes(axis, 3)

    def test_axis_names_must_match_and_be_unique(self) -> None:
        self.assertEqual(
            SHAPES.validate_axis_names(("store", "day", "metric"), 3),
            ("store", "day", "metric"),
        )
        for names in (("store", "day"), ("store", "day", "day"), ("store", "", "metric")):
            with self.subTest(names=names), self.assertRaises(SHAPES.ShapeContractError):
                SHAPES.validate_axis_names(names, 3)

    def test_reduction_shape_matches_numpy(self) -> None:
        array = np.empty((2, 3, 4))
        for axis in (None, 0, 1, -1, (0, 2)):
            for keepdims in (False, True):
                expected = np.sum(array, axis=axis, keepdims=keepdims).shape
                self.assertEqual(
                    SHAPES.reduction_shape(array.shape, axis, keepdims=keepdims),
                    expected,
                )

    def test_reduction_report_preserves_axis_meaning(self) -> None:
        report = SHAPES.build_report(
            (2, 3, 2),
            axis_names=("store", "day", "metric"),
            axis=1,
        )
        reduction = report["operations"]["reduction"]
        self.assertEqual(reduction["operated_axis_names"], ["day"])
        self.assertEqual(reduction["result_shape"], [2, 2])
        self.assertEqual(reduction["result_axis_names"], ["store", "metric"])

        kept = SHAPES.build_report(
            (2, 3, 2),
            axis_names=("store", "day", "metric"),
            axis=1,
            keepdims=True,
        )["operations"]["reduction"]
        self.assertEqual(kept["result_shape"], [2, 1, 2])
        self.assertEqual(kept["result_axis_names"], ["store", "day", "metric"])

    def test_reshape_infers_one_dimension_and_rejects_changed_size(self) -> None:
        self.assertEqual(SHAPES.reshape_shape((2, 3, 4), (6, -1)), (6, 4))
        with self.assertRaisesRegex(SHAPES.ShapeContractError, "element count"):
            SHAPES.reshape_shape((2, 3), (4, 2))

    def test_reshape_requires_a_new_semantic_contract(self) -> None:
        unnamed = SHAPES.build_report(
            (2, 3, 2),
            axis_names=("store", "day", "metric"),
            reshape=(6, 2),
        )["operations"]["reshape"]
        self.assertIsNone(unnamed["result_axis_names"])
        self.assertIn("axis meaning", unnamed["semantic_warning"])

        named = SHAPES.build_report(
            (2, 3, 2),
            axis_names=("store", "day", "metric"),
            reshape=(6, 2),
            reshape_axis_names=("store_day", "metric"),
        )["operations"]["reshape"]
        self.assertEqual(named["result_axis_names"], ["store_day", "metric"])

    def test_transpose_and_expand_shapes_match_numpy(self) -> None:
        array = np.empty((2, 3, 4))
        self.assertEqual(
            SHAPES.transpose_shape(array.shape, (2, 0, 1)),
            np.transpose(array, (2, 0, 1)).shape,
        )
        self.assertEqual(
            SHAPES.expand_dims_shape(array.shape, -1),
            np.expand_dims(array, -1).shape,
        )

    def test_transpose_and_expand_reports_move_axis_names(self) -> None:
        report = SHAPES.build_report(
            (2, 3, 2),
            axis_names=("store", "day", "metric"),
            transpose=(2, 0, 1),
            expand_axis=-1,
            expand_axis_name="scenario",
        )
        self.assertEqual(
            report["operations"]["transpose"]["result_axis_names"],
            ["metric", "store", "day"],
        )
        self.assertEqual(
            report["operations"]["expand_dims"]["result_axis_names"],
            ["store", "day", "metric", "scenario"],
        )

    def test_assert_shape_reports_role_and_named_expectation(self) -> None:
        with self.assertRaisesRegex(
            SHAPES.ShapeContractError,
            r"store_totals.*store=2, metric=2",
        ):
            SHAPES.assert_shape(
                np.zeros((3, 2)),
                (2, 2),
                name="store_totals",
                axis_names=("store", "metric"),
            )

    def test_cli_returns_structured_semantic_report(self) -> None:
        result = subprocess.run(
            [
                sys.executable,
                ARTIFACT,
                "--shape",
                "[2, 3, 2]",
                "--axis-names",
                '["store", "day", "metric"]',
                "--axis",
                "1",
                "--reshape",
                "[6, 2]",
                "--reshape-axis-names",
                '["store_day", "metric"]',
                "--transpose",
                "[2, 0, 1]",
                "--expand-axis",
                "-1",
                "--expand-axis-name",
                "scenario",
            ],
            check=False,
            capture_output=True,
            text=True,
        )
        self.assertEqual(result.returncode, 0, result.stderr)
        report = json.loads(result.stdout)
        self.assertEqual(report["operations"]["reduction"]["result_shape"], [2, 2])
        self.assertEqual(
            report["operations"]["reshape"]["result_axis_names"],
            ["store_day", "metric"],
        )
        self.assertEqual(
            report["operations"]["transpose"]["result_shape"],
            [2, 2, 3],
        )

    def test_cli_rejects_invalid_axis_without_traceback(self) -> None:
        result = subprocess.run(
            [sys.executable, ARTIFACT, "--shape", "[2, 3]", "--axis", "2"],
            check=False,
            capture_output=True,
            text=True,

from __future__ import annotations

import argparse
import json
import math
from collections.abc import Iterable, Sequence
from numbers import Integral
from pathlib import Path
from typing import Any

import numpy as np


class ShapeContractError(ValueError):
    """Raised when an operation cannot satisfy its shape contract."""


def validate_shape(shape: Iterable[int], *, name: str = "shape") -> tuple[int, ...]:
    raw_shape = tuple(shape)
    if any(isinstance(length, bool) or not isinstance(length, Integral) for length in raw_shape):
        raise ShapeContractError(f"{name} must contain integers")
    result = tuple(int(length) for length in raw_shape)
    if any(length < 0 for length in result):
        raise ShapeContractError(f"{name} cannot contain negative lengths")
    return result


def validate_axis_names(
    axis_names: Sequence[str],
    ndim: int,
    *,
    name: str = "axis names",
) -> tuple[str, ...]:
    if isinstance(axis_names, (str, bytes)):
        raise ShapeContractError(f"{name} must be a sequence of names")
    result = tuple(axis_names)
    if len(result) != ndim:
        raise ShapeContractError(f"{name} has {len(result)} names, expected {ndim}")
    if any(not isinstance(axis_name, str) or not axis_name.strip() for axis_name in result):
        raise ShapeContractError(f"{name} must contain non-empty strings")
    if len(set(result)) != len(result):
        raise ShapeContractError(f"{name} must be unique")
    return result


def normalize_axes(axis: int | Iterable[int] | None, ndim: int) -> tuple[int, ...]:
    if ndim < 0:
        raise ShapeContractError("ndim cannot be negative")
    if axis is None:
        return tuple(range(ndim))

    raw_axes = (axis,) if isinstance(axis, (Integral, bool)) else tuple(axis)
    normalized: list[int] = []
    for raw_axis in raw_axes:
        if isinstance(raw_axis, bool) or not isinstance(raw_axis, Integral):
            raise ShapeContractError("axis must contain integers")
        axis_number = int(raw_axis)
        current = axis_number + ndim if axis_number < 0 else axis_number
        if current < 0 or current >= ndim:
            raise ShapeContractError(f"axis {axis_number} is out of bounds for ndim={ndim}")
        if current in normalized:
            raise ShapeContractError(f"axis {axis_number} is repeated")
        normalized.append(current)
    return tuple(sorted(normalized))


def reduction_shape(
    shape: Sequence[int],
    axis: int | Iterable[int] | None,
    *,
    keepdims: bool = False,
) -> tuple[int, ...]:
    source = validate_shape(shape)
    axes = set(normalize_axes(axis, len(source)))
    if keepdims:
        return tuple(1 if index in axes else length for index, length in enumerate(source))
    return tuple(length for index, length in enumerate(source) if index not in axes)


def reshape_shape(shape: Sequence[int], target: Sequence[int]) -> tuple[int, ...]:
    source = validate_shape(shape)
    raw_target = tuple(target)
    if any(isinstance(length, bool) or not isinstance(length, Integral) for length in raw_target):
        raise ShapeContractError("target shape must contain integers")
    requested = tuple(int(length) for length in raw_target)
    if requested.count(-1) > 1 or any(length < -1 for length in requested):
        raise ShapeContractError("target shape allows at most one inferred -1")

    source_size = math.prod(source)
    known_product = math.prod(length for length in requested if length != -1)
    if -1 in requested:
        if known_product == 0 or source_size % known_product:
            raise ShapeContractError(f"cannot infer target {requested} from {source_size} elements")
        inferred = source_size // known_product
        requested = tuple(inferred if length == -1 else length for length in requested)

    target_size = math.prod(requested)
    if target_size != source_size:
        raise ShapeContractError(
            f"reshape changes element count from {source_size} to {target_size}"
        )
    return requested


def transpose_shape(shape: Sequence[int], axes: Sequence[int] | None = None) -> tuple[int, ...]:
    source = validate_shape(shape)
    raw_axes = tuple(reversed(range(len(source)))) if axes is None else tuple(axes)
    if any(isinstance(axis, bool) or not isinstance(axis, Integral) for axis in raw_axes):
        raise ShapeContractError("transpose axes must contain integers")
    permutation = tuple(int(axis) for axis in raw_axes)
    if sorted(permutation) != list(range(len(source))):
        raise ShapeContractError(f"axes {permutation} are not a permutation of dimensions")
    return tuple(source[index] for index in permutation)


def expand_dims_shape(shape: Sequence[int], axis: int) -> tuple[int, ...]:
    source = validate_shape(shape)
    if isinstance(axis, bool) or not isinstance(axis, Integral):
        raise ShapeContractError("expand axis must be an integer")
    result_ndim = len(source) + 1
    axis_number = int(axis)
    normalized = axis_number + result_ndim if axis_number < 0 else axis_number
    if normalized < 0 or normalized >= result_ndim:
        raise ShapeContractError(
            f"axis {axis_number} is out of bounds for expanded ndim={result_ndim}"
        )
    return (*source[:normalized], 1, *source[normalized:])


def axis_descriptors(
    shape: Sequence[int],
    axis_names: Sequence[str] | None = None,
) -> list[dict[str, int | str]]:
    valid_shape = validate_shape(shape)
    names = None if axis_names is None else validate_axis_names(axis_names, len(valid_shape))
    return [
        {
            "axis": index,
            **({"name": names[index]} if names is not None else {}),
            "length": length,
        }
        for index, length in enumerate(valid_shape)
    ]


def assert_shape(
    array: np.ndarray,
    expected: Sequence[int],
    *,
    name: str = "array",
    axis_names: Sequence[str] | None = None,
) -> None:
    expected_shape = validate_shape(expected, name="expected shape")
    names = None if axis_names is None else validate_axis_names(axis_names, len(expected_shape))
    if array.shape == expected_shape:
        return

    named_expected = (
        ", ".join(
            f"{axis_name}={length}"
            for axis_name, length in zip(names, expected_shape, strict=True)
        )
        if names is not None
        else str(expected_shape)
    )
    raise ShapeContractError(
        f"{name} has shape {array.shape}, expected {expected_shape} ({named_expected})"
    )


def parse_json_sequence(raw: str, *, name: str) -> list[Any]:
    try:
        value = json.loads(raw)
    except json.JSONDecodeError as error:
        raise ShapeContractError(f"{name} must be valid JSON") from error
    if not isinstance(value, list):
        raise ShapeContractError(f"{name} must be a JSON array")
    return value


def build_report(
    shape: Sequence[int],
    *,
    axis_names: Sequence[str] | None = None,
    axis: int | Iterable[int] | None = None,
    keepdims: bool = False,
    reshape: Sequence[int] | None = None,
    reshape_axis_names: Sequence[str] | None = None,
    transpose: Sequence[int] | None = None,
    expand_axis: int | None = None,
    expand_axis_name: str | None = None,
) -> dict[str, Any]:
    source = validate_shape(shape)
    names = None if axis_names is None else validate_axis_names(axis_names, len(source))
    normalized_reduction_axes = normalize_axes(axis, len(source))
    reduced_shape = reduction_shape(source, axis, keepdims=keepdims)
    reduced_names = None
    if names is not None:
        reduced_names = (
            names
            if keepdims
            else tuple(
                axis_name
                for index, axis_name in enumerate(names)
                if index not in normalized_reduction_axes
            )
        )

    report: dict[str, Any] = {
        "source_shape": list(source),
        "ndim": len(source),
        "size": math.prod(source),
        "axis_names": None if names is None else list(names),
        "axes": axis_descriptors(source, names),
        "operations": {
            "reduction": {
                "normalized_axes": list(normalized_reduction_axes),
                "operated_axis_names": (
                    None
                    if names is None
                    else [names[index] for index in normalized_reduction_axes]
                ),
                "keepdims": keepdims,
                "result_shape": list(reduced_shape),
                "result_axis_names": None if reduced_names is None else list(reduced_names),
            }
        },
    }

    if reshape is not None:
        reshaped = reshape_shape(source, reshape)
        target_names = (
            None
            if reshape_axis_names is None
            else validate_axis_names(reshape_axis_names, len(reshaped), name="reshape axis names")
        )
        reshape_report: dict[str, Any] = {
            "requested": list(reshape),
            "result_shape": list(reshaped),
            "result_axis_names": None if target_names is None else list(target_names),
        }
        if names is not None and target_names is None:
            reshape_report["semantic_warning"] = (
                "reshape preserves element count, not axis meaning; name target axes explicitly"
            )
        report["operations"]["reshape"] = reshape_report
    elif reshape_axis_names is not None:
        raise ShapeContractError("reshape axis names require a reshape target")

    if transpose is not None:
        transposed = transpose_shape(source, transpose)
        transposed_names = None if names is None else tuple(names[index] for index in transpose)
        report["operations"]["transpose"] = {
            "axes": list(transpose),
            "result_shape": list(transposed),
            "result_axis_names": (
                None if transposed_names is None else list(transposed_names)
            ),
        }

    if expand_axis is not None:
        expanded = expand_dims_shape(source, expand_axis)
        expanded_names = None
        if names is not None and expand_axis_name is not None:
            result_ndim = len(source) + 1
            normalized = expand_axis + result_ndim if expand_axis < 0 else expand_axis
            expanded_names = (*names[:normalized], expand_axis_name, *names[normalized:])
            expanded_names = validate_axis_names(expanded_names, result_ndim)
        expand_report: dict[str, Any] = {
            "axis": expand_axis,
            "result_shape": list(expanded),
            "result_axis_names": None if expanded_names is None else list(expanded_names),
        }
        if names is not None and expand_axis_name is None:
            expand_report["semantic_warning"] = "name the new length-one axis explicitly"
        report["operations"]["expand_dims"] = expand_report
    elif expand_axis_name is not None:
        raise ShapeContractError("expand axis name requires expand_axis")

    return report


def main() -> int:
    parser = argparse.ArgumentParser(
        description="Predict NumPy shapes and preserve the semantic names of their axes"
    )
    parser.add_argument("--shape", default="[2, 3, 2]", help="JSON array of axis lengths")
    parser.add_argument("--axis-names", help="Optional JSON array of semantic axis names")
    parser.add_argument("--axis", type=int, help="Reduction axis; default reduces all axes")
    parser.add_argument("--keepdims", action="store_true")
    parser.add_argument("--reshape", help="Optional target shape as JSON")
    parser.add_argument("--reshape-axis-names", help="Semantic names for reshape result axes")
    parser.add_argument("--transpose", help="Optional axis permutation as JSON")
    parser.add_argument("--expand-axis", type=int)
    parser.add_argument("--expand-axis-name", help="Semantic name for a newly inserted axis")
    parser.add_argument("--output", type=Path)
    args = parser.parse_args()

    try:
        report = build_report(
            parse_json_sequence(args.shape, name="shape"),
            axis_names=(
                parse_json_sequence(args.axis_names, name="axis names")
                if args.axis_names
                else None
            ),
            axis=args.axis,
            keepdims=args.keepdims,
            reshape=(parse_json_sequence(args.reshape, name="reshape") if args.reshape else None),
            reshape_axis_names=(
                parse_json_sequence(args.reshape_axis_names, name="reshape axis names")
                if args.reshape_axis_names
                else None
            ),
            transpose=(
                parse_json_sequence(args.transpose, name="transpose") if args.transpose else None
            ),
            expand_axis=args.expand_axis,
            expand_axis_name=args.expand_axis_name,
        )
    except ShapeContractError as error:
        parser.exit(2, f"shape-contract: {error}\n")

    text = json.dumps(report, ensure_ascii=False, indent=2) + "\n"
    if args.output:
        args.output.write_text(text, encoding="utf-8")
    else:
        print(text, end="")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
