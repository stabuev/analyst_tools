Кортеж size непосредственно задает форму возвращаемого массива; смысл каждой оси определяет автор симуляции.

Случайность и воспроизводимые симуляции

Случайные числа становятся воспроизводимым входом, когда явно зафиксированы генератор, seed, параметры и порядок вызовов.

Тип: Build Треки: Core Пререквизит: 02-numpy/07-broadcasting Время: ~60 минут Результат: проектирует предметные оси случайного эксперимента, использует локальный Generator, фиксирует seed и способ получения чисел и проверяет форму, тип, память и повторяемость симуляции.

Цели обучения

  • объяснить разницу между seed, состоянием генератора, потоком и отдельным случайным значением;
  • создавать локальный np.random.Generator и явно передавать его в вычислительную функцию;
  • задавать через size не просто размеры, а предметные оси случайного эксперимента;
  • превращать равномерные числа в простую модель бинарного события и применять параметры по группам через broadcasting;
  • проверять shape, dtype, диапазон, смысл осей и память до интерпретации результата;
  • фиксировать минимальные сведения, необходимые для повторения запуска;
  • отличать воспроизводимость вычисления от правильности вероятностной модели.

Проблема

Команда поддержки хочет проверить технический сценарий: сколько нарушений SLA может получиться при разных вероятностях нарушения в каналах chat, email и phone.

Для примера считаем заданными вероятности:

python
import numpy as np

channel_names = ("chat", "email", "phone")
breach_probabilities = np.array([0.08, 0.15, 0.30])

Нужно создать 1_000 сценариев. В каждом сценарии есть по 50 условных обращений каждого канала. Оси результата:

text
(scenario=1000, observation=50, channel=3)

Один элемент отвечает на вопрос:

text
произошло ли нарушение SLA
в сценарии s
для условного обращения i
в канале c

Наивная реализация может дать меняющиеся числа при каждом запуске, незаметно использовать глобальное состояние или применить вероятность не к той оси. Даже фиксированный seed не поможет коллеге, если неизвестны параметры, форма запроса и версия вычислительной среды.

Нам нужен не «случайный ответ», а воспроизводимый процесс:

  1. явно создать источник случайных чисел;
  2. назвать оси запрашиваемого массива;
  3. отдельно описать правило превращения числа в событие;
  4. сохранить параметры запуска;
  5. проверить форму, тип и память;
  6. не выдавать техническую симуляцию за доказанный прогноз.

Концепция

В симуляции есть три разных слоя

Полезно не смешивать три действия:

text
источник случайных чисел -> модель события -> сводка результатов

В нашем примере:

  1. Generator.random создает числа от 0 включительно до 1 не включительно;
  2. сравнение числа с вероятностью канала создает True или False;
  3. sum и mean сворачивают нужные оси.
python
draws = rng.random((1_000, 50, 3))
breaches = draws < breach_probabilities
breach_rate_by_channel = breaches.mean(axis=(0, 1))

Если итог выглядит неправдоподобно, можно проверять каждый слой отдельно. Это намного понятнее, чем считать rng.normal(...) или rng.choice(...) готовой моделью задачи.

Псевдослучайный поток является детерминированным

Обычный генератор NumPy является PRNG — генератором псевдослучайных чисел. Внутри него есть состояние. Каждый запрос:

  1. читает текущее состояние;
  2. детерминированно получает следующее значение;
  3. переводит генератор в новое состояние.

Упрощенная схема:

text
state_0 --next--> value_1, state_1
state_1 --next--> value_2, state_2
state_2 --next--> value_3, state_3

Числа выглядят случайными для задачи, но одинаковое начальное состояние и одинаковая последовательность запросов дают одинаковый поток.

Seed и состояние — не одно и то же

Seed используется для инициализации генератора:

python
first_rng = np.random.default_rng(42)
second_rng = np.random.default_rng(42)

Два свежих генератора начинают одинаково:

python
np.testing.assert_array_equal(first_rng.random(5), second_rng.random(5))

Но один генератор меняет состояние после вызова:

python
rng = np.random.default_rng(42)

first_part = rng.random(5)
second_part = rng.random(5)

assert not np.array_equal(first_part, second_part)

Seed не является числом, которое генератор «добавляет» к каждому результату. Он задает начало последовательности.

Локальный Generator делает зависимость видимой

Для нового кода NumPy предоставляет локальный объект:

python
rng = np.random.default_rng(42)

У него две роли:

  • Generator предоставляет методы random, integers, choice, normal и другие;
  • внутренний BitGenerator хранит состояние и создает исходный поток битов.

Проверить используемый BitGenerator можно явно:

python
assert type(rng).__name__ == "Generator"
assert type(rng.bit_generator).__name__ == "PCG64"

Старый стиль:

python
np.random.seed(42)
values = np.random.random(10)

меняет общее состояние модуля. Тогда удаленный участок программы может использовать или изменить тот же поток. Локальный объект проще передавать, тестировать и изолировать.

Кто должен создавать генератор

На границе приложения удобно принимать seed и создавать генератор один раз:

python
def run(seed):
    rng = np.random.default_rng(seed)
    return simulate(rng=rng)

Внутренняя вычислительная функция принимает готовый объект:

python
def simulate(*, rng):
    return rng.random(10)

Так функция явно сообщает, что использует случайность. Ее можно вызвать с новым потоком или продолжить уже существующий.

Плохой вариант:

python
def simulate():
    rng = np.random.default_rng(42)
    return rng.random(10)

Каждый вызов молча начинает один и тот же поток заново. Иногда это случайно выглядит как «идеальная воспроизводимость», хотя функция фактически не умеет продолжать эксперимент.

size задает оси результата

Аргумент size является частью модели данных:

python
draws = rng.random(size=(1_000, 50, 3))

Недостаточно сказать «трехмерный массив». Нужно назвать оси:

text
axis 0: scenario     — независимый повтор технического сценария
axis 1: observation  — позиция условного обращения внутри канала
axis 2: channel      — chat, email или phone
python
assert draws.shape == (1_000, 50, 3)
assert draws.dtype == np.dtype("float64")
assert np.all((draws >= 0.0) & (draws < 1.0))

Если поменять две оси местами, количество элементов не изменится, но последующие агрегаты будут отвечать на другие вопросы.

Простое бинарное событие строится сравнением

Пусть u равномерно принимает значения от 0 до 1. Правило:

text
event = u < p

создает событие с заданной вероятностью p.

Для одного значения:

python
draw = 0.12
probability = 0.15
event = draw < probability

assert event

Для массива:

python
breaches = draws < breach_probabilities

assert breaches.shape == draws.shape
assert breaches.dtype == np.dtype("bool")

Это минимальная модель бинарного события. Она не утверждает, что выбранные вероятности верны для реальной поддержки. Их происхождение и статистическая оценка являются отдельной задачей.

Broadcasting применяет вероятность к каналу

Формы перед сравнением:

text
draws:        (scenario=1000, observation=50, channel=3)
probabilities:                                      (3,)
aligned:        (1,              1,             3)
result:      (1000,             50,             3)

Последняя ось массива и единственная ось вероятностей обе означают channel. Поэтому broadcasting здесь совместим и позиционно, и по смыслу.

Если вероятности относятся к другой оси, единичные оси нужно добавить явно. Совпадение длин само по себе не доказывает правильность модели.

Типы меняются по ходу модели

В этом расчете есть три типа:

text
draws          float64   случайные числа [0, 1)
probabilities  float64   параметры модели
breaches       bool      результат сравнения

После агрегации:

python
count_by_scenario = breaches.sum(axis=(1, 2), dtype=np.int64)
rate_by_channel = breaches.mean(axis=(0, 1))

assert count_by_scenario.dtype == np.dtype("int64")
assert rate_by_channel.dtype == np.dtype("float64")

Случайность не отменяет обычные правила dtype, ufunc и агрегатов из предыдущих уроков.

Для повторения нужен не только seed

Условие точного повторения значительно строже записи seed=42. Нужно сохранить:

  • тип генератора и BitGenerator;
  • seed или иное начальное состояние;
  • название метода;
  • аргументы метода, включая size;
  • порядок всех предыдущих вызовов этого генератора;
  • параметры модели;
  • версию кода и вычислительной среды.

Например, это два разных протокола вызовов:

python
rng.random(6)
rng.random((2, 3))

Даже если текущая версия иногда возвращает совместимый поток, нельзя объявлять разные последовательности вызовов одним воспроизводимым контрактом. Фиксируйте фактически использованный вызов.

Воспроизводимость не доказывает правильность модели

Одинаковый seed позволяет повторить ошибку так же точно, как правильный расчет.

В нашем примере остаются предметные вопросы:

  • откуда получены вероятности нарушения SLA;
  • стабильны ли они во времени;
  • можно ли считать обращения независимыми;
  • одинаково ли определено нарушение во всех каналах;
  • соответствует ли число наблюдений ожидаемой нагрузке.

Этот урок проверяет вычислительный механизм. Статистическое обоснование моделей будет развиваться позже.

Оцените память до генерации

В основном примере создаются два массива одинаковой формы:

  • draws с float64: 8 байт на элемент;
  • breaches с bool: 1 байт на элемент.

Для формы (1_000, 50, 3):

text
element_count = 1_000 * 50 * 3 = 150_000
draws_bytes = 150_000 * 8 = 1_200_000
events_bytes = 150_000 * 1 =   150_000
minimum_working_bytes ≈      1_350_000

Форма (1_000_000, 50_000, 3) уже неприемлема. Проверка должна умножить длины осей и остановить запуск до выделения массива.

Соберите это

Шаг 1. Сделайте состояние явным

Уменьшенная модель PRNG нужна только для понимания перехода состояния. Она не подходит для аналитических расчетов:

python
def tiny_random_step(state):
    next_state = (5 * state + 1) % 16
    value = next_state / 16
    return next_state, value

Вызовите ее несколько раз:

python
state = 3
state, first = tiny_random_step(state)
state, second = tiny_random_step(state)
state, third = tiny_random_step(state)

assert [first, second, third] == [0.0, 0.0625, 0.375]

Новый seed 3 снова приведет к той же последовательности. Если продолжить с текущим state, появятся следующие значения, а не первые.

Шаг 2. Отделите числа от модели события

Возьмите заранее заданные числа, чтобы проверить модель без случайности:

python
draws = np.array(
    [
        [[0.05, 0.20, 0.10], [0.50, 0.10, 0.40]],
        [[0.07, 0.14, 0.31], [0.09, 0.16, 0.29]],
    ]
)
probabilities = np.array([0.08, 0.15, 0.30])

events = draws < probabilities

Проверьте контракт:

python
assert draws.shape == (2, 2, 3)
assert probabilities.shape == (3,)
assert events.shape == (2, 2, 3)
assert events.dtype == np.dtype("bool")

np.testing.assert_array_equal(
    events[0, 0],
    [True, False, True],
)

Теперь известно, что сравнение и broadcasting работают правильно независимо от PRNG.

Шаг 3. Передайте источник случайности явно

python
def simulate_binary_events(*, probabilities, scenario_count, observations, rng):
    shape = (scenario_count, observations, len(probabilities))
    draws = rng.random(shape)
    events = draws < probabilities
    return draws, events
python
rng = np.random.default_rng(42)
draws, events = simulate_binary_events(
    probabilities=probabilities,
    scenario_count=1_000,
    observations=50,
    rng=rng,
)

Функция не решает сама, какой seed использовать. Владелец эксперимента принимает это решение снаружи.

Шаг 4. Сохраните сведения о запуске

Минимальный manifest:

python
manifest = {
    "generator": type(rng).__name__,
    "bit_generator": type(rng.bit_generator).__name__,
    "seed": 42,
    "method": "Generator.random",
    "size": list(draws.shape),
    "probabilities": probabilities.tolist(),
}

Это не заменяет lock-файл и версию кода, но делает сам случайный вызов видимым.

Шаг 5. Проверьте память до вызова

python
element_count = 1_000 * 50 * 3
working_nbytes = element_count * (
    np.dtype("float64").itemsize + np.dtype("bool").itemsize
)

assert working_nbytes == 1_350_000

В готовом артефакте запуск отклоняется, если оценка превышает указанный лимит.

Используйте это

Создайте два одинаковых запуска

python
first_rng = np.random.default_rng(42)
second_rng = np.random.default_rng(42)

first_draws, first_events = simulate_binary_events(
    probabilities=probabilities,
    scenario_count=10,
    observations=5,
    rng=first_rng,
)
second_draws, second_events = simulate_binary_events(
    probabilities=probabilities,
    scenario_count=10,
    observations=5,
    rng=second_rng,
)

np.testing.assert_array_equal(first_draws, second_draws)
np.testing.assert_array_equal(first_events, second_events)

Здесь совпали не только seed, но и метод, форма запроса и порядок вызовов.

Продолжите существующий поток

python
rng = np.random.default_rng(42)

first_batch, _ = simulate_binary_events(
    probabilities=probabilities,
    scenario_count=10,
    observations=5,
    rng=rng,
)
second_batch, _ = simulate_binary_events(
    probabilities=probabilities,
    scenario_count=10,
    observations=5,
    rng=rng,
)

assert not np.array_equal(first_batch, second_batch)

Это правильное поведение: второй вызов продолжает поток. Для повторения всего расчета создайте новый генератор на границе запуска, а не переинициализируйте его внутри цикла.

Сведите правильные оси

python
breach_count_by_scenario = events.sum(axis=(1, 2), dtype=np.int64)
breach_rate_by_channel = events.mean(axis=(0, 1))

assert breach_count_by_scenario.shape == (1_000,)
assert breach_rate_by_channel.shape == (3,)

Первая сводка оставляет ось scenario. Вторая оставляет ось channel. Это прямое применение урока об агрегатах.

bash

Отчет содержит:

  • предметные имена осей;
  • shape, dtype и границы случайных чисел;
  • параметры бинарной модели;
  • сводки по группам и сценариям;

Fingerprint помогает обнаружить изменение потока, но не доказывает статистическую правильность результата и не заменяет проверку модели.

Опционально: распределение выборочного среднего

Этот раздел не нужен для завершения урока. Он рассчитан на тех, кто уже различает популяцию и выборку и знаком со стандартным отклонением. Подробно механизм отбора, распределения, оценки, standard error и repeated sampling рассматриваются в фазе 09 «Прикладная статистика».

Предположим, что задана вероятностная модель наблюдений со средним 100 и стандартным отклонением 15. Возьмем много независимых выборок по 25 наблюдений:

python
rng = np.random.default_rng(42)

samples = rng.normal(
    loc=100,
    scale=15,
    size=(10_000, 25),
)
sample_means = samples.mean(axis=1)

Назовем оси:

text
samples:      (repetition=10000, observation=25)
sample_means: (repetition=10000,)

Каждая строка является отдельной повторной выборкой. После mean(axis=1) остается одно среднее на повтор. Набор этих средних приближает sampling distribution среднего.

При независимых наблюдениях с конечным стандартным отклонением теоретический разброс среднего уменьшается с размером выборки:

text
SE = sigma / sqrt(n)
python
theoretical_se = 15 / np.sqrt(25)
empirical_se = sample_means.std(ddof=1)

np.testing.assert_allclose(empirical_se, theoretical_se, rtol=0.03)

Здесь симуляция иллюстрирует теоретический результат, а не заменяет его доказательство. Сначала должны быть обоснованы модель наблюдений, независимость и смысл параметров.

Если эти слова пока незнакомы, спокойно пропустите раздел. Для самостоятельного входа используйте открытый урок Penn State Sampling Distributions, а затем вернитесь к фазе 09 курса.

Сломайте это

Скрыть seed

python
def run():
    rng = np.random.default_rng()
    return rng.random(10)

Без явного seed NumPy получает свежую энтропию из операционной системы. Это допустимо для одноразового исследования, но такой запуск нельзя повторить только по сохраненным параметрам модели.

Изменить глобальное состояние

python
np.random.seed(42)

Библиотечная функция не должна незаметно управлять общим потоком процесса. Передавайте локальный Generator.

Создавать генератор заново внутри цикла

python
for _ in range(100):
    rng = np.random.default_rng(42)
    value = rng.random()

Цикл сто раз получает первое значение одного и того же потока. Создайте генератор один раз до цикла.

Сохранить seed, но изменить порядок вызовов

python

diagnostic = rng.random(10)  # новый вызов сдвинул состояние
simulation = rng.random(100)

Добавление диагностического запроса изменило все последующие значения. Seed совпадает, но протокол вызовов уже другой.

Применить параметры к случайно совпавшей оси

Пусть формы случайных чисел и вероятностей таковы:

text
draws:        (scenario=3, observation=50, channel=3)
probabilities:                                    (3,)

Если три вероятности на самом деле относятся к scenario, NumPy все равно применит их к channel. Broadcasting проверяет позиции и длины, а не смысл.

Передать невозможную вероятность

python
probabilities = np.array([-0.1, 0.5, 1.2])

Сравнение выполнится, но отрицательная вероятность никогда не даст событие, а значение выше единицы даст его всегда. Проверяйте конечность и диапазон [0, 1] заранее.

Перепутать повторяемость и правильность

text
seed = 42
phone_probability = 0.99  # значение ничем не обосновано

Расчет будет воспроизводимым, но модель останется необоснованной.

Запросить огромную форму

text
(scenario=1_000_000, observation=50_000, channel=3)

Нельзя сначала вызвать rng.random, а потом проверять nbytes: память уже могла быть выделена. Оцените произведение осей заранее.

Зафиксировать весь массив как вечный тест

Точный поток зависит не только от seed. Изменение BitGenerator, последовательности вызовов, версии NumPy или окружения может изменить значения. Для бизнес-логики прежде всего проверяйте инварианты: форму, диапазон, типы, оси и детерминированность внутри зафиксированной среды.

Термин Распространенное заблуждение Точное значение

Поток «Один случайный массив» Последовательность значений, получаемая из последовательных состояний

BitGenerator «Распределение данных» Компонент, который хранит состояние и создает исходные случайные биты

01

Два свежих Generator созданы с одним seed и получают одинаковый вызов random((10, 3)). Что ожидается?

Свежие генераторы с одинаковым началом и одинаковым протоколом вызовов воспроизводят один поток в согласованной среде.

02

Почему два последовательных вызова random(5) одного Generator обычно возвращают разные значения?

Каждый запрос продвигает внутреннее состояние, поэтому следующий вызов продолжает поток.

03

draws имеет оси (scenario, observation, channel) и форму (1000, 50, 3). Как применится probability формы (3,)?

Broadcasting выравнивает формы справа; здесь позиция совпадает и с предметным смыслом channel.

04

Какой dtype получится у выражения rng.random(shape) < probabilities?

Сравнение вещественных чисел с порогами создает булев массив событий.

05

Что произойдет, если внутри каждой итерации цикла создавать default_rng(42) и брать первое число?

Каждая итерация заново создает одинаковое начальное состояние. Generator нужно создать один раз до цикла.

06

Каких сведений недостаточно для надежного повторения симуляции?

Помимо seed нужны генератор, метод и аргументы, порядок вызовов, параметры модели, код и среда.

07

Форма симуляции содержит 1 000 000 × 50 000 × 3 элементов. Когда нужно оценивать память?

После вызова память уже может быть выделена. Сначала рассчитывают число элементов и стоимость рабочих массивов.

08

Симуляция с необоснованной вероятностью 0.99 точно повторяется по seed. Какой вывод корректен?

Воспроизводимость относится к вычислению и не подтверждает происхождение параметров или предпосылки модели.

from __future__ import annotations

import argparse
import hashlib
import json
import math
from collections.abc import Sequence
from numbers import Integral, Real
from pathlib import Path
from typing import Any

import numpy as np


class SimulationError(ValueError):
    """Raised when a simulation contract is incomplete or unsafe."""


def normalize_positive_integer(value: object, *, name: str) -> int:
    if isinstance(value, bool) or not isinstance(value, Integral):
        raise SimulationError(f"{name} must be an integer")
    normalized = int(value)
    if normalized <= 0:
        raise SimulationError(f"{name} must be positive")
    return normalized


def normalize_seed(value: object) -> int:
    if isinstance(value, bool) or not isinstance(value, Integral):
        raise SimulationError("seed must be an integer")
    seed = int(value)
    if seed < 0:
        raise SimulationError("seed must be non-negative")
    return seed


def normalize_probabilities(values: object) -> np.ndarray:
    if isinstance(values, np.ndarray):
        if values.ndim != 1:
            raise SimulationError("probabilities must be one-dimensional")
        raw_values: Sequence[object] = values.tolist()
    elif isinstance(values, (str, bytes)) or not isinstance(values, Sequence):
        raise SimulationError("probabilities must be a non-empty sequence")
    else:
        raw_values = values
    if not raw_values:
        raise SimulationError("probabilities must be a non-empty sequence")
    normalized: list[float] = []
    for value in raw_values:
        if isinstance(value, bool) or not isinstance(value, Real):
            raise SimulationError("each probability must be a real number")
        probability = float(value)
        if not math.isfinite(probability) or not 0.0 <= probability <= 1.0:
            raise SimulationError("each probability must be finite and between 0 and 1")
        normalized.append(probability)
    return np.asarray(normalized, dtype=np.float64)


def normalize_group_names(values: object, *, group_count: int) -> tuple[str, ...]:
    if isinstance(values, (str, bytes)) or not isinstance(values, Sequence):
        raise SimulationError("group_names must be a sequence")
    names = tuple(values)
    if len(names) != group_count:
        raise SimulationError(f"expected {group_count} group names, received {len(names)}")
    if any(not isinstance(name, str) or not name.strip() for name in names):
        raise SimulationError("group names must be non-empty strings")
    if len(set(names)) != len(names):
        raise SimulationError("group names must be unique")
    return names


def memory_contract(
    *,
    scenario_count: int,
    observations_per_group: int,
    group_count: int,
    memory_limit_mb: float,
) -> dict[str, int | float]:
    if (
        isinstance(memory_limit_mb, bool)
        or not isinstance(memory_limit_mb, Real)
        or not math.isfinite(float(memory_limit_mb))
        or memory_limit_mb <= 0
    ):
        raise SimulationError("memory_limit_mb must be a positive finite number")

    element_count = scenario_count * observations_per_group * group_count
    draws_nbytes = element_count * np.dtype(np.float64).itemsize
    events_nbytes = element_count * np.dtype(np.bool_).itemsize
    estimated_working_nbytes = draws_nbytes + events_nbytes
    memory_limit_bytes = int(float(memory_limit_mb) * 1024 * 1024)
    if estimated_working_nbytes > memory_limit_bytes:
        raise SimulationError(
            "simulation needs approximately "
            f"{estimated_working_nbytes} bytes, above limit {memory_limit_bytes}"
        )
    return {
        "element_count": element_count,
        "draws_nbytes": draws_nbytes,
        "events_nbytes": events_nbytes,
        "estimated_working_nbytes": estimated_working_nbytes,
        "memory_limit_mb": float(memory_limit_mb),
    }


def simulate_binary_events(
    *,
    probabilities: Sequence[object],
    scenario_count: int,
    observations_per_group: int,
    rng: np.random.Generator,
    memory_limit_mb: float = 256.0,
) -> tuple[np.ndarray, np.ndarray]:
    if not isinstance(rng, np.random.Generator):
        raise SimulationError("rng must be a numpy.random.Generator")
    normalized_probabilities = normalize_probabilities(probabilities)
    scenarios = normalize_positive_integer(scenario_count, name="scenario_count")
    observations = normalize_positive_integer(
        observations_per_group,
        name="observations_per_group",
    )
    shape = (scenarios, observations, normalized_probabilities.size)
    memory_contract(
        scenario_count=scenarios,
        observations_per_group=observations,
        group_count=normalized_probabilities.size,
        memory_limit_mb=memory_limit_mb,
    )

    draws = rng.random(shape)
    events = draws < normalized_probabilities
    return draws, events


def array_fingerprint(array: np.ndarray) -> str:
    contiguous = np.ascontiguousarray(array)
    return hashlib.sha256(contiguous.tobytes()).hexdigest()


def simulation_report(
    *,
    probabilities: Sequence[object],
    group_names: Sequence[object],
    scenario_count: int,
    observations_per_group: int,
    seed: int,
    memory_limit_mb: float = 256.0,
) -> dict[str, Any]:
    normalized_probabilities = normalize_probabilities(probabilities)
    names = normalize_group_names(
        group_names,
        group_count=normalized_probabilities.size,
    )
    scenarios = normalize_positive_integer(scenario_count, name="scenario_count")
    observations = normalize_positive_integer(
        observations_per_group,
        name="observations_per_group",
    )
    normalized_seed = normalize_seed(seed)
    memory = memory_contract(
        scenario_count=scenarios,
        observations_per_group=observations,
        group_count=normalized_probabilities.size,
        memory_limit_mb=memory_limit_mb,
    )

    rng = np.random.default_rng(normalized_seed)
    draws, events = simulate_binary_events(
        probabilities=normalized_probabilities,
        scenario_count=scenarios,
        observations_per_group=observations,
        rng=rng,
        memory_limit_mb=memory_limit_mb,
    )
    group_event_counts = events.sum(axis=(0, 1), dtype=np.int64)
    group_event_rates = events.mean(axis=(0, 1))
    scenario_event_counts = events.sum(axis=(1, 2), dtype=np.int64)

    return {
        "manifest": {
            "generator": type(rng).__name__,
            "bit_generator": type(rng.bit_generator).__name__,
            "seed": normalized_seed,
            "call_contract": {
                "method": "Generator.random",
                "size": list(draws.shape),
                "dtype": str(draws.dtype),
            },
        },
        "model": {
            "description": (
                "событие происходит, когда равномерное число меньше вероятности группы"
            ),
            "group_names": list(names),
            "event_probabilities": normalized_probabilities.tolist(),
        },
        "arrays": {
            "draws": {
                "shape": list(draws.shape),
                "axis_names": ["scenario", "observation", "group"],
                "dtype": str(draws.dtype),
                "minimum": float(draws.min()),
                "maximum": float(draws.max()),
                "sha256": array_fingerprint(draws),
            },
            "events": {
                "shape": list(events.shape),
                "axis_names": ["scenario", "observation", "group"],
                "dtype": str(events.dtype),
                "sha256": array_fingerprint(events),
            },
        },
        "summary": {
            "event_count_by_group": dict(zip(names, group_event_counts.tolist(), strict=True)),
            "event_rate_by_group": dict(zip(names, group_event_rates.tolist(), strict=True)),
            "first_five_scenario_event_counts": scenario_event_counts[:5].tolist(),
            "first_three_observations": events[0, :3, :].tolist(),
        },
        "memory": memory,
    }


def parse_json(raw: str, *, name: str) -> Any:
    try:
        return json.loads(raw)
    except json.JSONDecodeError as error:
        raise SimulationError(f"{name} must be valid JSON") from error


def main() -> int:
    parser = argparse.ArgumentParser(
        description="Run a reproducible binary-event simulation with a manifest"
    )
    parser.add_argument(
        "--probabilities",
        default="[0.08, 0.15, 0.30]",
        help="JSON array with one event probability per group",
    )
    parser.add_argument(
        "--group-names",
        default='["chat", "email", "phone"]',
        help="JSON array with semantic names for the group axis",
    )
    parser.add_argument("--scenarios", type=int, default=1_000)
    parser.add_argument("--observations-per-group", type=int, default=50)
    parser.add_argument("--seed", type=int, default=42)
    parser.add_argument("--memory-limit-mb", type=float, default=256.0)
    parser.add_argument("--output", type=Path)
    args = parser.parse_args()

    try:
        probabilities = parse_json(args.probabilities, name="probabilities")
        group_names = parse_json(args.group_names, name="group_names")
        report = simulation_report(
            probabilities=probabilities,
            group_names=group_names,
            scenario_count=args.scenarios,
            observations_per_group=args.observations_per_group,
            seed=args.seed,
            memory_limit_mb=args.memory_limit_mb,
        )
    except SimulationError as error:
        parser.exit(2, f"event-simulator: {error}\n")

    text = json.dumps(report, ensure_ascii=False, indent=2) + "\n"
    if args.output:
        args.output.write_text(text, encoding="utf-8")
    else:
        print(text, end="")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
