Как Python интерпретирует выражение [10, 20] * 2?

Какой набор свойств лучше всего описывает модель обычного ndarray?

Обычный ndarray хранит элементы одного dtype, имеет прямоугольную форму и не меняет общее число элементов без создания нового массива.

ndarray и модель массива

ndarray хранит однородные данные в прямоугольной многомерной структуре. Чтобы уверенно работать с NumPy, сначала научитесь читать контракт массива: ndim, shape, size и dtype.

Проблема

Допустим, аналитик получил наблюдения по двум магазинам за три дня:

python
sales = [
    [12, 15, 9],
    [10, 11, 14],
]

Вложенный список выглядит как таблица, но сам Python не гарантирует этого. Во второй строке может оказаться меньше значений:

python
ragged_sales = [
    [12, 15, 9],
    [10, 11],
]

Или среди чисел внезапно появится строка:

python
mixed_sales = [
    [12, 15, 9],
    [10, "нет данных", 14],
]

Для списка обе структуры допустимы. Список Python является универсальным изменяемым контейнером: его элементы могут иметь разные типы, вложенные списки могут иметь разную длину, а сам контейнер может расти.

Это удобно для общего программирования, но численным алгоритмам нужен более строгий контракт. Операция над таблицей должна заранее знать:

  • сколько у данных осей;
  • какова длина каждой оси;
  • сколько всего элементов;
  • как числа представлены в памяти.

Такой контракт дает numpy.ndarray.

Концепция

Импортируем NumPy по общепринятому короткому имени:

python
import numpy as np

Создадим массив из исходного списка:

python
sales_array = np.array(sales)
print(sales_array)
text
[[12 15  9]
 [10 11 14]]

У массива есть четыре базовых атрибута:

python
print(sales_array.ndim)
print(sales_array.shape)
print(sales_array.size)
print(sales_array.dtype)

Пример результата:

text
2
(2, 3)
6
int64

Точное имя целочисленного dtype может зависеть от платформы. Смысл атрибутов от этого не меняется.

Атрибут Вопрос Значение для примера
ndim Сколько осей у массива? 2
shape Какова длина каждой оси? (2, 3)
size Сколько всего элементов? 6
dtype Как представлен каждый элемент? целочисленный тип

Для любого обычного массива выполняются два полезных инварианта:

python
sales_array.ndim == len(sales_array.shape)
sales_array.size == np.prod(sales_array.shape)

В нашем примере:

text
ndim = len((2, 3)) = 2
size = 2 * 3 = 6

Оси и форма

Форма читается слева направо:

  • скаляр имеет форму ();
  • вектор из трех элементов имеет форму (3,);
  • матрица из двух строк и трех столбцов имеет форму (2, 3);
  • набор из четырех матриц формы (2, 3) имеет форму (4, 2, 3).

Запятая в (3,) важна: это кортеж с одним элементом, а не число в скобках.

В NumPy оси нумеруются с нуля. Для массива формы (2, 3):

  • ось 0 имеет длину 2;
  • ось 1 имеет длину 3.

Подробные операции по осям появятся в следующем уроке. Сейчас достаточно научиться читать форму как часть контракта данных.

Однородный тип

У списка Python нет одного общего типа элементов:

python
python_values = [1, 2.5, "3"]

У ndarray есть один dtype для всех элементов. При создании массива NumPy пытается найти общее представление:

python
values = np.array([1, 2.5])
print(values.dtype)
print(values)

Обычно целое 1 будет преобразовано в число с плавающей точкой, чтобы оба элемента имели один dtype:

text
float64
[1.  2.5]

Если добавить строку, NumPy может выбрать строковый dtype. Это формально корректный массив, но уже не числовой. Поэтому прикладной контракт часто должен быть строже возможностей самой библиотеки.

Список и массив решают разные задачи

python
python_list = [1, 2, 3]
numpy_array = np.array([1, 2, 3])

print(python_list * 2)
print(numpy_array * 2)
text
[1, 2, 3, 1, 2, 3]
[2 4 6]

Для списка оператор * повторяет контейнер. Для массива он по умолчанию применяется поэлементно. Это не значит, что массив всегда лучше списка. Выбирайте модель по задаче:

Свойство list ndarray
Типы элементов могут различаться представлены одним dtype
Вложенная структура может быть рваной имеет одну прямоугольную shape
Размер контейнера может расти число элементов фиксировано
Численные операторы часто управляют контейнером обычно работают поэлементно

Значения внутри существующего массива менять можно. Но добавление элементов создает новый массив, потому что исходная прямоугольная область памяти не растет как список.

Соберите это

Прежде чем вызывать NumPy, построим упрощенную проверку формы самостоятельно. Она поможет увидеть, откуда берется shape.

Скаляр не добавляет новую ось, поэтому его форма равна пустому кортежу:

python
def infer_shape(value):
    if isinstance(value, (int, float)) and not isinstance(value, bool):
        return ()

Последовательность добавляет ось длиной len(value):

python
def infer_shape(value):
    if isinstance(value, list):
        if not value:
            return (0,)

        child_shapes = [infer_shape(item) for item in value]
        expected = child_shapes[0]

        if any(shape != expected for shape in child_shapes[1:]):
            raise ValueError("ragged nested sequence")

        return (len(value), *expected)

    if isinstance(value, (int, float)) and not isinstance(value, bool):
        return ()

    raise TypeError("expected a real number or a nested list")

Проверим несколько случаев:

python
assert infer_shape(7) == ()
assert infer_shape([7, 8, 9]) == (3,)
assert infer_shape([[1, 2, 3], [4, 5, 6]]) == (2, 3)

Для матрицы алгоритм рассуждает так:

  1. Каждый скаляр имеет форму ().
  2. Каждая строка из трех скаляров имеет форму (3,).
  3. Две строки одинаковой формы образуют массив формы (2, 3).

Рваная структура не проходит проверку:

python
infer_shape([[1, 2], [3]])
text
ValueError: ragged nested sequence

Учебный артефакт расширяет этот алгоритм: сообщает путь до проблемного элемента, принимает JSON, проверяет только вещественные числа и сопоставляет ручной результат с array.shape.

Используйте это

Создание из существующих данных

Используйте np.array, когда значения уже находятся в последовательности Python:

python
sales = np.array([[12, 15, 9], [10, 11, 14]])

При необходимости dtype можно задать явно:

python
sales = np.array([[12, 15, 9], [10, 11, 14]], dtype=np.float32)

Явный dtype полезен, когда формат чисел является частью контракта. Подробно выбор типов, точность и переполнение будут разобраны в отдельном уроке.

Заполнение известной формы

Когда важна форма, а значения должны начинаться с нулей или единиц:

python
zeros = np.zeros((2, 3))
ones = np.ones((2, 3))

По умолчанию эти функции обычно создают float64:

python
integer_zeros = np.zeros((2, 3), dtype=np.int64)

Функция np.empty((2, 3)) тоже выделяет массив нужной формы, но не заполняет его предсказуемыми значениями. Она предназначена для случаев, когда вы сразу перезапишете каждый элемент. Для исходных данных урока безопаснее использовать zeros или ones.

Числовая последовательность с шагом

np.arange похож на range, но возвращает массив:

python
even_numbers = np.arange(0, 10, 2)
print(even_numbers)
text
[0 2 4 6 8]

Аргументы означают начало, невключенную правую границу и шаг.

Для нецелого шага точное количество элементов arange может быть неочевидным из-за арифметики с плавающей точкой. Если важнее количество точек, используйте linspace.

Фиксированное количество точек

np.linspace делит интервал на заданное число точек:

python
grid = np.linspace(0, 1, num=5)
print(grid)
text
[0.   0.25 0.5  0.75 1.  ]

Правая граница включена по умолчанию. Здесь контракт читается как «создать ровно пять точек от нуля до единицы».

Ошибка 2. Незаметно превратить числа в строки

Сам NumPy может найти общий строковый dtype:

python

Но инспектор моделирует числовой аналитический контракт и отклоняет строку:

Ошибка 3. Перенести семантику списка на массив

python

Ошибка 4. Считать shape количеством элементов

Для формы (2, 3):

Ошибка 5. Читать значения из np.empty

Проверьте себя

Ответьте после практики. Объяснение появится сразу после выбора.

01

У массива shape == (3, 4). Какое утверждение обязано быть верным?

В shape два измерения, поэтому ndim равен 2; произведение 3 * 4 даёт 12 элементов.

02

Почему [[1, 2], [3]] нельзя считать обычным двумерным числовым массивом?

Рваная вложенная последовательность не задаёт единую длину второй оси и нарушает прямоугольную модель ndarray.

03

Аналитику нужно ровно пять равномерно расположенных точек от 0 до 1 включительно. Какой конструктор выражает намерение точнее?

linspace фиксирует количество точек и обе границы интервала; это точнее требования, чем шаг arange.

from __future__ import annotations

import importlib.util
import json
import subprocess
import sys
import unittest
from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
ARTIFACT = ROOT / "outputs" / "array_inspector.py"
SPEC = importlib.util.spec_from_file_location("array_inspector", ARTIFACT)
if SPEC is None or SPEC.loader is None:
    raise RuntimeError(f"cannot load {ARTIFACT}")
INSPECTOR = importlib.util.module_from_spec(SPEC)
SPEC.loader.exec_module(INSPECTOR)


class ArrayInspectorTest(unittest.TestCase):
    def test_manual_shape_matches_scalar_vector_and_matrix(self) -> None:
        self.assertEqual(INSPECTOR.infer_shape(7), ())
        self.assertEqual(INSPECTOR.infer_shape([7, 8, 9]), (3,))
        self.assertEqual(
            INSPECTOR.infer_shape([[1, 2, 3], [4, 5, 6]]),
            (2, 3),
        )

    def test_ragged_sequence_is_rejected_before_numpy(self) -> None:
        with self.assertRaisesRegex(INSPECTOR.ArrayContractError, "ragged"):
            INSPECTOR.infer_shape([[1, 2], [3]])

    def test_non_numeric_value_is_rejected(self) -> None:
        with self.assertRaisesRegex(INSPECTOR.ArrayContractError, "real number"):
            INSPECTOR.inspect_values([1, "2", 3])

    def test_report_exposes_core_ndarray_attributes(self) -> None:
        report = INSPECTOR.inspect_values([[12, 15, 9], [10, 11, 14]])

        self.assertEqual(report["array_type"], "ndarray")
        self.assertEqual(report["ndim"], 2)
        self.assertEqual(report["shape"], [2, 3])
        self.assertEqual(report["size"], 6)
        self.assertIn(report["dtype_kind"], {"i", "u"})
        self.assertTrue(all(report["invariants"].values()))

    def test_requested_dtype_is_applied(self) -> None:
        report = INSPECTOR.inspect_values([1, 2, 3], "float32")

        self.assertEqual(report["dtype"], "float32")
        self.assertEqual(report["values"], [1.0, 2.0, 3.0])

    def test_markdown_is_a_standalone_cheatsheet(self) -> None:
        markdown = INSPECTOR.render_markdown(INSPECTOR.inspect_values([1, 2, 3]))

        self.assertIn("## Creation recipes", markdown)
        self.assertIn("np.linspace(0, 1, num=5)", markdown)
        self.assertIn("## Python list and ndarray", markdown)
        self.assertIn("`shape`: `(3,)`", markdown)

    def test_cli_prints_json_report(self) -> None:
        result = subprocess.run(
            [
                sys.executable,
                ARTIFACT,
                "--values",

from __future__ import annotations

import argparse
import json
import math
from collections.abc import Sequence
from numbers import Real
from pathlib import Path
from typing import Any

import numpy as np

DEFAULT_VALUES = "[[12, 15, 9], [10, 11, 14]]"
CREATION_RECIPES = (
    {
        "goal": "Convert a Python sequence",
        "code": "np.array([12, 15, 9])",
    },
    {
        "goal": "Fill a known shape with zeros",
        "code": "np.zeros((2, 3))",
    },
    {
        "goal": "Fill a known shape with ones",
        "code": "np.ones((2, 3))",
    },
    {
        "goal": "Create an integer progression",
        "code": "np.arange(0, 10, 2)",
    },
    {
        "goal": "Create a fixed number of interval points",
        "code": "np.linspace(0, 1, num=5)",
    },
)
LIST_COMPARISON = (
    {
        "property": "Element types",
        "python_list": "May be heterogeneous",
        "ndarray": "Represented by one dtype",
    },
    {
        "property": "Nested structure",
        "python_list": "May be ragged",
        "ndarray": "Uses one rectangular shape",
    },
    {
        "property": "Container size",
        "python_list": "Can grow in place",
        "ndarray": "Element count is fixed",
    },
    {
        "property": "Arithmetic",
        "python_list": "Operators often manage the container",
        "ndarray": "Operators act elementwise by default",
    },
)


class ArrayContractError(ValueError):
    """Raised when values cannot form the lesson's numeric array model."""


def is_sequence(value: object) -> bool:
    return isinstance(value, Sequence) and not isinstance(value, (str, bytes, bytearray))


def infer_shape(value: object, path: str = "values") -> tuple[int, ...]:
    """Infer a rectangular shape without using NumPy."""
    if is_sequence(value):
        items = list(value)
        if not items:
            return (0,)
        child_shapes = [infer_shape(item, f"{path}[{index}]") for index, item in enumerate(items)]
        expected = child_shapes[0]
        for index, shape in enumerate(child_shapes[1:], start=1):
            if shape != expected:
                raise ArrayContractError(
                    f"ragged nested sequence at {path}[{index}]: "
                    f"expected child shape {expected}, got {shape}"
                )
        return (len(items), *expected)

    if isinstance(value, Real) and not isinstance(value, bool):
        return ()

    raise ArrayContractError(f"{path} must be a real number or a nested sequence of real numbers")


def parse_json_value(raw: str) -> Any:
    try:
        return json.loads(raw)
    except json.JSONDecodeError as error:
        raise ArrayContractError(
            f"invalid JSON at line {error.lineno}, column {error.colno}"
        ) from error


def inspect_values(values: object, dtype: str | None = None) -> dict[str, Any]:
    expected_shape = infer_shape(values)
    try:
        array = np.array(values, dtype=dtype)
    except (OverflowError, TypeError, ValueError) as error:
        raise ArrayContractError(f"NumPy could not create the requested array: {error}") from error

    actual_shape = tuple(int(length) for length in array.shape)
    if actual_shape != expected_shape:
        raise ArrayContractError(
            f"manual shape {expected_shape} disagrees with NumPy shape {actual_shape}"
        )

    return {
        "source_type": type(values).__name__,
        "array_type": type(array).__name__,
        "dtype_request": dtype,
        "ndim": int(array.ndim),
        "shape": list(actual_shape),
        "size": int(array.size),
        "dtype": str(array.dtype),
        "dtype_kind": array.dtype.kind,
        "values": array.tolist(),
        "invariants": {
            "rectangular": True,
            "homogeneous_dtype": True,
            "ndim_matches_shape": array.ndim == len(array.shape),
            "size_matches_shape": array.size == math.prod(array.shape),
        },
        "creation_recipes": list(CREATION_RECIPES),
        "list_comparison": list(LIST_COMPARISON),
    }


def format_shape(shape: Sequence[int]) -> str:
    if not shape:
        return "()"
    if len(shape) == 1:
        return f"({shape[0]},)"
    return f"({', '.join(str(length) for length in shape)})"


def render_markdown(report: dict[str, Any]) -> str:
    lines = [
        "# NumPy array inspection",
        "",
        f"- Source type: `{report['source_type']}`",
        f"- Array type: `{report['array_type']}`",
        f"- `ndim`: `{report['ndim']}`",
        f"- `shape`: `{format_shape(report['shape'])}`",
        f"- `size`: `{report['size']}`",
        f"- `dtype`: `{report['dtype']}`",
        f"- Values: `{json.dumps(report['values'], ensure_ascii=False)}`",
        "",
        "## Verified invariants",
        "",
    ]
    for name, passed in report["invariants"].items():
        lines.append(f"- [{'x' if passed else ' '}] `{name}`")

    lines.extend(
        [
            "",
            "## Creation recipes",
            "",
            "| Goal | NumPy code |",
            "|---|---|",
        ]
    )
    for recipe in report["creation_recipes"]:
        lines.append(f"| {recipe['goal']} | `{recipe['code']}` |")

    lines.extend(
        [
            "",
            "## Python list and ndarray",
            "",
            "| Property | Python list | ndarray |",
            "|---|---|---|",
        ]
    )
    for row in report["list_comparison"]:
        lines.append(f"| {row['property']} | {row['python_list']} | {row['ndarray']} |")
    return "\n".join(lines) + "\n"


def main() -> int:
    parser = argparse.ArgumentParser(
        description="Inspect numeric JSON values as a rectangular NumPy array"
    )
    parser.add_argument(
        "--values",
        default=DEFAULT_VALUES,
        help="JSON scalar or nested sequence of real numbers",
    )
    parser.add_argument("--dtype", help="Optional NumPy dtype, for example float32")
    parser.add_argument("--format", choices=("markdown", "json"), default="markdown")
    parser.add_argument("--output", type=Path)
    args = parser.parse_args()

    try:
        report = inspect_values(parse_json_value(args.values), args.dtype)
    except ArrayContractError as error:
        parser.exit(2, f"array-inspector: {error}\n")

    text = (
        json.dumps(report, ensure_ascii=False, indent=2) + "\n"
        if args.format == "json"
        else render_markdown(report)
    )
    if args.output:
        args.output.write_text(text, encoding="utf-8")
    else:
        print(text, end="")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
