Проверка перед уроком

Ответьте до чтения: результат поможет заметить исходные предположения.

01

Массивы одинаковой формы складываются поэлементно. Какие значения образуют один результат?

Поэлементная операция сопоставляет значения в одинаковых координатах; broadcasting расширяет именно это правило на разные совместимые формы.

02

Какова форма результата выражения np.array([1, 2, 3]).T?

У одномерного массива только одна ось, поэтому транспонирование не превращает его ни в строку, ни в столбец.

Broadcasting без магии

Broadcasting сопоставляет позиции осей, а не их смысл: сначала докажите совместимость форм, затем отдельно проверьте предметные оси, dtype и размер результата.

Тип: Build Треки: Core Пререквизит: 02-numpy/06-aggregations Время: ~90 минут Результат: по предметным осям и dtype операндов предсказывает результат поэлементной broadcast-операции, явно добавляет единичные оси и проверяет семантику, тип, запись на месте и память.

Цели обучения

  • объяснить поэлементную операцию как сопоставление координат и отличить ее от редукции и матричного умножения;
  • различать скаляр (), вектор (n,), строку (1, n) и столбец (n, 1);
  • вручную вывести общую форму двух или нескольких операндов справа налево;
  • показать, какая координата малого операнда используется для каждого элемента результата;
  • применять np.newaxis, reshape и keepdims для явного расположения параметра на предметной оси;
  • отдельно проверять совместимость форм, смысл осей, допустимость операции для dtype и тип результата;
  • отличать логическое расширение от физической копии и заранее оценивать память результата;
  • распознавать семантически неверные, in-place несовместимые и слишком большие операции.

Проблема

В предыдущем уроке мы рассчитали среднее и стандартное отклонение каждого показателя в массиве ежедневных метрик:

python
import numpy as np

daily_metrics = np.array(
    [
        [[10.0, 1000.0], [12.0, 1440.0], [8.0, 880.0]],
        [[7.0, 840.0], [9.0, 990.0], [11.0, 1320.0]],
    ]
)

assert daily_metrics.shape == (2, 3, 2)

Контракт осей:

text
(store=2, day=3, metric=2)

Агрегаты сохраняют позиции свернутых осей:

python
metric_means = daily_metrics.mean(axis=(0, 1), keepdims=True)
metric_stds = daily_metrics.std(axis=(0, 1), keepdims=True)

assert metric_means.shape == (1, 1, 2)
assert metric_stds.shape == (1, 1, 2)

Их контракт:

text
(summarized_store=1, summarized_day=1, metric=2)

Теперь нужно применить два параметра каждого показателя ко всем магазинам и дням:

python
standardized = (daily_metrics - metric_means) / metric_stds

NumPy принимает эту запись, хотя формы различаются. Он не создает руками шесть копий каждого среднего. Вместо этого NumPy сопоставляет оси и для каждой координаты результата решает, какой элемент каждого операнда использовать.

Но у автоматизма есть границы:

  • совместимые длины могут относиться к разным предметным осям;
  • одномерный вектор не сообщает, является он строкой или столбцом;
  • одна и та же форма может дать разный dtype для сложения, деления и сравнения;
  • маленький операнд не требует копии, но результат операции может оказаться огромным;
  • операция вне выражения может работать, а запись результата обратно через += — нет.

Поэтому broadcasting — не трюк для устранения циклов, а контракт из нескольких независимых вопросов.

Концепция

Сначала поэлементная операция одинаковых форм

Рассмотрим два массива одной формы:

python
left = np.array([[1, 2], [3, 4]])
right = np.array([[10, 20], [30, 40]])

result = left + right

Каждая координата результата использует ту же координату обоих операндов:

text
result[0, 0] = left[0, 0] + right[0, 0]
result[0, 1] = left[0, 1] + right[0, 1]
result[1, 0] = left[1, 0] + right[1, 0]
result[1, 1] = left[1, 1] + right[1, 1]
python
np.testing.assert_array_equal(result, [[11, 22], [33, 44]])

Так работают +, -, *, /, сравнения и многие функции NumPy. Они применяют одну маленькую операцию к соответствующим элементам всего массива.

Что такое ufunc на нужном нам уровне

np.add, np.subtract, np.multiply, np.true_divide, np.maximum и np.equal — универсальные функции, или ufunc. Для этого урока достаточно трех свойств ufunc:

  1. функция принимает скалярные элементы и применяет один механизм ко всему массиву;
  2. перед вычислением она согласует формы операндов по правилам broadcasting;
  3. по dtype входов она выбирает подходящую реализацию и dtype выхода.

Оператор обычно является удобной записью ufunc:

python
np.testing.assert_array_equal(left + right, np.add(left, right))

Broadcasting не является правилом любой функции NumPy. Например:

  • * вызывает поэлементное умножение и использует broadcasting;
  • @ выполняет матричное умножение и имеет отдельный контракт внутренних осей;
  • sum(axis=...) является редукцией и удаляет оси;
  • reshape меняет представление осей, но не выполняет поэлементную арифметику.

Скаляр — первый случай broadcasting

Нульмерный массив имеет форму ():

python
scalar = np.array(10)
matrix = np.array([[1, 2], [3, 4]])

assert scalar.shape == ()
assert (matrix + scalar).shape == (2, 2)
np.testing.assert_array_equal(matrix + scalar, [[11, 12], [13, 14]])

Для каждой координаты матрицы используется единственное значение скаляра. Обычное Python-число ведет себя в такой операции сходным образом:

python
np.testing.assert_array_equal(matrix * 2, [[2, 4], [6, 8]])

Но Python-скаляр и нульмерный NumPy-массив могут по-разному участвовать в продвижении типов. Эту деталь мы не будем запоминать по таблице: для критичной операции проверим np.result_type и фактический dtype результата.

Вектор не является ни строкой, ни столбцом

Одномерный массив содержит одну ось:

python
vector = np.array([10, 20, 30])

assert vector.shape == (3,)
assert vector.ndim == 1

В его форме нет отдельного признака «горизонтальный» или «вертикальный». Поэтому .T не добавляет ось:

python
assert vector.T.shape == (3,)

Строку и столбец нужно создать явно:

python
row = vector[np.newaxis, :]
column = vector[:, np.newaxis]

assert row.shape == (1, 3)
assert column.shape == (3, 1)

Их broadcasting различается:

text
row:       (1, 3)
column:    (3, 1)
result:    (3, 3)
python
outer_sum = row + column

np.testing.assert_array_equal(
    outer_sum,
    [
        [20, 30, 40],
        [30, 40, 50],
        [40, 50, 60],
    ],
)

Это уже не применение одного вектора к другому. Каждая строка и каждый столбец образуют все попарные сочетания.

Формы выравниваются справа налево

Перед сравнением NumPy мысленно дополняет более короткую форму единицами слева:

text
values:     (2, 3, 2)
parameter:        (2,)
aligned:    (1, 1, 2)

Затем длины сравниваются справа налево. Две выровненные длины совместимы, если:

  1. они равны;
  2. левая равна 1;
  3. правая равна 1.

Результирующая длина определяется не общей формулой max. Правильный алгоритм:

text
если a == b: результат a
иначе если a == 1: результат b
иначе если b == 1: результат a
иначе: формы несовместимы

Различие важно для пустой оси:

text
(0,) и (1,) -> (0,)

Единичная ось расширяется до длины другой оси, даже если та равна нулю. Выражение max(0, 1) ошибочно предсказало бы (1,).

Как получить итоговую форму

Пример с матрицей и вектором:

text
matrix: (100, 3)
vector:      (3,)
aligned: (1, 3)
result: (100, 3)

Последние длины равны 3. Отсутствующая первая ось вектора считается единичной и расширяется до 100.

Пример двух единичных осей:

text
left:   (5, 1)
right:  (1, 6)
result: (5, 6)

Пример конфликта:

text
matrix: (100, 3)
vector:    (100,)
aligned: (1, 100)

Последняя пара 3 и 100 несовместима. NumPy не пытается догадаться, что 100 возможно относится к строкам.

Координата единичной оси всегда равна нулю

Пусть выполняется операция:

text
values shape:    (2, 3)
parameter shape: (1, 3)
result shape:    (2, 3)

Для результата result[i, j]:

text
result[i, j] = values[i, j] operation parameter[0, j]

Первая ось parameter имеет длину один, поэтому независимо от i используется координата 0. Именно это означает логическое расширение единичной оси.

Если ось отсутствовала, эффект тот же. Вектор (3,) мысленно становится (1, 3).

NumPy сопоставляет позиции, а не предметные имена

Вернемся к форме:

text
daily_metrics: (store=2, day=3, metric=2)
parameter:                              (2,)

NumPy применит parameter к последней оси metric. Он не хранит наши имена осей.

Теперь предположим, что два числа параметра относятся не к метрикам, а к магазинам:

text
parameter: (store=2,)

Форма по-прежнему совместима, потому что длина последней оси metric случайно тоже равна двум. Операция выполнится без ошибки и применит магазинные параметры к метрикам.

Чтобы применить параметр магазина ко всем дням и метрикам, расположите его на оси store явно:

python
store_adjustment = np.array([0.9, 1.1])[:, np.newaxis, np.newaxis]

assert store_adjustment.shape == (2, 1, 1)
adjusted = daily_metrics * store_adjustment
assert adjusted.shape == daily_metrics.shape

Предметные имена осей являются нашим внешним контрактом. NumPy проверяет только позиции и длины.

keepdims сохраняет правильные позиции

Предыдущий урок подготовил:

python
metric_means = daily_metrics.mean(axis=(0, 1), keepdims=True)
assert metric_means.shape == (1, 1, 2)

Форма уже выровнена с (store, day, metric). Обе свернутые оси имеют длину один и расширяются, а metric совпадает:

text
values: (2, 3, 2)
means:  (1, 1, 2)
result: (2, 3, 2)

Без keepdims получится (2,). Здесь это тоже работает, потому что metric является последней осью. Но форма (1, 1, 2) лучше сохраняет происхождение параметра и не зависит от мысленного добавления двух осей.

Broadcasting и dtype — независимые контракты

Совместимые формы отвечают только на вопрос, какие элементы можно сопоставить. После этого ufunc должен выбрать реализацию для типов входов.

Для каждой операции задайте три вопроса:

  1. Определена ли операция для этой пары dtype?
  2. Какой dtype получится?
  3. Безопасны ли диапазон значений и запись результата обратно?

Несколько характерных случаев в NumPy 2.x:

Входы и операция Типичный результат Что проверить
integer + integer integer после продвижения переполнение остается возможным
int32 + float32 float64 точность и память результата
integer / integer float64 деление не остается целочисленным
любое числовое сравнение bool форма маски совпадает с broadcast-формой
bool + bool bool для логики яснее &, `
datetime64 - datetime64 timedelta64 единицы времени результата
datetime64 + timedelta64 datetime64 согласованность временных единиц
строки - число операция не определена не все dtype поддерживают каждую ufunc
object зависит от Python-объектов производительность и поведение не гарантируются NumPy-типом

Проверим несколько результатов:

python
integers = np.array([1, 2], dtype=np.int32)
floats = np.array([0.5, 1.5], dtype=np.float32)

assert (integers + floats).dtype == np.dtype("float64")
assert (integers / integers).dtype == np.dtype("float64")
assert (integers < 2).dtype == np.dtype("bool")

np.result_type предсказывает общий тип входных значений:

python
assert np.result_type(integers.dtype, floats.dtype) == np.dtype("float64")

Но это не всегда тип конкретной операции. Для двух int32 общий тип остается int32, а np.true_divide возвращает float64. Сравнение всегда возвращает bool, а вычитание дат — интервал. Поэтому проверяем и общий тип, и фактический контракт ufunc.

Булевы значения: арифметика возможна, смысл не следует автоматически

В NumPy арифметика двух булевых массивов может быть определена:

python
left_mask = np.array([True, False])
right_mask = np.array([True, True])

assert (left_mask + right_mask).dtype == np.dtype("bool")

Но + плохо выражает намерение объединить условия или посчитать истины:

python
union = left_mask | right_mask
both = left_mask & right_mask
true_count = left_mask.sum()

Операция, разрешенная библиотекой, не обязательно является лучшим предметным контрактом.

In-place операция предъявляет дополнительные требования

Обычное выражение создает результат новой формы и типа:

python
integers = np.array([[1, 2], [3, 4]], dtype=np.int32)
result = integers + 0.5

assert result.dtype == np.dtype("float64")

Но integers += 0.5 пытается записать вещественный результат обратно в int32 и обычно завершается ошибкой приведения. Для in-place операции нужны одновременно:

  • итоговая broadcast-форма, которая точно помещается в форму левого операнда;
  • допустимое приведение выходного dtype обратно в dtype левого операнда.

Еще один пример:

python
left = np.zeros((3, 1))
right = np.ones((3,))

assert (left + right).shape == (3, 3)

left += right не может сохранить (3, 3) в буфере формы (3, 1), хотя обычное сложение формально допустимо.

Логическое расширение не равно физическому копированию

Вектор (metric=2,) логически используется шесть раз при операции с (store=2, day=3, metric=2), но NumPy не обязан создавать шесть физических копий до операции.

Это можно увидеть через np.broadcast_to:

python
parameter = np.array([10.0, 1000.0])
expanded = np.broadcast_to(parameter, (2, 3, 2))

assert expanded.shape == (2, 3, 2)
assert np.shares_memory(parameter, expanded)
assert not expanded.flags.writeable

Broadcast-представление обычно является read-only view. Несколько логических координат могут ссылаться на один исходный элемент, поэтому запись через такое представление была бы неоднозначной.

np.tile и np.repeat физически строят повторения и нужны только тогда, когда отдельные значения действительно должны существовать в памяти. Для обычной поэлементной операции они чаще скрывают контракт и расходуют лишнюю память.

Результат операции все равно может быть большим

Broadcasting экономит память малого операнда, но обычная арифметика создает итоговый массив:

text
left:   (50_000,      1, 100)
right:  (     1, 50_000, 100)
result: (50_000, 50_000, 100)

Это 250 000 000 000 элементов. При float64 результат потребует примерно два терабайта. Формы совместимы, но вычисление неприемлемо.

До запуска оцените:

text
result_elements = product(result_shape)
result_bytes = result_elements * result_dtype.itemsize

Для оценки нужен именно выходной dtype, а не тип одного из входов.

Соберите это

Шаг 1. Воспроизведите одинаковые формы

Сначала реализуем поэлементное сложение без NumPy и без broadcasting:

python
def add_same_shape(left, right):
    if len(left) != len(right):
        raise ValueError("row counts differ")
    result = []
    for left_row, right_row in zip(left, right):
        if len(left_row) != len(right_row):
            raise ValueError("column counts differ")
        result.append(
            [left_value + right_value for left_value, right_value in zip(left_row, right_row)]
        )
    return result


assert add_same_shape([[1, 2]], [[10, 20]]) == [[11, 22]]

Здесь каждая координата обоих операндов существует явно.

Шаг 2. Объедините две выровненные длины

python
def merge_lengths(left, right):
    if left == right:
        return left
    if left == 1:
        return right
    if right == 1:
        return left
    raise ValueError("incompatible lengths")
python
assert merge_lengths(3, 3) == 3
assert merge_lengths(1, 5) == 5
assert merge_lengths(5, 1) == 5
assert merge_lengths(0, 1) == 0

Последняя проверка показывает, почему нельзя использовать max(left, right).

Шаг 3. Предскажите общую форму

python
def broadcast_shape(left, right):
    width = max(len(left), len(right))
    aligned_left = (1,) * (width - len(left)) + left
    aligned_right = (1,) * (width - len(right)) + right

    return tuple(
        merge_lengths(left_length, right_length)
        for left_length, right_length in zip(aligned_left, aligned_right)
    )
python
assert broadcast_shape((100, 3), (3,)) == (100, 3)
assert broadcast_shape((5, 1), (1, 6)) == (5, 6)
assert broadcast_shape((0,), (1,)) == (0,)

Готовый артефакт обобщает механизм на несколько форм и сообщает, на какой паре длин возник конфликт.

Шаг 4. Постройте таблицу выравнивания

Для каждого операнда запишите:

  1. исходную форму;
  2. добавленные слева единицы;
  3. предметное имя каждой выровненной оси;
  4. оси, которые логически расширятся;
  5. итоговую длину.

Пример:

text
output axis       store        day       metric
values length       2           3           2
means length        1           1           2
result length       2           3           2
means expands       yes         yes         no

Если в одной позиции оказываются store=2 и metric=2, длины совместимы, но таблица должна выдать семантическое предупреждение.

Шаг 5. Разделите общий тип и тип операции

python
left_dtype = np.dtype("int32")
right_dtype = np.dtype("int32")

common_dtype = np.result_type(left_dtype, right_dtype)
operation_dtype = np.true_divide(
    np.ones((), dtype=left_dtype),
    np.ones((), dtype=right_dtype),
).dtype

assert common_dtype == np.dtype("int32")
assert operation_dtype == np.dtype("float64")

Нульмерные образцы позволяют спросить ufunc о типе выхода, не создавая большой broadcast-результат. Для реальных значений дополнительно нужны проверки диапазона, деления на ноль и конечности — они появятся в уроке о численной точности.

Используйте это

Примените агрегаты прошлого урока

python
metric_means = daily_metrics.mean(axis=(0, 1), keepdims=True)
metric_stds = daily_metrics.std(axis=(0, 1), keepdims=True)

if np.any(metric_stds == 0):
    raise ValueError("constant metric cannot be standardized")

standardized = (daily_metrics - metric_means) / metric_stds

assert standardized.shape == daily_metrics.shape
np.testing.assert_allclose(
    standardized.mean(axis=(0, 1)),
    [0.0, 0.0],
    atol=1e-12,
)
np.testing.assert_allclose(
    standardized.std(axis=(0, 1)),
    [1.0, 1.0],
    atol=1e-12,
)

Broadcasting здесь решает только применение параметров. Методология среднего, стандартного отклонения, пропусков и ddof остается контрактом предыдущего урока.

Примените параметры выбранной предметной оси

Параметр каждой метрики может оставаться вектором, потому что metric — последняя ось:

python
metric_weight = np.array([1.0, 0.001])
weighted = daily_metrics * metric_weight

assert metric_weight.shape == (2,)
assert weighted.shape == (2, 3, 2)

Параметр магазина нужно поставить на первую ось:

python
store_weight = np.array([0.9, 1.1])[:, np.newaxis, np.newaxis]

assert store_weight.shape == (2, 1, 1)
assert (daily_metrics * store_weight).shape == daily_metrics.shape

Параметр дня ставится на среднюю ось:

python
day_weight = np.array([1.0, 0.8, 1.2])[np.newaxis, :, np.newaxis]

assert day_weight.shape == (1, 3, 1)
assert (daily_metrics * day_weight).shape == daily_metrics.shape

Позиция единичных осей выражает, к какому измерению относится параметр.

Постройте условие с порогом каждой метрики

np.where согласует форму условия и двух вариантов результата:

python
thresholds = np.array([9.0, 1000.0])
above_threshold = daily_metrics >= thresholds

assert above_threshold.shape == daily_metrics.shape
assert above_threshold.dtype == np.dtype("bool")

flagged = np.where(above_threshold, daily_metrics, np.nan)
assert flagged.shape == daily_metrics.shape
assert flagged.dtype == np.dtype("float64")

Здесь broadcasting используется дважды: при сравнении с порогами и при выборе между числовым массивом и скаляром NaN.

Постройте попарные разности осознанно

python
left = np.array([10.0, 20.0, 30.0])[:, np.newaxis]
right = np.array([8.0, 25.0])[np.newaxis, :]

differences = left - right

assert left.shape == (3, 1)
assert right.shape == (1, 2)
assert differences.shape == (3, 2)

Каждый элемент результата представляет одну пару (left_item, right_item). Такой прием полезен для расстояний и сравнений, но размер результата растет как произведение длин.

Используйте официальные предикторы

python
assert np.broadcast_shapes((2, 3, 2), (2,)) == (2, 3, 2)
assert np.broadcast_shapes((5, 1), (1, 6), (6,)) == (5, 6)

np.broadcast_shapes не создает результат и подходит для ранней проверки памяти.

np.broadcast_arrays возвращает согласованные представления нескольких массивов, а np.broadcast_to — представление одного массива в заданной форме. Они полезны для диагностики, но в обычной формуле явное построение представлений чаще не требуется.

Сломайте это

Несовместимые последние оси

python
np.zeros((4, 3)) + np.zeros(4)

После выравнивания последние длины 3 и 4 конфликтуют. Намерение применить параметр к строкам не меняет позиционных правил.

Совместимые длины с разным смыслом

text
values:    (store=2, day=3, metric=2)
parameter:                         (store=2,)

NumPy видит совпадающие двойки и выполняет операцию по metric. Только реестр предметных осей обнаруживает, что параметр магазина попал не на ту позицию.

Попытаться транспонировать одномерный вектор

python
parameter = np.array([0.9, 1.1])
wrong = parameter.T

assert wrong.shape == (2,)

Для оси store трехмерного массива нужна форма (2, 1, 1), а не .T.

Добавить единичную ось не в ту позицию

python
store_weight = np.array([0.9, 1.1])

wrong = store_weight[np.newaxis, np.newaxis, :]  # (1, 1, 2): metric
right = store_weight[:, np.newaxis, np.newaxis]  # (2, 1, 1): store

Обе формы совместимы с (2, 3, 2), но применяют значения к разным осям.

Принять np.result_type за тип любой операции

python

result = np.array([1], dtype=np.int32) / np.array([2], dtype=np.int32)
assert result.dtype == np.dtype("float64")

Продвижение входов и выход конкретной ufunc — связанные, но не идентичные контракты.

Использовать арифметику вместо булевой логики

mask_a + mask_b для bool может выполниться, но не выражает однозначно ни логическое «ИЛИ», ни количество истинных условий. Используйте |, &, np.logical_or или явную редукцию маски.

Ожидать, что любая операция поддерживает любой dtype

python
labels = np.array(["A", "B"])
labels - 1  # операция не определена

Broadcast-совместимость формы (2,) со скаляром не создает реализацию вычитания строк и чисел.

Записать расширенный результат на месте

python
left = np.zeros((3, 1))
right = np.ones(3)

assert (left + right).shape == (3, 3)
left += right  # результат формы (3, 3) не помещается в (3, 1)

In-place операция не может изменить форму буфера левого операнда.

Записать float обратно в integer

python
integers = np.array([1, 2, 3], dtype=np.int32)
integers += 0.5  # небезопасное приведение float64 -> int32

Обычная операция создала бы новый вещественный массив. In-place контракт строже.

Переполнить integer после успешного согласования

python
values = np.array([120], dtype=np.int8)
increment = np.array([20], dtype=np.int8)
result = values + increment

Формы и типы согласованы, но диапазона int8 недостаточно. Broadcasting не проверяет значения. Подробные проверки overflow и tolerances будут в уроке о численной точности.

Создать огромный попарный результат

Малые операнды не копируются заранее, но результат float64 требует около двух терабайт. Проверяйте форму, количество элементов и dtype до арифметики.

Использовать tile для сокрытия непонятной формы

Физическое повторение может заставить формы совпасть, но не доказывает правильность осей. Сначала назовите назначение параметра и вставьте единичные оси. tile оставьте случаям, где отдельная физическая копия действительно является требуемым результатом.

Ключевые термины

Термин Распространенное заблуждение Точное значение
Поэлементная операция «Любая функция над массивом» Один скалярный механизм, применяемый к сопоставленным координатам
Ufunc «Только быстрая математическая функция» NumPy-функция с правилами типов, broadcasting и поэлементного выполнения
Broadcasting «Физическое копирование малого массива» Позиционные правила логического согласования форм операндов
Trailing axis «Первая ось массива» Ось в правой части shape, с которой начинается сравнение
Singleton axis «Пустая ось» Ось длины 1, единственная координата которой может использоваться повторно
Неявная ведущая ось «Новая ось в исходном массиве» Мысленная единичная ось, добавленная слева только для выравнивания форм
Семантическая совместимость «Совпадение длин» Соответствие выровненных позиций одним и тем же предметным осям
Продвижение типов «Broadcasting меняет dtype» Отдельные правила выбора общего типа входов для операции
In-place операция «То же выражение без создания переменной» Запись результата обратно в существующие форму и dtype левого операнда
Логическая форма «Объем уже выделенной памяти» Форма доступных координат, которая может быть представлением без копии

Quiz · 8 вопроса

Проверьте себя

Ответьте после практики. Объяснение появится сразу после выбора.

01

Какова общая форма операндов (2, 3, 4) и (4,)?

Формы выравниваются справа: (4,) мысленно становится (1, 1, 4) и расширяется по первым двум осям.

02

Нужно вычесть вектор формы (3,) из каждого столбца матрицы формы (3, 5). Как подготовить вектор?

Для сопоставления с первой осью нужна явная форма (3, 1). У одномерного vector выражение vector.T по-прежнему имеет форму (3,).

03

Массив имеет оси (store, day, metric) и форму (2, 3, 2), а параметр store имеет форму (2,). Что произойдет при прямом умножении?

Broadcasting не знает предметных имен осей и выравнивает позиции справа. Совпавшие длины не доказывают правильность смысла.

04

Какова broadcast-форма массивов (0,) и (1,)?

Ось длины 1 расширяется до длины другой оси, в том числе до 0. Поэтому правило нельзя заменять безусловным max длин.

05

Оба операнда имеют dtype int32. Что верно для np.true_divide(left, right)?

np.result_type описывает общий тип входов, но конкретный ufunc может иметь собственное правило выходного типа. Для true_divide целых это float64.

06

Какой dtype имеет результат сравнения float32-массива с float32-порогом?

Сравнение возвращает маску истинности с dtype bool независимо от вещественного dtype сравниваемых значений.

07

Почему values += offsets может не сработать, хотя values + offsets работает?

Обычное выражение может создать массив новой формы и типа, а in-place запись ограничена существующим хранилищем левого операнда.

08

Формы (50000, 1, 100) и (1, 50000, 100) совместимы. Что нужно проверить до вычитания?

Малые входы могут породить огромный материальный результат. Для float64 такая форма потребует около 2 ТБ памяти.

import importlib.util
import json

from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
ARTIFACT = ROOT / "outputs" / "broadcast_contract.py"
SPEC = importlib.util.spec_from_file_location("broadcast_contract", ARTIFACT)
if SPEC is None or SPEC.loader is None:
    raise RuntimeError(f"cannot load {ARTIFACT}")
BROADCAST = importlib.util.module_from_spec(SPEC)
SPEC.loader.exec_module(BROADCAST)


def operand(
    name: str,
    shape: list[int],
    axis_names: list[str],
    dtype: str = "float64",
) -> dict[str, object]:
    return {
        "name": name,
        "shape": shape,
        "axis_names": axis_names,
        "dtype": dtype,
    }


class BroadcastContractTest(unittest.TestCase):
    def test_manual_predictor_matches_numpy_for_core_cases(self) -> None:
        cases = [
            ((3, 4), (3, 4)),
            ((3, 4), ()),
            ((3, 4), (4,)),
            ((5, 1), (1, 6)),
            ((8, 1, 6, 1), (7, 1, 5)),
        ]
        for left, right in cases:
            with self.subTest(left=left, right=right):
                self.assertEqual(
                    BROADCAST.broadcast_shape(left, right),
                    np.broadcast_shapes(left, right),
                )

    def test_predictor_supports_more_than_two_shapes(self) -> None:
        shapes = ((2, 3, 4), (3, 1), (4,))
        self.assertEqual(
            BROADCAST.broadcast_shape(*shapes),
            np.broadcast_shapes(*shapes),
        )

    def test_zero_length_axis_follows_numpy_instead_of_max_rule(self) -> None:
        self.assertEqual(BROADCAST.broadcast_shape((0,), (1,)), (0,))
        self.assertEqual(
            BROADCAST.broadcast_shape((2, 0, 3), (1, 3)),
            np.broadcast_shapes((2, 0, 3), (1, 3)),
        )

    def test_incompatible_shapes_are_rejected(self) -> None:
        with self.assertRaisesRegex(BROADCAST.BroadcastingError, "incompatible"):
            BROADCAST.broadcast_shape((4, 3), (4,))

    def test_one_dimensional_transpose_does_not_create_column(self) -> None:
        vector = np.array([1, 2, 3])
        self.assertEqual(vector.T.shape, (3,))
        self.assertEqual(vector[np.newaxis, :].shape, (1, 3))
        self.assertEqual(vector[:, np.newaxis].shape, (3, 1))

    def test_named_alignment_explains_implicit_and_singleton_axes(self) -> None:
        report = BROADCAST.analyze_broadcast(
            [
                operand("values", [2, 3, 2], ["store", "day", "metric"]),
                operand("center", [2], ["metric"]),
            ]
        )
        self.assertEqual(report["result"]["shape"], [2, 3, 2])
        self.assertEqual(report["result"]["axis_names"], ["store", "day", "metric"])
        self.assertTrue(report["alignment"][0]["operands"][1]["implicit_leading_axis"])
        self.assertTrue(report["alignment"][1]["operands"][1]["expands"])
        self.assertEqual(report["operands"][1]["logical_expansion_factor"], 6)

    def test_keepdims_names_match_the_original_semantic_axes(self) -> None:

                operand("values", [2, 3, 2], ["store", "day", "metric"]),
                operand(
                    "center",
                    [1, 1, 2],

                ),
            ]
        )

        )

    def test_equal_lengths_with_different_meanings_produce_warning(self) -> None:

from __future__ import annotations

import argparse
import json
import math
from collections.abc import Sequence
from numbers import Integral, Real
from pathlib import Path
from typing import Any

import numpy as np


class BroadcastingError(ValueError):
    """Raised when a broadcast contract cannot be constructed."""


OPERATIONS: dict[str, np.ufunc] = {
    "add": np.add,
    "subtract": np.subtract,
    "multiply": np.multiply,
    "true_divide": np.true_divide,
    "maximum": np.maximum,
    "equal": np.equal,
    "less": np.less,
}

COMPARISON_OPERATIONS = {"equal", "less"}
ARITHMETIC_OPERATIONS = {"add", "subtract", "multiply", "true_divide"}


def validate_shape(shape: Sequence[object]) -> tuple[int, ...]:
    normalized: list[int] = []
    for length in shape:
        if isinstance(length, bool) or not isinstance(length, Integral):
            raise BroadcastingError("shape lengths must be integers")
        number = int(length)
        if number < 0:
            raise BroadcastingError("shape lengths cannot be negative")
        normalized.append(number)
    return tuple(normalized)


def merge_lengths(left: int, right: int) -> int:
    """Merge two aligned lengths, including the zero-versus-one case."""
    if left == right:
        return left
    if left == 1:
        return right
    if right == 1:
        return left
    raise BroadcastingError(f"aligned lengths {left} and {right} are incompatible")


def broadcast_shape(*shapes: Sequence[object]) -> tuple[int, ...]:
    """Predict NumPy's common shape without allocating the output."""
    result: tuple[int, ...] = ()
    for raw_shape in shapes:
        shape = validate_shape(raw_shape)
        width = max(len(result), len(shape))
        left = (1,) * (width - len(result)) + result
        right = (1,) * (width - len(shape)) + shape
        try:
            result = tuple(
                merge_lengths(left_length, right_length)
                for left_length, right_length in zip(left, right, strict=True)
            )
        except BroadcastingError as error:
            raise BroadcastingError(
                f"shapes {result} and {shape} are incompatible: {error}"
            ) from error
    return result


def validate_axis_names(
    axis_names: object,
    *,
    ndim: int,
    operand_name: str,
) -> tuple[str, ...]:
    if axis_names is None:
        return tuple(f"axis_{index}" for index in range(ndim))
    if isinstance(axis_names, (str, bytes)) or not isinstance(axis_names, Sequence):
        raise BroadcastingError(f"{operand_name} axis_names must be a sequence")
    names = tuple(axis_names)
    if len(names) != ndim:
        raise BroadcastingError(
            f"{operand_name} has ndim={ndim}, but {len(names)} axis names were supplied"
        )
    if any(not isinstance(name, str) or not name.strip() for name in names):
        raise BroadcastingError(f"{operand_name} axis names must be non-empty strings")
    if len(set(names)) != len(names):
        raise BroadcastingError(f"{operand_name} axis names must be unique")
    return names


def normalize_operand(raw: object, index: int) -> dict[str, Any]:
    if not isinstance(raw, dict):
        raise BroadcastingError("each operand must be a JSON object")
    name = raw.get("name", f"operand_{index}")
    if not isinstance(name, str) or not name.strip():
        raise BroadcastingError("operand name must be a non-empty string")
    if "shape" not in raw:
        raise BroadcastingError(f"{name} must declare shape")
    shape_raw = raw["shape"]
    if isinstance(shape_raw, (str, bytes)) or not isinstance(shape_raw, Sequence):
        raise BroadcastingError(f"{name} shape must be a sequence")
    shape = validate_shape(shape_raw)
    axis_names = validate_axis_names(
        raw.get("axis_names"),
        ndim=len(shape),
        operand_name=name,
    )
    try:
        dtype = np.dtype(raw.get("dtype", "float64"))
    except TypeError as error:
        raise BroadcastingError(f"{name} has invalid dtype {raw.get('dtype')!r}") from error
    return {
        "name": name,
        "shape": shape,
        "axis_names": axis_names,
        "dtype": dtype,
    }


def semantic_axis_name(name: str) -> str:
    prefix = "summarized_"
    return name[len(prefix) :] if name.startswith(prefix) else name


def build_alignment(
    operands: Sequence[dict[str, Any]],
    output_shape: tuple[int, ...],
) -> tuple[list[dict[str, Any]], list[dict[str, Any]]]:
    width = len(output_shape)
    aligned_operands: list[dict[str, Any]] = []
    for operand in operands:
        padding = width - len(operand["shape"])
        aligned_operands.append(
            {
                **operand,
                "aligned_shape": (1,) * padding + operand["shape"],
                "aligned_axis_names": (None,) * padding + operand["axis_names"],
                "padding": padding,
            }
        )

    rows: list[dict[str, Any]] = []
    warnings: list[dict[str, Any]] = []
    for output_axis, output_length in enumerate(output_shape):
        cells: list[dict[str, Any]] = []
        semantic_names: set[str] = set()
        preferred_names: list[str] = []
        for operand in aligned_operands:
            length = operand["aligned_shape"][output_axis]
            axis_name = operand["aligned_axis_names"][output_axis]
            implicit = output_axis < operand["padding"]
            input_axis = None if implicit else output_axis - operand["padding"]
            if axis_name is not None:
                semantic_names.add(semantic_axis_name(axis_name))
                if length == output_length and length != 1:
                    preferred_names.append(semantic_axis_name(axis_name))
            cells.append(
                {
                    "operand": operand["name"],
                    "input_axis": input_axis,
                    "axis_name": axis_name,
                    "length": length,
                    "implicit_leading_axis": implicit,
                    "expands": length == 1 and output_length != 1,
                }
            )

        if len(semantic_names) > 1:
            warnings.append(
                {
                    "code": "axis_name_conflict",
                    "output_axis": output_axis,
                    "message": (
                        f"aligned output axis {output_axis} combines semantic axes "
                        f"{sorted(semantic_names)}"
                    ),
                }
            )

        if preferred_names:
            output_name = preferred_names[0]
        elif len(semantic_names) == 1:
            output_name = next(iter(semantic_names))
        else:
            output_name = f"axis_{output_axis}"
        rows.append(
            {
                "output_axis": output_axis,
                "position_from_right": output_axis - width,
                "output_axis_name": output_name,
                "output_length": output_length,
                "operands": cells,
            }
        )
    return rows, warnings


def resolve_operation_dtype(
    left_dtype: np.dtype[Any],
    right_dtype: np.dtype[Any],
    operation: str,
) -> np.dtype[Any]:
    ufunc = OPERATIONS[operation]
    left = np.zeros((), dtype=left_dtype)
    right = np.zeros((), dtype=right_dtype)
    try:
        with np.errstate(all="ignore"):
            result = ufunc(left, right)
    except (TypeError, ValueError) as error:
        raise BroadcastingError(
            f"operation {operation!r} is not defined for {left_dtype} and {right_dtype}"
        ) from error
    return result.dtype


def expansion_factor(shape: tuple[int, ...], output_shape: tuple[int, ...]) -> int | None:
    source_size = math.prod(shape)
    output_size = math.prod(output_shape)
    if source_size == 0:
        return None
    return output_size // source_size


def analyze_broadcast(
    raw_operands: Sequence[object],
    *,
    operation: str = "subtract",
    memory_limit_mb: float = 256.0,
) -> dict[str, Any]:
    if len(raw_operands) != 2:
        raise BroadcastingError("operation contract expects exactly two operands")
    if operation not in OPERATIONS:
        raise BroadcastingError(f"unknown operation {operation!r}")
    if (
        isinstance(memory_limit_mb, bool)
        or not isinstance(memory_limit_mb, Real)
        or not math.isfinite(float(memory_limit_mb))
        or memory_limit_mb <= 0
    ):
        raise BroadcastingError("memory_limit_mb must be a positive finite number")

    operands = [normalize_operand(raw, index) for index, raw in enumerate(raw_operands)]
    output_shape = broadcast_shape(*(operand["shape"] for operand in operands))
    alignment, warnings = build_alignment(operands, output_shape)

    left_dtype = operands[0]["dtype"]
    right_dtype = operands[1]["dtype"]
    common_dtype = np.result_type(left_dtype, right_dtype)
    operation_error: str | None = None
    try:
        result_dtype = resolve_operation_dtype(left_dtype, right_dtype, operation)
    except BroadcastingError as error:
        result_dtype = None
        operation_error = str(error)
        warnings.append(
            {
                "code": "operation_not_defined",
                "message": operation_error,
            }
        )

    output_elements = math.prod(output_shape)
    output_nbytes = None if result_dtype is None else output_elements * result_dtype.itemsize
    memory_limit_bytes = int(float(memory_limit_mb) * 1024 * 1024)
    if output_nbytes is not None and output_nbytes > memory_limit_bytes:
        warnings.append(
            {
                "code": "output_exceeds_memory_limit",
                "message": (
                    f"logical result needs {output_nbytes} bytes, above limit "
                    f"{memory_limit_bytes} bytes"
                ),
            }
        )
    if (
        result_dtype is not None
        and operation in ARITHMETIC_OPERATIONS
        and np.issubdtype(result_dtype, np.integer)
    ):
        warnings.append(
            {
                "code": "integer_range_not_checked",
                "message": "integer result dtype does not guarantee that values cannot overflow",
            }
        )
    if operation in ARITHMETIC_OPERATIONS and all(
        np.issubdtype(operand["dtype"], np.bool_) for operand in operands
    ):
        warnings.append(
            {
                "code": "boolean_arithmetic",
                "message": (
                    "boolean arithmetic is defined, but logical operators express intent better"
                ),
            }
        )

    left_shape_can_store = operands[0]["shape"] == output_shape
    dtype_can_store = bool(
        result_dtype is not None and np.can_cast(result_dtype, left_dtype, casting="same_kind")
    )

    return {
        "operation": {
            "name": operation,
            "category": ("comparison" if operation in COMPARISON_OPERATIONS else "arithmetic"),
            "defined_for_dtypes": result_dtype is not None,
            "error": operation_error,
        },
        "operands": [
            {
                "name": operand["name"],
                "shape": list(operand["shape"]),
                "axis_names": list(operand["axis_names"]),
                "dtype": str(operand["dtype"]),
                "logical_expansion_factor": expansion_factor(operand["shape"], output_shape),
            }
            for operand in operands
        ],
        "alignment": alignment,
        "result": {
            "shape": list(output_shape),
            "axis_names": [row["output_axis_name"] for row in alignment],
            "element_count": output_elements,
            "common_input_dtype": str(common_dtype),
            "operation_dtype": None if result_dtype is None else str(result_dtype),
            "estimated_nbytes": output_nbytes,
        },
        "in_place_on_left": {
            "shape_can_store_result": left_shape_can_store,
            "dtype_can_store_result_with_same_kind": dtype_can_store,
            "allowed_by_shape_and_dtype": left_shape_can_store and dtype_can_store,
        },
        "memory_limit_mb": float(memory_limit_mb),
        "warnings": warnings,
    }


def parse_json(raw: str) -> Any:
    try:
        return json.loads(raw)
    except json.JSONDecodeError as error:
        raise BroadcastingError("operands must be valid JSON") from error


def main() -> int:
    parser = argparse.ArgumentParser(
        description="Explain a NumPy broadcast operation before allocating its result"
    )
    parser.add_argument(
        "--operands",
        default=(
            '[{"name":"daily_metrics","shape":[2,3,2],'
            '"axis_names":["store","day","metric"],"dtype":"float64"},'
            '{"name":"metric_means","shape":[1,1,2],'
            '"axis_names":["summarized_store","summarized_day","metric"],'
            '"dtype":"float64"}]'
        ),
    )
    parser.add_argument("--operation", choices=tuple(OPERATIONS), default="subtract")
    parser.add_argument("--memory-limit-mb", type=float, default=256.0)
    parser.add_argument("--output", type=Path)
    args = parser.parse_args()

    try:
        operands = parse_json(args.operands)
        if not isinstance(operands, list):
            raise BroadcastingError("operands JSON must contain a list")
        report = analyze_broadcast(
            operands,
            operation=args.operation,
            memory_limit_mb=args.memory_limit_mb,
        )
    except BroadcastingError as error:
        parser.exit(2, f"broadcast-contract: {error}\n")

    text = json.dumps(report, ensure_ascii=False, indent=2) + "\n"
    if args.output:
        args.output.write_text(text, encoding="utf-8")
    else:
        print(text, end="")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
