Сборка, сортировка и согласованность массивов

Массивы мало уметь создать и отфильтровать. В аналитическом расчете приходится соединять порции данных, менять их порядок и считать категории так, чтобы значения, относящиеся к одному наблюдению, не разъехались.

  • найти уникальные значения, частоты и обратные индексы через unique;
  • отличить значение минимума или максимума от позиции первого экстремума;
  • проверить shape, dtype, порядок, политику NaN и сохранение соответствия наблюдений.

Проблема

После предыдущих уроков мы умеем описывать массив через shape, оси и dtype, менять его форму и выбирать наблюдения маской. Но реальные данные редко приходят одним готовым массивом.

Представьте два пакета заказов:

python
import numpy as np

amount_batch_1 = np.array([120.0, 80.0])
amount_batch_2 = np.array([120.0, 95.0])

order_id_batch_1 = np.array([101, 102])
order_id_batch_2 = np.array([103, 104])

Мы хотим:

  1. собрать пакеты в одну последовательность;
  2. расположить заказы по сумме;
  3. сохранить связь суммы с order_id;
  4. посчитать, сколько раз встречается каждая сумма;
  5. найти заказ с минимальной и максимальной суммой.

Наивный код легко дает правдоподобный, но неверный результат:

python
amounts = np.sort(np.concatenate([amount_batch_1, amount_batch_2]))
order_ids = np.concatenate([order_id_batch_1, order_id_batch_2])

Теперь первая сумма равна 80, а первый идентификатор — 101. В исходных данных сумма 80 принадлежала заказу 102. Оба массива по отдельности выглядят нормально, длины совпадают, исключения нет — но аналитический смысл разрушен.

Главная мысль урока:

Если несколько массивов описывают одни наблюдения, их общая позиция является частью данных. Любое изменение порядка нужно применять к ним одной и той же перестановкой.

Концепция

Наблюдение связывает параллельные массивы

В NumPy одно наблюдение часто хранится не в одной «строке», а в нескольких связанных массивах:

python
order_ids = np.array([101, 102, 103, 104])
amounts = np.array([120.0, 80.0, 120.0, 95.0])
segments = np.array(["A", "B", "C", "A"])

Позиция i означает один заказ:

text
order_ids[i] <-> amounts[i] <-> segments[i]

Это и есть инвариант согласованности. Он не записан в dtype или shape. NumPy не знает, что три массива связаны бизнес-смыслом, поэтому проверить и сохранить эту связь должен автор расчета.

Перед структурной операцией задайте четыре вопроса:

  1. Что является одним наблюдением?
  2. Какая ось перечисляет наблюдения?
  3. Создаем ли мы новую предметную ось или удлиняем существующую?
  4. Какие массивы должны пройти через ту же операцию или перестановку?

concatenate: удлинить существующую ось

np.concatenate соединяет массивы вдоль уже существующей оси. Число осей не меняется.

python
first = np.array([[120.0, 2.0], [80.0, 1.0]])
second = np.array([[120.0, 4.0], [95.0, 2.0]])

features = np.concatenate([first, second], axis=0)

assert first.shape == (2, 2)       # (order, metric)
assert second.shape == (2, 2)      # (order, metric)
assert features.shape == (4, 2)    # (order, metric)

Мы не создали ось batch. Мы объявили, что после загрузки пакет больше не является отдельной аналитической сущностью, и продолжили существующую ось order.

Правило формы для concatenate:

  • длины по выбранной оси складываются;
  • длины по всем остальным осям должны совпадать;
  • ndim результата равен ndim входов.

Например, формы (2, 3) и (4, 3) можно соединить по axis=0 в (6, 3). По axis=1 их соединить нельзя: длина нетронутой оси 0 различается.

Соединение по оси признаков тоже возможно:

python
amount = np.array([[120.0], [80.0], [95.0]])
items = np.array([[2.0], [1.0], [2.0]])

features = np.concatenate([amount, items], axis=1)
assert features.shape == (3, 2)  # (order, metric)

Но здесь особенно важно заранее доказать, что строка i в обоих массивах описывает один и тот же заказ. Совпадение shape само по себе этого не доказывает.

stack: создать новую ось

np.stack соединяет массивы вдоль новой оси. Все входные формы должны совпасть точно, а ndim увеличивается на один.

Пусть две матрицы имеют оси (store, day): план и факт продаж.

python
plan = np.array([[10, 12, 11], [8, 9, 10]])
fact = np.array([[9, 13, 11], [7, 10, 10]])

comparison = np.stack([plan, fact], axis=0)

assert plan.shape == (2, 3)          # (store, day)
assert comparison.shape == (2, 2, 3) # (scenario, store, day)

Длина новой оси scenario равна количеству входных массивов. Позиция 0 означает plan, позиция 1 — fact.

Сравним операции:

Вопрос concatenate stack
Ось уже существовала? Да Нет
Требование к формам Совпадают вне выбранной оси Совпадают полностью
Меняется ndim? Нет Увеличивается на 1
Типичный смысл Еще наблюдения или признаки Сценарий, модель, версия, повтор

Выбор функции определяется не внешним видом результата, а предметным смыслом оси. vstack, hstack и похожие сокращения удобны, но в начале обучения скрывают главный вопрос: какая именно ось меняется. Поэтому в этом уроке мы используем явные concatenate(..., axis=...) и stack(..., axis=...).

Сборка обычно создает новый буфер

В предыдущем уроке мы различали view и copy. concatenate и stack обычно выделяют новый массив и копируют в него значения:

python
left = np.array([1, 2])
right = np.array([3, 4])
result = np.concatenate([left, right])

assert not np.shares_memory(result, left)
assert not np.shares_memory(result, right)

Это важно для больших данных: однократная сборка нескольких частей нормальна, а добавление по одному элементу в цикле многократно копирует уже собранное.

Плохой шаблон:

python
result = np.array([], dtype=np.int64)
for value in range(10_000):
    result = np.concatenate([result, np.array([value])])

Если число элементов известно, создайте массив нужной формы заранее. Если части уже существуют, соберите их одним вызовом.

При сборке dtype может измениться

NumPy должен найти единый dtype результата:

python
integers = np.array([1, 2], dtype=np.int32)
floats = np.array([3.5], dtype=np.float64)
combined = np.concatenate([integers, floats])

assert combined.dtype == np.float64

Такое продвижение типа может быть правильным, но не должно быть неожиданным. Опаснее смешивание чисел и строк: числа могут превратиться в строки, после чего арифметика перестанет иметь исходный смысл. Поэтому до и после сборки проверяйте dtype.

sort возвращает значения, argsort — позиции

np.sort(amounts) отвечает на вопрос «каковы суммы по возрастанию?». Но для связанных массивов этого недостаточно: нужны позиции исходных заказов в новом порядке.

python
order_ids = np.array([101, 102, 103, 104])
amounts = np.array([120.0, 80.0, 120.0, 95.0])
segments = np.array(["A", "B", "C", "A"])

permutation = np.argsort(amounts, kind="stable")

sorted_amounts = amounts[permutation]
sorted_order_ids = order_ids[permutation]
sorted_segments = segments[permutation]

assert permutation.tolist() == [1, 3, 0, 2]
assert sorted_order_ids.tolist() == [102, 104, 101, 103]

permutation — это индексный массив из предыдущего урока. Он описывает одно изменение порядка и применяется ко всем массивам с grain order.

Механизм можно увидеть без NumPy:

python
permutation = sorted(range(len(amounts)), key=amounts.__getitem__)

Сначала сортируются не сами заказы, а их позиции. Затем позиции выбирают наблюдения в нужном порядке.

Почему сортировка должна быть устойчивой

Два заказа имеют сумму 120. Устойчивая сортировка сохраняет их прежний взаимный порядок: сначала заказ 101, затем 103.

Это полезно, когда:

  • равные значения встречаются регулярно;
  • исходный порядок уже имеет смысл, например время поступления;
  • результат сравнивают между запусками;
  • следующий шаг выбирает «первый» элемент среди равных.

Поэтому мы явно используем kind="stable", а не полагаемся на алгоритм по умолчанию.

Для убывания есть тонкая ловушка:

python
wrong = np.argsort(amounts, kind="stable")[::-1]

Срез [::-1] переворачивает весь результат, а вместе с ним — порядок внутри группы равных значений. Сортировка перестает быть устойчивой в желаемом смысле. Учебный артефакт ниже разворачивает группы различных значений, но сохраняет исходный порядок внутри каждой группы.

Обратная перестановка возвращает исходный порядок

Иногда после расчета в отсортированном порядке результат нужно вернуть к исходным заказам. Для этого строят обратную перестановку:

python
inverse = np.empty_like(permutation)
inverse[permutation] = np.arange(len(permutation))

assert np.array_equal(sorted_amounts[inverse], amounts)

Проверка восстановления — сильнее простой проверки длины. Она подтверждает, что ни одна позиция не потеряна и не повторена.

Пропуски в ключе сортировки требуют решения

NaN не означает обычное большое или маленькое число. Автоматическое расположение пропусков в начале или конце может повлиять на выбор лучших и худших наблюдений.

До сортировки зафиксируйте политику:

  • запретить пропуски и остановить расчет;
  • расположить их в конце;
  • отделить их в самостоятельную ветку;
  • заполнить значением — только если такое заполнение имеет предметный смысл.

В артефакте политика по умолчанию — ошибка. Вариант last нужно выбрать явно.

unique: не только список значений

np.unique по умолчанию возвращает отсортированные уникальные значения:

python
segments = np.array(["B", "A", "B", "C", "A"])
unique = np.unique(segments)

assert unique.tolist() == ["A", "B", "C"]

Это не порядок первого появления. Для аналитики обычно нужны дополнительные результаты:

python
unique, first_index, inverse, counts = np.unique(
    segments,
    return_index=True,
    return_inverse=True,
    return_counts=True,
)

Их смысл:

Результат Вопрос
unique Какие различные значения есть?
counts Сколько раз встретилось каждое значение из unique?
first_index Где значение впервые встретилось в исходном массиве?
inverse Как для каждой исходной позиции указать номер уникального значения?

Обратные индексы позволяют восстановить исходный массив:

python
assert np.array_equal(unique[inverse], segments)

А порядок первого появления можно получить через позиции:

python
first_seen = unique[np.argsort(first_index, kind="stable")]
assert first_seen.tolist() == ["B", "A", "C"]

Таким образом, unique одновременно дает словарь категорий, частоты и код каждой категории в исходных наблюдениях. Более глубокая работа с категориальными столбцами будет в фазе pandas; здесь важно понять позиционный механизм.

Значение экстремума и его позиция — разные ответы

python
amounts = np.array([120.0, 80.0, 120.0, 95.0])

minimum = amounts.min()       # 80.0
minimum_position = amounts.argmin()  # 1
maximum_position = amounts.argmax()  # 0

min и max возвращают значения. argmin и argmax возвращают позиции. Позиция нужна, чтобы получить связанное наблюдение:

python
assert order_ids[minimum_position] == 102
assert order_ids[maximum_position] == 101

Если экстремум встречается несколько раз, argmin и argmax возвращают первое вхождение в порядке обхода. Это не «единственный лучший заказ», а первый среди равных. При наличии NaN используйте заранее выбранную политику, а не случайную смесь argmax и nanargmax.

Минимальный структурный контракт

Для сборки запишите:

text
входные shape и dtype
предметные имена осей
существующая или новая ось
ожидаемые shape, оси и dtype результата

Для изменения порядка добавьте:

text
ключ сортировки
направление и политика равенств
политика NaN
перечень всех связанных массивов
проверка обратного восстановления

Такой контракт не является бюрократией. Он фиксирует именно те решения, которые NumPy не может вывести из бизнес-смысла самостоятельно.

Соберите это

Сначала реализуем главную часть вручную на маленьком примере. Так станет видно, что сортировка связанных данных — это работа с позициями.

python
order_ids = [101, 102, 103, 104]
amounts = [120.0, 80.0, 120.0, 95.0]
segments = ["A", "B", "C", "A"]

permutation = sorted(range(len(amounts)), key=amounts.__getitem__)

sorted_amounts = [amounts[i] for i in permutation]
sorted_order_ids = [order_ids[i] for i in permutation]
sorted_segments = [segments[i] for i in permutation]

assert permutation == [1, 3, 0, 2]
assert sorted_amounts == [80.0, 95.0, 120.0, 120.0]
assert sorted_order_ids == [102, 104, 101, 103]
assert sorted_segments == ["B", "A", "A", "C"]

Обратите внимание: мы не сортируем order_ids и segments независимо. Их значения не являются ключами сортировки. Они только следуют за общей перестановкой.

Теперь повторим на NumPy:

python
order_ids = np.array(order_ids)
amounts = np.array(amounts)
segments = np.array(segments)

permutation = np.argsort(amounts, kind="stable")

assert np.array_equal(amounts[permutation], sorted_amounts)
assert np.array_equal(order_ids[permutation], sorted_order_ids)
assert np.array_equal(segments[permutation], sorted_segments)

Добавьте обратную перестановку и проверьте восстановление каждого массива:

python
inverse = np.empty_like(permutation)
inverse[permutation] = np.arange(permutation.size)

assert np.array_equal(amounts[permutation][inverse], amounts)
assert np.array_equal(order_ids[permutation][inverse], order_ids)
assert np.array_equal(segments[permutation][inverse], segments)

Сломайте это

Ошибка 1. Перепутать продолжение оси и создание оси

python
np.stack([amount_batch_1, amount_batch_2], axis=0).shape
# (2, 2): (batch, order_in_batch)

np.concatenate([amount_batch_1, amount_batch_2], axis=0).shape
# (4,): (order)

Обе операции выполняются, но отвечают на разные предметные вопросы. Назовите оси до выбора функции.

Ошибка 2. Соединить матрицы по неверной оси

Два пакета (order, metric) нужно продолжить по оси order. Соединение по metric либо завершится ошибкой формы, либо создаст правдоподобную матрицу с неверным смыслом.

Ошибка 3. Не заметить продвижение dtype

Соедините int32 и float64. Затем попробуйте числа и строки. После каждого опыта проверьте dtype и объясните, допустим ли он для следующей операции.

Ошибка 4. Сортировать связанные массивы независимо

python
np.sort(amounts)
np.sort(order_ids)

Результаты имеют одинаковую длину, но больше не описывают одни заказы. Структурная проверка должна использовать единую перестановку, а не только shape.

Ошибка 5. Перевернуть устойчивую сортировку целиком

python
np.argsort(amounts, kind="stable")[::-1]

Проверьте порядок заказов 101 и 103 с одинаковой суммой 120. Простое обращение массива меняет порядок внутри группы равных значений.

Ошибка 6. Считать порядок unique порядком появления

Для ['B', 'A', 'B'] результат np.unique начинается с 'A'. Если нужен порядок первого появления, используйте return_index и явно восстановите его.

Ошибка 7. Считать argmax доказательством уникального лидера

При двух максимумах argmax возвращает первую позицию. Сначала посчитайте число вхождений максимума, если уникальность лидера важна для решения.

Ошибка 8. Собирать массив по одному элементу

Повторный concatenate в цикле постоянно создает новые буферы. Сравните этот подход с одной сборкой списка частей или предварительным выделением массива.

01

Какой shape получится при concatenate массивов (2, 3) и (4, 3) по axis=0?

Длины существующей оси 0 складываются, а длина нетронутой оси 1 совпадает.

02

Почему stack массивов формы (2, 3) и (4, 3) не подходит без предварительного решения о данных?

Новая ось может быть добавлена только поверх массивов с одинаковой формой.

03

Как безопасно расположить amounts, order_ids и segments по возрастанию amounts?

Одна перестановка сохраняет связь всех значений с тем же наблюдением.

04

Что гарантирует kind='stable' при равных ключах сортировки?

Устойчивая сортировка не переставляет относительно друг друга наблюдения с одинаковым ключом.

05

Почему argsort(..., kind='stable')[::-1] не является общим безопасным способом устойчивой сортировки по убыванию?

Чтобы сохранить устойчивость, нужно изменить порядок групп значений, не обращая элементы внутри каждой равной группы.

06

Какой результат np.unique нужен, чтобы восстановить исходный массив как unique[inverse]?

Обратные индексы указывают для каждой исходной позиции номер соответствующего уникального значения.

07

В массиве [5, 2, 5, 2] чему равны argmin и argmax без указания оси?

Функции возвращают позиции первых вхождений экстремумов: первый минимум 2 стоит на позиции 1, первый максимум 5 — на позиции 0.

08

Какая проверка лучше всего подтверждает, что перестановка не потеряла и не повторила позиции?

Обратное восстановление проверяет всю позиционную структуру, а не только размер или крайние значения.

from __future__ import annotations

import argparse
import json
from collections.abc import Mapping, Sequence
from numbers import Integral
from pathlib import Path
from typing import Any

import numpy as np


class StructureError(ValueError):
    """Raised when arrays cannot be combined or reordered safely."""


def _as_array(values: object, *, name: str) -> np.ndarray:
    try:
        array = np.asarray(values)
    except (TypeError, ValueError) as error:
        raise StructureError(f"{name} must form a rectangular array") from error
    if array.ndim == 0:
        raise StructureError(f"{name} must have at least one axis")
    if array.size == 0 or any(length == 0 for length in array.shape):
        raise StructureError(f"{name} must not contain empty axes")
    if array.dtype == object:
        raise StructureError(f"{name} must have a homogeneous non-object dtype")
    return array


def _axis_names(axis_names: Sequence[str], ndim: int) -> tuple[str, ...]:
    names = tuple(axis_names)
    if len(names) != ndim:
        raise StructureError(f"expected {ndim} axis names, got {len(names)}")
    if any(not isinstance(name, str) or not name.strip() for name in names):
        raise StructureError("axis names must be non-empty strings")
    if len(set(names)) != len(names):
        raise StructureError("axis names must be unique")
    return names


def _normalize_axis(axis: int, ndim: int, *, insertion: bool = False) -> int:
    if not isinstance(axis, Integral) or isinstance(axis, bool):
        raise StructureError("axis must be an integer")
    output_ndim = ndim + 1 if insertion else ndim
    normalized = int(axis) + output_ndim if axis < 0 else int(axis)
    if normalized < 0 or normalized >= output_ndim:
        action = "insertion" if insertion else "existing"
        raise StructureError(f"axis {axis} is not a valid {action} axis for ndim={ndim}")
    return normalized


def combine_report(
    arrays: Sequence[object],
    *,
    mode: str,
    axis: int,
    axis_names: Sequence[str],
    new_axis_name: str | None = None,
    allow_dtype_promotion: bool = False,
) -> dict[str, Any]:
    """Combine arrays and expose the structural contract of the result."""
    if len(arrays) < 2:
        raise StructureError("combine expects at least two arrays")
    converted = [_as_array(values, name=f"arrays[{index}]") for index, values in enumerate(arrays)]
    ndim = converted[0].ndim
    if any(array.ndim != ndim for array in converted[1:]):
        raise StructureError("all arrays must have the same number of axes")
    names = _axis_names(axis_names, ndim)

    dtypes = [array.dtype for array in converted]
    if not allow_dtype_promotion and any(dtype != dtypes[0] for dtype in dtypes[1:]):
        raise StructureError(
            "input dtypes differ; align them first or choose allow_dtype_promotion explicitly"
        )

    input_shapes = [array.shape for array in converted]
    if mode == "concatenate":
        normalized_axis = _normalize_axis(axis, ndim)
        reference = input_shapes[0]
        for shape in input_shapes[1:]:
            mismatches = [
                index
                for index, (left, right) in enumerate(zip(reference, shape, strict=True))
                if index != normalized_axis and left != right
            ]
            if mismatches:
                raise StructureError(
                    "concatenate requires equal lengths on every non-concatenated axis"
                )
        result = np.concatenate(converted, axis=normalized_axis)
        output_axis_names = names
        axis_role = "existing"
    elif mode == "stack":
        normalized_axis = _normalize_axis(axis, ndim, insertion=True)
        if any(shape != input_shapes[0] for shape in input_shapes[1:]):
            raise StructureError("stack requires exactly equal input shapes")
        if not isinstance(new_axis_name, str) or not new_axis_name.strip():
            raise StructureError("stack requires a non-empty new_axis_name")
        if new_axis_name in names:
            raise StructureError("new_axis_name must not duplicate an existing axis name")
        result = np.stack(converted, axis=normalized_axis)
        output_axis_names = names[:normalized_axis] + (new_axis_name,) + names[normalized_axis:]
        axis_role = "new"
    else:
        raise StructureError("mode must be 'concatenate' or 'stack'")

    if result.dtype == object:
        raise StructureError("combination produced object dtype")
    return {
        "operation": mode,
        "axis": normalized_axis,
        "axis_role": axis_role,
        "input": {
            "shapes": [list(shape) for shape in input_shapes],
            "dtypes": [str(dtype) for dtype in dtypes],
            "axis_names": list(names),
        },
        "output": {
            "shape": list(result.shape),
            "dtype": str(result.dtype),
            "axis_names": list(output_axis_names),
            "value": result.tolist(),
            "nbytes": int(result.nbytes),
            "shares_memory_with_input": [
                bool(np.shares_memory(result, array)) for array in converted
            ],
        },
        "dtype_promotion_allowed": allow_dtype_promotion,
    }


def manual_stable_permutation(keys: Sequence[float], *, direction: str = "ascending") -> list[int]:
    """Show that sorting related arrays starts with sorting positions."""
    if direction not in {"ascending", "descending"}:
        raise StructureError("direction must be 'ascending' or 'descending'")
    if len(keys) == 0:
        raise StructureError("keys must not be empty")
    try:
        return sorted(range(len(keys)), key=keys.__getitem__, reverse=direction == "descending")
    except TypeError as error:
        raise StructureError("keys must be mutually comparable") from error


def _descending_stable_permutation(keys: np.ndarray, ascending: np.ndarray) -> np.ndarray:
    """Reverse value groups while preserving the order inside every tie."""
    sorted_keys = keys[ascending]
    starts = np.r_[0, np.flatnonzero(sorted_keys[1:] != sorted_keys[:-1]) + 1]
    ends = np.r_[starts[1:], len(ascending)]
    groups = [ascending[start:end] for start, end in zip(starts, ends, strict=True)]
    return np.concatenate(groups[::-1])


def _numeric_keys(values: object) -> np.ndarray:
    keys = _as_array(values, name="keys")
    if keys.ndim != 1:
        raise StructureError("keys must be one-dimensional")
    if (
        not np.issubdtype(keys.dtype, np.number)
        or np.issubdtype(keys.dtype, np.bool_)
        or np.issubdtype(keys.dtype, np.complexfloating)
    ):
        raise StructureError("keys must contain real numbers")
    if np.isinf(keys).any():
        raise StructureError("infinite keys are not supported")
    return keys


def _arrays_equal(left: np.ndarray, right: np.ndarray) -> bool:
    if np.issubdtype(left.dtype, np.inexact):
        return bool(np.array_equal(left, right, equal_nan=True))
    return bool(np.array_equal(left, right))


def aligned_order_report(
    keys: object,
    payloads: Mapping[str, object],
    *,
    direction: str = "ascending",
    nan_policy: str = "error",
) -> dict[str, Any]:
    """Apply one stable ordering permutation to keys and all related values."""
    key_array = _numeric_keys(keys)
    if direction not in {"ascending", "descending"}:
        raise StructureError("direction must be 'ascending' or 'descending'")
    if nan_policy not in {"error", "last"}:
        raise StructureError("nan_policy must be 'error' or 'last'")
    if not isinstance(payloads, Mapping) or not payloads:
        raise StructureError("payloads must be a non-empty mapping")

    converted_payloads: dict[str, np.ndarray] = {}
    for name, values in payloads.items():
        if not isinstance(name, str) or not name:
            raise StructureError("payload names must be non-empty strings")
        array = _as_array(values, name=f"payloads[{name!r}]")
        if array.ndim != 1 or len(array) != len(key_array):
            raise StructureError("every payload must be 1D and have the same length as keys")
        converted_payloads[name] = array

    missing = np.isnan(key_array)
    if missing.any() and nan_policy == "error":
        raise StructureError("NaN keys found; choose nan_policy='last' explicitly")
    valid_positions = np.flatnonzero(~missing)
    if valid_positions.size == 0:
        raise StructureError("at least one finite key is required")

    local_ascending = np.argsort(key_array[valid_positions], kind="stable")
    ascending = valid_positions[local_ascending]
    if direction == "descending":
        ordered_valid = _descending_stable_permutation(key_array, ascending)
    else:
        ordered_valid = ascending
    permutation = np.concatenate((ordered_valid, np.flatnonzero(missing)))
    inverse = np.empty(len(permutation), dtype=np.intp)
    inverse[permutation] = np.arange(len(permutation))

    sorted_keys = key_array[permutation]
    sorted_payloads = {name: array[permutation] for name, array in converted_payloads.items()}
    restored_keys = sorted_keys[inverse]
    return {
        "direction": direction,
        "nan_policy": nan_policy,
        "tie_policy": "stable: equal keys keep their original relative order",
        "permutation": permutation.tolist(),
        "inverse_permutation": inverse.tolist(),
        "sorted_keys": sorted_keys.tolist(),
        "sorted_payloads": {name: array.tolist() for name, array in sorted_payloads.items()},
        "argmin_first": int(np.nanargmin(key_array)),
        "argmax_first": int(np.nanargmax(key_array)),
        "restoration_check": _arrays_equal(restored_keys, key_array),
    }


def unique_report(values: object) -> dict[str, Any]:
    """Explain unique values through counts, first positions, and reconstruction."""
    array = _as_array(values, name="values")
    if array.ndim != 1:
        raise StructureError("values must be one-dimensional")
    unique, first_indices, inverse, counts = np.unique(
        array,
        return_index=True,
        return_inverse=True,
        return_counts=True,
    )
    first_seen_order = np.argsort(first_indices, kind="stable")
    reconstructed = unique[inverse]
    return {
        "input_size": int(array.size),
        "unique_size": int(unique.size),
        "duplicate_count": int(array.size - unique.size),
        "sorted_unique_values": unique.tolist(),
        "counts": counts.tolist(),
        "inverse_indices": inverse.tolist(),
        "first_indices_in_sorted_unique_order": first_indices.tolist(),
        "values_in_first_seen_order": unique[first_seen_order].tolist(),
        "reconstruction_check": _arrays_equal(reconstructed, array),
    }


def _parse_json(raw: str, *, name: str) -> Any:
    try:
        return json.loads(raw)
    except json.JSONDecodeError as error:
        raise StructureError(f"{name} must be valid JSON") from error


def _json_safe(value: Any) -> Any:
    if isinstance(value, dict):
        return {key: _json_safe(item) for key, item in value.items()}
    if isinstance(value, list):
        return [_json_safe(item) for item in value]
    if isinstance(value, float) and not np.isfinite(value):
        return None
    return value


def _write_report(report: dict[str, Any], output: Path | None) -> None:
    rendered = json.dumps(_json_safe(report), ensure_ascii=False, indent=2, allow_nan=False)
    if output is None:
        print(rendered)
    else:
        output.write_text(rendered + "\n", encoding="utf-8")


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        description="Audit composition, ordering, and uniqueness of related NumPy arrays"
    )
    subparsers = parser.add_subparsers(dest="command", required=True)

    combine = subparsers.add_parser("combine", help="concatenate or stack arrays")
    combine.add_argument("--arrays", required=True, help="JSON list of arrays")
    combine.add_argument("--mode", choices=("concatenate", "stack"), required=True)
    combine.add_argument("--axis", type=int, required=True)
    combine.add_argument("--axis-names", nargs="+", required=True)
    combine.add_argument("--new-axis-name")
    combine.add_argument("--allow-dtype-promotion", action="store_true")
    combine.add_argument("--output", type=Path)

    order = subparsers.add_parser("order", help="sort related arrays by one key")
    order.add_argument("--keys", required=True, help="JSON list of numeric keys")
    order.add_argument("--payloads", required=True, help="JSON object of related arrays")
    order.add_argument("--direction", choices=("ascending", "descending"), default="ascending")
    order.add_argument("--nan-policy", choices=("error", "last"), default="error")
    order.add_argument("--output", type=Path)

    unique = subparsers.add_parser("unique", help="explain unique values and counts")
    unique.add_argument("--values", required=True, help="JSON list")
    unique.add_argument("--output", type=Path)
    return parser


def main() -> int:
    parser = build_parser()
    args = parser.parse_args()
    try:
        if args.command == "combine":
            report = combine_report(
                _parse_json(args.arrays, name="arrays"),
                mode=args.mode,
                axis=args.axis,
                axis_names=args.axis_names,
                new_axis_name=args.new_axis_name,
                allow_dtype_promotion=args.allow_dtype_promotion,
            )
        elif args.command == "order":
            payloads = _parse_json(args.payloads, name="payloads")
            if not isinstance(payloads, dict):
                raise StructureError("payloads JSON must be an object")
            report = aligned_order_report(
                _parse_json(args.keys, name="keys"),
                payloads,
                direction=args.direction,
                nan_policy=args.nan_policy,
            )
        else:
            report = unique_report(_parse_json(args.values, name="values"))
        _write_report(report, args.output)
    except (OSError, StructureError) as error:
        parser.exit(2, f"error: {error}\n")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
