Произведение длин shape даёт size, а nbytes оценивается как size × itemsize.

Tiny-выборка содержит значения 0..200, но контракт источника разрешает 0..20 000 без пропусков. Какой вывод наиболее обоснован?

Dtype, память и диапазоны

Выбирайте dtype не по значениям маленького примера, а по предметному контракту: будущему диапазону, пропускам, допустимой ошибке и операциям, которые будут выполнены над массивом.

После урока вы сможете:

  • различать наблюдаемый, ожидаемый и возникающий после вычислений диапазоны;
  • выбрать политику хранения пропусков до преобразования массива;
  • объяснить практическую разницу между float32 и float64 через допустимую ошибку;
  • обнаружить потерю дробной части, выход за диапазон и переполнение узкого integer;
  • использовать astype, np.can_cast и np.result_type, понимая границы каждой проверки;
  • проверить кандидат на соответствие dtype-контракту и оценить память планируемого массива.

Проблема

В первом уроке мы выяснили, что у обычного ndarray один общий dtype. Во втором — научились записывать форму вместе с предметными именами осей и предсказывать форму результата.

Теперь объединим эти две идеи. Аналитическая система должна хранить почасовое количество событий для магазинов:

text
(store=500, day=365, hour=24)

Из контракта формы уже можно получить число элементов:

text
size = 500 * 365 * 24 = 4 380 000

Tiny-выборка выглядит так:

python
sample = [0, 12, 200]

Python и NumPy на обычной 64-битной платформе, скорее всего, создадут из нее int64. Буфер планируемого массива тогда потребует:

text
4 380 000 элементов * 8 байт = 35 040 000 байт

Если посмотреть только на текущие значения 0..200, возникает соблазн выбрать uint8:

text
4 380 000 элементов * 1 байт = 4 380 000 байт

Памяти потребуется в восемь раз меньше. Но описание источника разрешает рекламные пики до 20 000 событий в час. uint8 хранит только 0..255, поэтому экономный тип нарушает контракт будущих данных. Для диапазона 0..20 000 нужен как минимум uint16:

text
4 380 000 элементов * 2 байта = 8 760 000 байт

uint16 все еще экономит 75% буфера относительно int64, но не отбрасывает допустимые значения.

Даже это решение еще не окончательное. Перед преобразованием нужно выяснить:

  1. Могут ли после корректировок появиться отрицательные значения?
  2. Возможны ли пропуски и как они должны храниться?
  3. Будут ли значения складываться, умножаться или усредняться?
  4. Какой диапазон появится после этих операций?
  5. Допустима ли потеря точности, если тип вещественный?

Именно поэтому задача урока — не найти самый маленький dtype. Нужно найти достаточно компактный тип, который сохраняет смысл данных и будущих вычислений.

Концепция

dtype — общий контракт элементов массива

Python-список может одновременно хранить целое число, строку и None. Обычный массив NumPy интерпретирует все элементы по одному правилу:

python
import numpy as np

values = np.array([1, 2, 3], dtype=np.int16)

int16 сообщает NumPy как минимум три вещи:

  • каждый элемент является целым числом;
  • на каждый элемент отводится 16 бит, то есть 2 байта;
  • допустимый диапазон ограничен возможностями знакового 16-битного представления.

У отдельного Python-числа нет обязанности занимать тот же фиксированный объём, что и элемент ndarray. Поэтому Python int и NumPy int16 нельзя считать взаимозаменяемыми контрактами только потому, что оба сейчас показывают число 12.

Один общий dtype также напоминает о границе модели. В прошлом уроке массив имел ось metric, где рядом стояли количество заказов и выручка. Если показатели требуют разных типов, правил пропусков или единиц хранения, удобнее держать их в отдельных однородных массивах либо перейти к именованной табличной структуре. Узкий dtype не должен быть причиной смешивать разные сущности.

Как форма превращается в оценку памяти

Мы уже знаем:

text
size = произведение длин shape

Теперь добавляется размер одного элемента:

text
nbytes = size * itemsize

NumPy показывает оба значения непосредственно:

python
values = np.zeros((2, 3), dtype=np.uint16)

assert values.size == 6
assert values.itemsize == 2
assert values.nbytes == 12
Атрибут Что означает Что не означает
size число элементов число байтов
itemsize байты одного элемента диапазон значений сам по себе
nbytes размер буфера элементов вся память процесса Python

nbytes не учитывает объект массива, служебные структуры, временные результаты и другие объекты программы. Но для больших плотных массивов это полезная нижняя оценка и удобный способ сравнить кандидатов до выделения памяти.

Планируемую память можно посчитать без создания огромного массива:

python
from math import prod

planned_shape = (500, 365, 24)
planned_size = prod(planned_shape)
uint16_bytes = planned_size * np.dtype("uint16").itemsize

assert planned_size == 4_380_000
assert uint16_bytes == 8_760_000

Почему восемь бит дают диапазон 0..255

Один бит принимает два состояния. b бит дают 2**b комбинаций. Для восьми бит:

text
2**8 = 256 комбинаций

У беззнакового типа все комбинации отданы неотрицательным числам. Отсчет начинается с нуля, поэтому последняя комбинация соответствует не 256, а 255:

text
uint8: 0..255

Общая формула:

text
unsigned minimum = 0
unsigned maximum = 2**b - 1

Знаковый тип делит пространство между отрицательной и неотрицательной частями. Для привычного представления целых в NumPy:

text
signed minimum = -(2**(b - 1))
signed maximum = 2**(b - 1) - 1

Отсюда получаются границы:

dtype Байты Минимум Максимум
uint8 1 0 255
int8 1 -128 127
uint16 2 0 65 535
int16 2 -32 768 32 767
uint32 4 0 4 294 967 295
int32 4 -2 147 483 648 2 147 483 647

Запоминать всю таблицу не нужно. Важно понимать устройство диапазона и проверять точные границы через np.iinfo.

unsigned не является «более точным integer». Он просто не хранит отрицательные значения и использует освободившиеся комбинации для более высокой верхней границы. Выбирайте его только тогда, когда отрицательные значения запрещены предметным контрактом, а не потому, что текущий файл случайно не содержит минусов.

Три диапазона вместо одного

Безопасное решение учитывает три разных диапазона.

1. Наблюдаемый диапазон

Это минимум и максимум конкретной выборки:

text
sample: 0..200

Он нужен для диагностики текущих данных, но не описывает все будущие значения.

2. Ожидаемый диапазон

Это границы из предметного контракта, спецификации источника или согласованной политики:

text
allowed hourly events: 0..20 000

Именно этот диапазон должен помещаться в тип, если массив будет использоваться в production. Если границы неизвестны, честный результат аудита — предупреждение, а не уверенная рекомендация.

3. Производный диапазон

Это значения, которые могут появиться после вычисления. Если сложить 365 дневных значений до 20 000, теоретический максимум равен:

text
365 * 20 000 = 7 300 000

Исходное значение помещается в uint16, а годовой итог — уже нет. NumPy часто использует более широкий аккумулятор для сумм узких integer, но полагаться на это без проверки нельзя. Нужно предсказывать тип и диапазон результата, особенно перед сохранением обратно в узкий массив или внешнюю схему.

Таким образом, «тип подходит исходному столбцу» и «тип подходит всему pipeline» — разные утверждения.

Идентификатор не становится измерением из-за integer dtype

Числовой идентификатор может выглядеть как обычное целое, но над ним обычно не нужно складывать или вычислять среднее. При выборе хранения важно учитывать:

  • максимальное значение всей системы, а не текущего файла;
  • специальные или внешние диапазоны идентификаторов;
  • возможную смену генератора ключей;
  • необходимость сохранить ведущие нули, если это на самом деле код.

Поэтому правило «возьмем минимальный integer по min и max» особенно опасно для ID и кодов. Сначала определите семантический тип значения, затем физическое представление.

Пропуск — часть контракта, а не случайное значение

Обычные integer dtype NumPy не имеют отдельного представления NaN:

python
np.array([1, np.nan, 3], dtype=np.int16)

Такое преобразование невозможно. Перед выбором типа нужно решить, что означает отсутствие значения.

Политика Когда подходит Риск
Остановить расчет пропуски запрещены контрактом pipeline требует исправления источника
Floating dtype с NaN дробные значения естественны или потеря integer-семантики допустима целые представляются как float
Отдельная маска валидности нужен числовой буфер и явное состояние отсутствия два массива нужно хранить согласованно
Sentinel, например -1 значение гарантированно вне предметного диапазона будущий диапазон может пересечь sentinel
Nullable integer таблицы нужны целые значения и пропуски в табличной модели это уже модель pandas, а не обычный ndarray

Нельзя автоматически заменить пропуск нулем: ноль часто является настоящим наблюдением. Нельзя автоматически выбрать -1: в данных с корректировками отрицательное значение может быть допустимым. Политика должна быть названа явно.

float32 и float64: вопрос допустимой ошибки

Вещественные типы хранят приближения. Большинство десятичных дробей не имеют точного конечного двоичного представления, поэтому значения после вычислений могут немного отличаться от математического результата.

Для практического выбора достаточно помнить порядок величин:

  • float32 занимает 4 байта и обычно сохраняет около 7 значащих десятичных цифр;
  • float64 занимает 8 байт и обычно сохраняет около 15–16 значащих цифр.

Число значащих цифр важнее количества знаков после запятой. Посмотрим на значение большого масштаба:

python
source = np.array([10_000_000.1], dtype=np.float64)
narrow = source.astype(np.float32)

print(source)
print(narrow)

В float32 небольшая дробная часть рядом с десятью миллионами теряется. Ошибку можно измерить:

python
restored = narrow.astype(np.float64)
max_abs_error = np.max(np.abs(restored - source))

assert max_abs_error > 0.09

Если допустимая ошибка признака модели равна 0.2, такое преобразование может пройти контракт. Если финансовый итог должен совпадать до копейки, оно недопустимо.

np.finfo(dtype).eps показывает расстояние от 1.0 до следующего представимого числа данного типа. Это свойство типа, а не универсальный допуск для любого массива. Ошибка зависит от масштаба значений и последовательности операций. Устойчивые сравнения и накопление ошибки подробно рассматриваются в уроке 02/10.

Для денежных сумм часто безопаснее хранить целое число минимальных денежных единиц, если такой контракт поддерживает предметную задачу. Но это не универсальное решение: валюты, округление и внешние системы требуют отдельной спецификации.

astype выполняет преобразование, но не утверждает его безопасность

python
values = np.array([12.9, -2.1])
converted = values.astype(np.int16)

print(converted)
text
[12 -2]

Дробная часть отброшена в сторону нуля. NumPy выполнил запрошенную операцию, но не знает, допустима ли такая потеря для бизнеса.

Выход за диапазон при преобразовании уже созданного массива также может не остановить расчет:

python
values = np.array([256])
converted = values.astype(np.uint8)

assert converted[0] == 0

Поэтому безопасная последовательность выглядит так:

  1. проверить наблюдаемый и ожидаемый диапазоны;
  2. проверить пропуски и дробные значения;
  3. выполнить преобразование;
  4. измерить ошибку или точное совпадение после round-trip;
  5. только затем заменить исходный тип в pipeline.

Что проверяют np.can_cast и np.result_type

np.can_cast сравнивает типы, а не конкретные значения:

python
np.can_cast(np.dtype("int64"), np.dtype("uint16"), casting="safe")
# False

Ответ консервативен и правильный на уровне типов: произвольный int64 может быть отрицательным или превышать 65 535. Но конкретный предметный контракт 0..20 000 помещается в uint16. Поэтому np.can_cast полезен как строгий барьер, но не заменяет проверку диапазона конкретных данных и будущих границ.

np.result_type помогает предсказать тип результата совместной операции:

python
result_dtype = np.result_type(
    np.array([1], dtype=np.uint8),
    np.array([1], dtype=np.int16),
)

assert result_dtype == np.dtype("int16")

Это полезнее догадки по одному операнду. Точные правила продвижения NumPy 2.x сложнее одной таблицы; на этом этапе достаточно проверять тип результата для критичных операций, а не запоминать все сочетания.

Алгоритм выбора dtype

Перед изменением типа пройдите семь шагов.

  1. Назовите смысл значений. Это количество, измерение, сумма, коэффициент, ID или код?
  2. Задайте допустимые будущие границы. Не ограничивайтесь минимумом и максимумом tiny-выборки.
  3. Определите политику пропусков. Ошибка, NaN, маска, sentinel или nullable-таблица?
  4. Учтите следующие операции. Какой диапазон и тип появятся после суммы, разности или умножения?
  5. Задайте допустимую ошибку. Для floating-преобразования она должна следовать из задачи.
  6. Сравните память кандидатов. Используйте планируемый shape, а не только размер примера.
  7. Проверьте преобразование. Диапазон, дробную часть, round-trip и тип результата операции.

Самый маленький тип рассматривается только на шестом шаге. Экономия памяти не может компенсировать неверные данные.

Соберите это

Построим прозрачную версию dtype-контракта до использования готовых справочников NumPy.

Шаг 1. Оцените число элементов и память

python
from math import prod

planned_shape = (500, 365, 24)
planned_size = prod(planned_shape)

bytes_per_int64 = 8
bytes_per_uint16 = 2

int64_nbytes = planned_size * bytes_per_int64
uint16_nbytes = planned_size * bytes_per_uint16

assert planned_size == 4_380_000
assert int64_nbytes == 35_040_000
assert uint16_nbytes == 8_760_000

Это продолжение shape-контракта прошлого урока: длины осей определяют size, а dtype добавляет стоимость каждого элемента.

Шаг 2. Выведите диапазон integer

python
def integer_bounds(bits, signed):
    if signed:
        return -(2 ** (bits - 1)), 2 ** (bits - 1) - 1
    return 0, 2**bits - 1
python
assert integer_bounds(8, signed=False) == (0, 255)
assert integer_bounds(8, signed=True) == (-128, 127)
assert integer_bounds(16, signed=False) == (0, 65_535)
assert integer_bounds(16, signed=True) == (-32_768, 32_767)

Шаг 3. Разделите наблюдение и контракт

python
sample_min = 0
sample_max = 200

expected_min = 0
expected_max = 20_000

Проверка uint8 по выборке пройдет:

python
uint8_min, uint8_max = integer_bounds(8, signed=False)

observed_fits_uint8 = (
    sample_min >= uint8_min
    and sample_max <= uint8_max
)

assert observed_fits_uint8

Проверка предметного контракта — нет:

python
expected_fits_uint8 = (
    expected_min >= uint8_min
    and expected_max <= uint8_max
)

assert not expected_fits_uint8

Для uint16 проходят обе проверки.

Шаг 4. Зафиксируйте политику пропусков

Минимальная спецификация может выглядеть так:

python
dtype_contract = {
    "semantic_type": "event_count",
    "expected_min": 0,
    "expected_max": 20_000,
    "allow_missing": False,
    "target_dtype": "uint16",
}

Если пропуск обнаружен при allow_missing=False, расчет должен остановиться. Если пропуски разрешены, спецификация обязана назвать способ хранения; одного флага в production-контракте недостаточно.

Шаг 5. Проверьте преобразование float через round-trip

python
source = np.array([10_000_000.1], dtype=np.float64)
converted = source.astype(np.float32)
restored = converted.astype(np.float64)

error = np.max(np.abs(restored - source))

assert error > 0.09

Теперь решение можно сравнить с бизнес-допуском:

python
max_allowed_error = 0.01

assert error > max_allowed_error

Значит, float32 должен быть отклонен для этого сценария, хотя само преобразование синтаксически успешно.

Шаг 6. Сформулируйте итог до кода

Для массива событий решение выглядит так:

text
observed range: 0..200
expected range: 0..20 000
missing policy: missing values are rejected
planned shape: (500, 365, 24)
target dtype: uint16
projected buffer: 8 760 000 bytes
decision: approved for storage, derived aggregates require a separate dtype check

Это и есть прозрачный механизм, который автоматизирует артефакт урока.

Ошибка 4. Незаметно отбросить дробную часть

Ошибка 5. Использовать sentinel, который стал допустимым значением

В первой версии контракта -1 мог означать пропуск, потому что события были только неотрицательными. После появления корректировок -1 стал настоящим значением. Старые пропуски теперь невозможно отличить от наблюдений без дополнительной информации.

Ошибка 6. Потерять малое изменение рядом с большим float

Операция выполнилась, но выбранный масштаб и точность float32 не позволили сохранить прибавление.

Ошибка 7. Переполнить уже существующий узкий массив

python

Математический результат 260 не помещается в uint8, и вычисление обернулось по диапазону. Проверка при создании массива не защищает все будущие операции.

Ошибка 8. Проверить хранение, но не тип агрегата

python
daily = np.full(365, 20_000, dtype=np.uint16)

print(year_total)

Ошибка 9. Принять nbytes за всю память pipeline

Даже если исходный буфер занимает 9 МБ, операция может создать временный массив другого типа, копию или несколько промежуточных результатов. nbytes — полезная оценка хранения, но не профиль пикового потребления памяти.

Аудитор принимает sample значений и отдельно заданный контракт:

Дополнительное чтение

  • Яндекс Образование: модули math и numpy (RU) — начните с разделов «Какие атрибуты и методы есть у объектов ndarray», «Типы данных в numpy» и «Автоматический выбор типа данных». Они ещё раз связывают однородность массива, dtype, itemsize, явный выбор типа и риск переполнения на коротких примерах.
  • Дмитрий Макаров: массив NumPy (RU) — прочитайте разделы «Тип данных элементов массива» и «Свойства (атрибуты) массива». Особенно полезен разбор связи dtype, itemsize, size и nbytes: по нему можно самостоятельно оценить память до создания большого массива.
  • NumPy: Data types (EN) — основной официальный обзор. Сосредоточьтесь на преобразованиях между типами, числовых типах фиксированной ширины, переполнении и отличиях NumPy-скаляров от чисел Python; строки и extended precision пока можно пропустить.
  • numpy.iinfo (EN) — точный справочник по границам целочисленного dtype. Используйте его перед сужением типа: сравните фактические минимум и максимум данных с np.iinfo(target_dtype).min и .max.
  • numpy.finfo (EN) — справочник по диапазону и точности вещественных типов. Для этого урока достаточно разобраться с eps, min, max и bits; normal и subnormal значения понадобятся только при более глубоком изучении численной устойчивости.
  • ndarray.astype (EN) — точный контракт явного преобразования массива. Обратите внимание на параметры casting и copy, а затем проверьте на своих данных, чем успешное выполнение операции отличается от сохранения значений.
  • numpy.can_cast (EN) — определения режимов safe, same_kind и unsafe. Читайте вместе с примером из урока: функция проверяет совместимость типов, а не то, помещаются ли конкретные значения текущего массива.
  • numpy.result_type (EN) — небольшой API-справочник для предсказания dtype результата до вычисления. Повторите примеры, а затем передайте типы из своего аудита и сравните прогноз с фактическим результатом операции.
  • NumPy: Data type promotion (EN) — понятное объяснение того, как NumPy выбирает общий тип для разных операндов. В первую очередь разберите иерархию uint → int → float → complex и отдельные правила для Python-скаляров в NumPy 2.x.
  • NEP 50: Promotion rules for Python scalars (EN) — первичный документ, объясняющий, почему современные правила продвижения типов были изменены. Это материал для углубления: читайте мотивацию и примеры поведения, а детали реализации можно пропустить.

Quiz · 8 вопроса

Проверьте себя

Ответьте после практики. Объяснение появится сразу после выбора.

01

Почему observed range 0..200 недостаточен для production-рекомендации dtype?

Наблюдаемая выборка описывает только текущие данные. Нужны ожидаемые и производные границы.

02

После изменения контракта допустимы значения от -500 до 20 000. Какой двухбайтовый тип подходит?

int16 покрывает -32 768..32 767. uint16 не допускает отрицательные значения.

03

В массиве целочисленных измерений появился пропуск. Какое действие корректно?

Способ хранения пропуска следует из его предметного смысла. Обычный integer dtype не представляет NaN.

04

Как обосновать преобразование float64 в float32 для аналитического расчёта?

Экономия памяти не доказывает сохранение точности. Нужен явно заданный бюджет ошибки.

05

Что важно учесть при np.array([256]).astype(np.uint8)?

astype исполняет преобразование и может обернуть значение по диапазону узкого integer.

06

Почему np.can_cast(int64, uint16, casting='safe') может вернуть False, хотя конкретный контракт 0..20 000 проходит аудит?

Это разные уровни проверки: совместимость типов для всех значений и совместимость заданного диапазона.

07

Каждое дневное значение помещается в uint16 и может достигать 20 000. Что нужно учесть при сумме за 365 дней?

Диапазон результата операции может быть шире входного. Тип хранения исходных значений не является автоматическим контрактом агрегата.

08

Что именно измеряет array.nbytes?

nbytes равен size × itemsize и не включает все накладные расходы и промежуточные массивы.

class DtypeAuditTest(unittest.TestCase):
    def test_manual_bounds_match_signed_and_unsigned_types(self) -> None:
        self.assertEqual(AUDIT.manual_integer_bounds(8, signed=True), (-128, 127))
        self.assertEqual(AUDIT.manual_integer_bounds(8, signed=False), (0, 255))

    def test_smallest_integer_dtype_uses_sign_and_expected_range(self) -> None:
        self.assertEqual(AUDIT.smallest_integer_dtype(0, 255), "uint8")
        self.assertEqual(AUDIT.smallest_integer_dtype(0, 20_000), "uint16")
        self.assertEqual(AUDIT.smallest_integer_dtype(-1, 127), "int8")
        self.assertEqual(AUDIT.smallest_integer_dtype(-500, 20_000), "int16")

    def test_planned_shape_connects_size_dtype_and_memory(self) -> None:
        report = AUDIT.audit_values(
            [0, 12, 200],
            target_dtype="uint16",
            expected_min=0,
            expected_max=20_000,
            planned_shape=(500, 365, 24),
        )
        planned_size = 500 * 365 * 24
        self.assertEqual(report["status"], "approved")
        self.assertEqual(report["planning"]["size"], planned_size)
        self.assertEqual(
            report["planning"]["source_dtype_projected_nbytes"],
            planned_size * np.asarray([0, 12, 200]).itemsize,
        )
        self.assertEqual(report["target"]["projected_nbytes"], planned_size * 2)

    def test_observed_only_recommendation_is_marked_as_warning(self) -> None:
        report = AUDIT.audit_values([0, 12, 200])
        self.assertEqual(report["status"], "warning")
        self.assertEqual(report["recommendation"]["basis"], "observed_values")
        self.assertEqual(report["recommendation"]["smallest_integer_dtype"], "uint8")
        self.assertFalse(report["recommendation"]["production_ready"])

    def test_expected_range_can_reject_dtype_that_fits_the_sample(self) -> None:
        report = AUDIT.audit_values(
            [0, 12, 200],
            target_dtype="uint8",
            expected_min=0,
            expected_max=20_000,
        )
        self.assertEqual(report["status"], "rejected")
        self.assertIn("expected_range_fits_target", report["failed_checks"])
        self.assertNotIn("observed_range_fits_target", report["failed_checks"])

    def test_observed_out_of_range_value_is_rejected_before_cast(self) -> None:
        report = AUDIT.audit_values(
            [0, 256],
            target_dtype="uint8",
            expected_min=0,
            expected_max=255,
        )

from __future__ import annotations

import argparse
import json
import math
from collections.abc import Sequence
from numbers import Integral, Real
from pathlib import Path
from typing import Any

import numpy as np

INTEGER_DTYPES = (
    np.dtype("uint8"),
    np.dtype("int8"),
    np.dtype("uint16"),
    np.dtype("int16"),
    np.dtype("uint32"),
    np.dtype("int32"),
    np.dtype("uint64"),
    np.dtype("int64"),
)


class DtypeAuditError(ValueError):
    """Raised when a dtype contract cannot be evaluated."""


def manual_integer_bounds(bits: int, *, signed: bool) -> tuple[int, int]:
    if isinstance(bits, bool) or not isinstance(bits, Integral) or bits <= 0:
        raise DtypeAuditError("integer bit width must be a positive integer")
    if signed:
        return (-(2 ** (int(bits) - 1)), 2 ** (int(bits) - 1) - 1)
    return (0, 2 ** int(bits) - 1)


def smallest_integer_dtype(minimum: int, maximum: int) -> str | None:
    if minimum > maximum:
        raise DtypeAuditError("minimum cannot exceed maximum")
    preferred_kind = "u" if minimum >= 0 else "i"
    candidates = sorted(
        INTEGER_DTYPES,
        key=lambda dtype: (dtype.itemsize, dtype.kind != preferred_kind),
    )
    for dtype in candidates:
        info = np.iinfo(dtype)
        if minimum >= info.min and maximum <= info.max:
            return dtype.name
    return None


def validate_shape(shape: Sequence[int]) -> tuple[int, ...]:
    if isinstance(shape, (str, bytes)):
        raise DtypeAuditError("planned shape must be a sequence of integers")
    raw_shape = tuple(shape)
    if any(isinstance(length, bool) or not isinstance(length, Integral) for length in raw_shape):
        raise DtypeAuditError("planned shape must contain integers")
    result = tuple(int(length) for length in raw_shape)
    if any(length < 0 for length in result):
        raise DtypeAuditError("planned shape cannot contain negative lengths")
    return result


def replace_missing(value: object) -> object:
    if value is None:
        return np.nan
    if isinstance(value, Sequence) and not isinstance(value, (str, bytes, bytearray)):
        return [replace_missing(item) for item in value]
    return value


def parse_json(raw: str, *, name: str = "values") -> Any:
    try:
        return json.loads(raw)
    except json.JSONDecodeError as error:
        raise DtypeAuditError(
            f"{name} must be valid JSON at line {error.lineno}, column {error.colno}"
        ) from error


def dtype_limits(dtype: np.dtype[Any]) -> dict[str, Any]:
    if np.issubdtype(dtype, np.integer):
        info = np.iinfo(dtype)
        manual_min, manual_max = manual_integer_bounds(
            info.bits,
            signed=np.issubdtype(dtype, np.signedinteger),
        )
        return {
            "kind": "integer",
            "bits": info.bits,
            "min": int(info.min),
            "max": int(info.max),
            "manual_bounds_match": (manual_min, manual_max) == (int(info.min), int(info.max)),
        }
    if np.issubdtype(dtype, np.floating):
        info = np.finfo(dtype)
        return {
            "kind": "floating",
            "bits": info.bits,
            "min": float(info.min),
            "max": float(info.max),
            "eps": float(info.eps),
        }
    raise DtypeAuditError(f"expected an integer or floating dtype, got {dtype}")


def _is_integral_number(value: Real) -> bool:
    return float(value).is_integer()


def _range_fits_dtype(minimum: Real, maximum: Real, dtype: np.dtype[Any]) -> bool:
    limits = dtype_limits(dtype)
    return minimum >= limits["min"] and maximum <= limits["max"]


def _round_trip_error(values: np.ndarray, target: np.dtype[Any]) -> float | None:
    if not values.size:
        return None
    if np.issubdtype(target, np.integer):
        return 0.0
    original = values.astype(np.longdouble)
    with np.errstate(over="ignore", invalid="ignore"):
        restored = values.astype(target).astype(np.longdouble)
    errors = np.abs(restored - original)
    return float(errors.max())


def _check(name: str, passed: bool, detail: str) -> dict[str, str | bool]:
    return {"name": name, "passed": bool(passed), "detail": detail}


def audit_values(
    values: object,
    *,
    target_dtype: str | None = None,
    expected_min: Real | None = None,
    expected_max: Real | None = None,
    allow_missing: bool = False,
    max_abs_error: float | None = None,
    planned_shape: Sequence[int] | None = None,
) -> dict[str, Any]:
    if (expected_min is None) != (expected_max is None):
        raise DtypeAuditError("expected_min and expected_max must be provided together")
    if expected_min is not None:
        if (
            isinstance(expected_min, bool)
            or isinstance(expected_max, bool)
            or not isinstance(expected_min, Real)
            or not isinstance(expected_max, Real)
        ):
            raise DtypeAuditError("expected bounds must be numeric")
        if not math.isfinite(float(expected_min)) or not math.isfinite(float(expected_max)):
            raise DtypeAuditError("expected bounds must be finite")
        if expected_min > expected_max:
            raise DtypeAuditError("expected_min cannot exceed expected_max")
    if max_abs_error is not None and (
        not math.isfinite(max_abs_error) or max_abs_error < 0
    ):
        raise DtypeAuditError("max_abs_error must be finite and non-negative")

    normalized = replace_missing(values)
    try:
        array = np.asarray(normalized)
    except (TypeError, ValueError) as error:
        raise DtypeAuditError(f"cannot create a rectangular array: {error}") from error
    if array.dtype.kind not in "iuf":
        raise DtypeAuditError(f"expected numeric input, got dtype {array.dtype}")

    if array.dtype.kind == "f":
        missing_mask = np.isnan(array)
        infinite_mask = np.isinf(array)
        finite = array[np.isfinite(array)]
    else:
        missing_mask = np.zeros(array.shape, dtype=bool)
        infinite_mask = np.zeros(array.shape, dtype=bool)
        finite = array.reshape(-1)

    missing_count = int(missing_mask.sum())
    infinite_count = int(infinite_mask.sum())
    finite_min = finite.min().item() if finite.size else None
    finite_max = finite.max().item() if finite.size else None
    shape_for_planning = array.shape if planned_shape is None else validate_shape(planned_shape)
    planned_size = math.prod(shape_for_planning)

    recommendation_min = expected_min if expected_min is not None else finite_min
    recommendation_max = expected_max if expected_max is not None else finite_max
    integer_recommendation = None
    if (
        recommendation_min is not None
        and recommendation_max is not None
        and _is_integral_number(recommendation_min)
        and _is_integral_number(recommendation_max)
    ):
        integer_recommendation = smallest_integer_dtype(
            int(recommendation_min),
            int(recommendation_max),
        )

    checks = [
        _check(
            "missing_policy",
            allow_missing or missing_count == 0,
            (
                f"found {missing_count} missing values"
                if missing_count
                else "no missing values observed"
            ),
        ),
        _check(
            "finite_values",
            infinite_count == 0,
            (
                f"found {infinite_count} infinite values"
                if infinite_count
                else "no infinite values observed"
            ),
        ),
    ]
    warnings: list[str] = []
    if expected_min is None:
        warnings.append(
            "expected bounds are missing; an observed-value recommendation "
            "is not a production contract"
        )

    target_report = None
    if target_dtype is None:
        warnings.append("target dtype is missing; no conversion can be approved")
    else:
        try:
            target = np.dtype(target_dtype)
        except TypeError as error:
            raise DtypeAuditError(f"unknown target dtype {target_dtype!r}") from error
        if target.kind not in "iuf":
            raise DtypeAuditError(f"target dtype must be numeric, got {target}")

        target_is_integer = np.issubdtype(target, np.integer)
        observed_fits = (
            True
            if finite_min is None
            else _range_fits_dtype(finite_min, finite_max, target)
        )
        expected_fits = (
            True
            if expected_min is None
            else _range_fits_dtype(expected_min, expected_max, target)
        )
        observed_integral = bool(
            not finite.size or np.all(np.equal(finite, np.trunc(finite)))
        )
        expected_integral = bool(
            expected_min is None
            or (
                _is_integral_number(expected_min)
                and _is_integral_number(expected_max)
            )
        )
        target_supports_missing = missing_count == 0 or not target_is_integer

        checks.extend(
            [
                _check(
                    "observed_range_fits_target",
                    observed_fits,
                    (
                        "finite observed values fit the target range"
                        if observed_fits
                        else f"observed range {finite_min}..{finite_max} exceeds {target}"
                    ),
                ),
                _check(
                    "expected_range_fits_target",
                    expected_fits,
                    (
                        "expected bounds fit the target range"
                        if expected_fits
                        else f"expected range {expected_min}..{expected_max} exceeds {target}"
                    ),
                ),
                _check(
                    "integer_target_preserves_fractional_values",
                    not target_is_integer or (observed_integral and expected_integral),
                    (
                        "integer target is compatible with integral values"
                        if target_is_integer and observed_integral and expected_integral
                        else (
                            "target is floating"
                            if not target_is_integer
                            else "integer target would discard a fractional part"
                        )
                    ),
                ),
                _check(
                    "target_supports_missing_policy",
                    target_supports_missing,
                    (
                        "target can represent the observed missing-value policy"
                        if target_supports_missing
                        else f"{target} cannot represent NaN"
                    ),
                ),
            ]
        )

        can_measure_error = (
            observed_fits
            and expected_fits
            and (not target_is_integer or (observed_integral and expected_integral))
        )
        round_trip_error = _round_trip_error(finite, target) if can_measure_error else None
        if max_abs_error is not None:
            error_passed = round_trip_error is not None and round_trip_error <= max_abs_error
            checks.append(
                _check(
                    "round_trip_error_within_budget",
                    error_passed,
                    (
                        f"max absolute round-trip error {round_trip_error} <= {max_abs_error}"
                        if error_passed
                        else (
                            f"max absolute round-trip error {round_trip_error} "
                            f"exceeds {max_abs_error}"
                        )
                    ),
                )
            )
        elif np.issubdtype(target, np.floating) and target.itemsize < array.dtype.itemsize:
            warnings.append(
                "floating dtype is narrowed without max_abs_error; precision loss is not approved"
            )

        target_report = {
            "dtype": target.name,
            "itemsize_bytes": int(target.itemsize),
            "projected_nbytes": int(planned_size * target.itemsize),
            "projected_memory_change_bytes": int(
                planned_size * target.itemsize - planned_size * array.itemsize
            ),
            "limits": dtype_limits(target),
            "numpy_type_level_safe_cast": bool(
                np.can_cast(array.dtype, target, casting="safe")
            ),
            "round_trip_max_abs_error": round_trip_error,
        }

    failed_checks = [check["name"] for check in checks if not check["passed"]]
    status = "rejected" if failed_checks else ("warning" if warnings else "approved")

    return {
        "status": status,
        "source": {
            "shape": list(array.shape),
            "size": int(array.size),
            "dtype": array.dtype.name,
            "itemsize_bytes": int(array.itemsize),
            "nbytes": int(array.nbytes),
            "missing_count": missing_count,
            "infinite_count": infinite_count,
            "finite_min": finite_min,
            "finite_max": finite_max,
        },
        "contract": {
            "expected_min": expected_min,
            "expected_max": expected_max,
            "allow_missing": allow_missing,
            "max_abs_error": max_abs_error,
        },
        "planning": {
            "shape": list(shape_for_planning),
            "size": int(planned_size),
            "source_dtype_projected_nbytes": int(planned_size * array.itemsize),
        },
        "recommendation": {
            "basis": "expected_contract" if expected_min is not None else "observed_values",
            "smallest_integer_dtype": integer_recommendation,
            "production_ready": expected_min is not None,
        },
        "target": target_report,
        "checks": checks,
        "failed_checks": failed_checks,
        "warnings": warnings,
    }


def main() -> int:
    parser = argparse.ArgumentParser(
        description=(
            "Audit a NumPy dtype against range, missing-value, precision, "
            "and memory contracts"
        )
    )
    parser.add_argument("--values", default="[0, 12, 200]", help="Numeric JSON sample")
    parser.add_argument("--target-dtype", help="Candidate NumPy dtype")
    parser.add_argument("--expected-min", type=float)
    parser.add_argument("--expected-max", type=float)
    parser.add_argument("--allow-missing", action="store_true")
    parser.add_argument("--max-abs-error", type=float)
    parser.add_argument("--planned-shape", help="JSON shape used only for memory planning")
    parser.add_argument("--output", type=Path)
    args = parser.parse_args()

    try:
        report = audit_values(
            parse_json(args.values),
            target_dtype=args.target_dtype,
            expected_min=args.expected_min,
            expected_max=args.expected_max,
            allow_missing=args.allow_missing,
            max_abs_error=args.max_abs_error,
            planned_shape=(
                parse_json(args.planned_shape, name="planned shape")
                if args.planned_shape
                else None
            ),
        )
    except DtypeAuditError as error:
        parser.exit(2, f"dtype-audit: {error}\n")

    text = json.dumps(report, ensure_ascii=False, indent=2, allow_nan=False) + "\n"
    if args.output:
        args.output.write_text(text, encoding="utf-8")
    else:
        print(text, end="")
    return 1 if report["status"] == "rejected" else 0


if __name__ == "__main__":
    raise SystemExit(main())
