Версии Python и совместимость

«У меня запускается» описывает один интерпретатор; воспроизводимый проект описывает поддерживаемый диапазон и проверяет, какой Python выполняет код прямо сейчас.

Тип: Build Треки: Core Пререквизиты: 00/06 — Секреты и безопасная работа с данными Время: ~45 минут Результат: объясняет связь интерпретатора, окружения и совместимости зависимостей

Цели обучения

  • Отличать команду python в shell от интерпретатора текущего процесса.
  • Читать sys.version_info и sys.executable.
  • Задавать диапазон совместимости через [project].requires-python.
  • Объяснять различие между requires-python и .python-version.
  • Проверять нижнюю и верхнюю границы на матрице кандидатных версий.
  • Понимать, почему смена Python требует пересоздания окружения и повторной проверки зависимостей.

Проблема

Аналитик запускает notebook, скрипт и тесты командами, которые выглядят одинаково:

bash
python analysis.py
python -m pytest
jupyter lab

Но за ними могут стоять разные интерпретаторы:

text
/usr/bin/python3
/opt/homebrew/bin/python3
project/.venv/bin/python
kernel из старого окружения

На одной машине код использует Python 3.12, на другой — 3.10. В первом есть нужная возможность стандартной библиотеки, во втором импорт завершается ошибкой. Или сам код запускается, но выбранная версия зависимости больше не поддерживает старый Python.

Фраза «проект написан на Python 3» почти ничего не гарантирует. Нужны ответы на три разных вопроса:

  1. Какой Python совместим с проектом?
  2. Какой Python должен выбрать локальный инструмент?
  3. Какой бинарник выполняет этот конкретный процесс?

Если ответы не зафиксированы и не проверяются, окружение нельзя воспроизвести предсказуемо.

Концепция

Интерпретатор, окружение и зависимости

Интерпретатор — исполняемый файл Python. Внутри процесса его можно определить через:

python
import sys

print(sys.version_info)
print(sys.executable)

sys.version_info сообщает версию запущенного Python. sys.executable обычно содержит абсолютный путь к бинарнику, который выполняет код.

Окружение связывает конкретный интерпретатор с каталогами установленных пакетов и командами. Virtual environment не превращает Python 3.11 в Python 3.12. Оно создаётся на основе определённого интерпретатора.

Зависимости имеют собственные ограничения: поддерживаемые версии Python, доступные wheels, platform tags и зависимости от других пакетов. Поэтому совместимость проекта выглядит как пересечение:

text
возможности кода
∩ requires-python проекта
∩ поддержка Python зависимостями
∩ доступные сборки для платформы

После смены базового Python старое окружение нельзя считать доказательством воспроизводимости. Его нужно пересоздать и снова установить и проверить зависимости.

requires-python — диапазон поддержки

В pyproject.toml поле requires-python является стандартной metadata проекта:

toml
[project]
name = "analytics-project"
version = "0.1.0"
requires-python = ">=3.11,<3.13"

Здесь clauses разделены запятыми и применяются одновременно:

  • >=3.11 — Python 3.11 включён;
  • <3.13 — Python 3.13 исключён;
  • Python 3.12 находится внутри диапазона.

Это не утверждение, что каждая версия внутри диапазона уже установлена. Это обещание проекта: такие интерпретаторы считаются поддерживаемыми и должны участвовать в проверках.

Нижняя граница определяется самой ранней версией, на которой работают:

  • синтаксис проекта;
  • используемые модули стандартной библиотеки;
  • зависимости;
  • тесты и инструменты.

Верхняя граница нужна, если будущая или уже выпущенная версия ещё не проверена либо несовместима с критичной зависимостью. Не добавляйте её автоматически из страха, но и не обещайте >=3.11 навсегда, если проект никогда не проверялся на следующей major/minor версии.

.python-version — выбор для инструмента

Файл:

text
3.11

сообщает совместимым инструментам предпочтительную локальную версию. Например, uv учитывает .python-version, а при работе с проектом также уважает requires-python.

Это два разных утверждения:

text
requires-python = ">=3.11,<3.13"  # поддерживаются 3.11 и 3.12
.python-version = 3.11            # локально сначала выбираем 3.11

.python-version является соглашением инструментов, а не заменой packaging metadata. Если оставить только selector, установщик пакета не узнает диапазон поддержки. Если оставить только диапазон, инструменту может понадобиться самому выбрать одну из нескольких доступных версий.

Версия в shell и версия процесса

Команда:

bash
python3 --version

показывает бинарник, который shell нашёл для имени python3. Она не доказывает, что уже запущенный Jupyter kernel использует тот же путь.

Диагностику выполняйте внутри проблемного процесса:

python
import sys

print("version:", ".".join(map(str, sys.version_info[:3])))
print("executable:", sys.executable)

Сравнивайте абсолютный путь, а не только номер версии. Два Python 3.12 могут иметь разные пакеты и разные окружения.

Значение 3.11 выбирает любую подходящую patch-версию линии 3.11. Фиксация patch до 3.11.9 даёт более точный выбор, но может усложнить запуск на машине, где доступна другая security patch. Уровень точности должен соответствовать способу поставки проекта.

Для учебного курса достаточно minor selector и диапазона metadata. Управление установкой Python и окружением появится в следующем уроке про uv.

Запишите:

Шаг 4: проверьте контракт CLI

Из каталога урока:

bash
python3 outputs/python_version_check.py check path/to/version-lab \
  --candidate 3.10.14 \
  --candidate 3.11.9 \
  --candidate 3.12.4 \
  --candidate 3.13.0

Из корня курса:

bash
python3 phases/01-reproducible-project/01-python-versions/outputs/python_version_check.py \
  check version-lab \
  --candidate 3.10.14 \
  --candidate 3.11.9 \
  --candidate 3.12.4 \
  --candidate 3.13.0

Проверка завершится успешно только если:

  • requires-python существует и понятен CLI;
  • .python-version находится внутри диапазона;
  • текущий Python входит в диапазон;
  • текущий major/minor совпадает с selector;
  • кандидатные версии записаны корректно.

Если текущая машина использует Python 3.12, а selector равен 3.11, инструмент честно покажет несовпадение. Для продолжения практики выберите установленную и поддерживаемую версию, но не расширяйте диапазон только ради зелёного отчёта.

Используйте это

Проверяйте процесс, а не предположение

Перед расследованием «пакет установлен, но import не работает» выполните:

bash
python3 -c 'import sys; print(sys.executable)'
python3 -m pip --version

Обе команды должны указывать на ожидаемое окружение. Вызов python3 -m pip связывает pip с выбранным Python яснее, чем отдельная команда pip.

В notebook:

python
import sys

{
    "version": tuple(sys.version_info[:3]),
    "executable": sys.executable,
}

Если путь kernel отличается от shell, установка пакета в shell не меняет уже выбранный kernel.

Стройте матрицу до изменения диапазона

Сформируйте отчёт:

bash
python3 outputs/python_version_check.py check . \
  --candidate 3.11 \
  --candidate 3.12 \
  --candidate 3.13 \
  --format json \
  --output /tmp/python-compatibility.json

Матрица отвечает только на вопрос, допускает ли версия текущий metadata-контракт. Она не запускает код на неустановленных интерпретаторах. Настоящая поддержка требует CI или локального запуска тестов на каждой заявленной линии.

Это важное различие:

text
specifier допускает версию ≠ проект доказанно работает на версии

Меняйте нижнюю границу обоснованно

Причины поднять минимум:

  • код использует новый синтаксис;
  • нужен модуль, появившийся в новой стандартной библиотеке;
  • критичная зависимость прекратила поддержку старой версии;
  • команда больше не может надёжно тестировать старую линию.

В pull request укажите:

text
Минимум поднят с 3.10 до 3.11, потому что проект использует tomllib из стандартной
библиотеки. CI проверяет 3.11 и 3.12.

Фраза «у меня установлен 3.11» не является причиной изменить публичный контракт.

Не обещайте совместимость по одной машине

Если автор работает на 3.12, из этого не следует ни ==3.12, ни >=3.12.

  • ==3.12.7 может быть слишком узким и запрещать совместимые security patches.
  • >=3.12 без верхней границы обещает поддержку ещё не проверенных будущих версий.
  • >=3.11,<3.13 разумно только при тестах 3.11 и 3.12 и осознанном исключении 3.13.

Контракт описывает политику поддержки проекта, а не снимок ноутбука автора.

Сломайте это

Запустите другой Python

Измените .python-version на minor, отличающийся от текущего:

bash
printf '3.11\n' > .python-version
python3 path/to/python_version_check.py check .

Если процесс работает на 3.12, selection-match завершится ошибкой. Это не означает, что 3.12 несовместим с проектом: selector и диапазон отвечают на разные вопросы.

Выберите версию вне диапазона

toml
requires-python = ">=3.11,<3.13"
text
# .python-version
3.13

Проверка selector остановит контракт. Инструмент не должен выбирать Python, который сам проект объявил неподдерживаемым.

Запишите требование человеческой фразой

toml
requires-python = "Python 3.11 or newer"

Это не version specifier. Используйте формальный синтаксис:

toml
requires-python = ">=3.11"

CLI урока поддерживает числовые clauses >=, >, <=, <, ==, !=, ~= и wildcard для ==/!=. Pre-release, local versions и сложные варианты намеренно не реализованы: для полного синтаксиса источником истины остаётся спецификация PyPA и production packaging tools.

Уберите верхнюю границу без проверки

toml
requires-python = ">=3.11"

Синтаксически это корректно. Но теперь metadata допускает любую будущую версию. Если проект публикуется как пакет или разворачивается автоматически, resolver может выбрать комбинацию, которую команда никогда не тестировала.

Решение не в обязательном <3.13 для всех проектов. Решение — согласовать metadata, матрицу CI и реальную политику поддержки.

Поменяйте Python внутри существующего окружения

Представьте .venv, созданный на 3.11, после чего selector изменили на 3.12. Старый каталог может продолжать ссылаться на прежний интерпретатор и содержать несовместимые binary wheels.

Не «ремонтируйте» такое окружение последовательностью случайных установок. Пересоздайте его из контракта и lock-файла. Эту процедуру мы соберём в уроке 01/02.

01

Чем .python-version отличается от requires-python?

Selector помогает инструменту выбрать интерпретатор, но не заменяет metadata о поддерживаемом диапазоне.

02

Почему команда python --version недостаточна для диагностики notebook?

Нужно проверять sys.executable внутри самого процесса, который выполняет код.

03

Как следует выбирать нижнюю границу requires-python?

Контракт должен отражать доказанную совместимость, а не случайную конфигурацию одной машины.

Практика без перехода на GitHub

Файлы урока

Тесты, reference-код и артефакты встроены в страницу. Сначала попробуйте решить задачу самостоятельно, затем раскройте нужный файл и сравните подход.

Тесты — это техническое задание

tests/test_main.pyПоказать
from __future__ import annotations

import importlib.util
import sys
from pathlib import Path
from tempfile import TemporaryDirectory
from unittest import TestCase


LESSON_ROOT = Path(__file__).resolve().parents[1]
TOOL_PATH = LESSON_ROOT / "outputs" / "python_version_check.py"


def load_tool():
    spec = importlib.util.spec_from_file_location("python_version_test", TOOL_PATH)
    if spec is None or spec.loader is None:
        raise RuntimeError("Cannot load Python version checker")
    module = importlib.util.module_from_spec(spec)
    sys.modules[spec.name] = module
    spec.loader.exec_module(module)
    return module

from __future__ import annotations

import argparse
import json
import platform
import re
import sys
import tomllib
from dataclasses import dataclass
from pathlib import Path
from typing import Any


VERSION_PATTERN = re.compile(r"^\d+(?:\.\d+){1,2}$")
CLAUSE_PATTERN = re.compile(
    r"^(~=|==|!=|<=|>=|<|>)\s*(\d+(?:\.\d+){1,2})(\.\*)?$"
)
PROJECT_NAME_PATTERN = re.compile(r"^[A-Za-z0-9]+(?:[-_.][A-Za-z0-9]+)*$")


@dataclass(frozen=True)
class Version:
    release: tuple[int, int, int]
    precision: int

    @classmethod
    def parse(cls, value: str) -> "Version":
        candidate = value.strip()
        if not VERSION_PATTERN.fullmatch(candidate):
            raise ValueError(
                f"unsupported Python release '{value}'; expected MAJOR.MINOR[.PATCH]"
            )
        parts = tuple(int(part) for part in candidate.split("."))
        return cls(parts + (0,) * (3 - len(parts)), len(parts))

    def matches_prefix(self, other: "Version") -> bool:
        return self.release[: self.precision] == other.release[: self.precision]

    def __str__(self) -> str:
        return ".".join(str(part) for part in self.release[: self.precision])


@dataclass(frozen=True)
class Clause:
    operator: str
    version: Version
    wildcard: bool = False

    def contains(self, candidate: Version) -> bool:
        current = candidate.release
        expected = self.version.release
        if self.wildcard:
            matched = self.version.matches_prefix(candidate)
            return matched if self.operator == "==" else not matched
        if self.operator == "==":
            return current == expected
        if self.operator == "!=":
            return current != expected
        if self.operator == ">=":
            return current >= expected
        if self.operator == "<=":
            return current <= expected
        if self.operator == ">":
            return current > expected
        if self.operator == "<":
            return current < expected
        if self.operator == "~=":
            if current < expected:
                return False
            if self.version.precision == 2:
                upper = (expected[0] + 1, 0, 0)
            else:
                upper = (expected[0], expected[1] + 1, 0)
            return current < upper
        raise AssertionError(f"unknown operator: {self.operator}")

    def __str__(self) -> str:
        suffix = ".*" if self.wildcard else ""
        return f"{self.operator}{self.version}{suffix}"


def parse_specifier(value: str) -> list[Clause]:
    if not isinstance(value, str) or not value.strip():
        raise ValueError("requires-python must be a non-empty string")
    clauses: list[Clause] = []
    for raw_clause in value.split(","):
        candidate = raw_clause.strip()
        match = CLAUSE_PATTERN.fullmatch(candidate)
        if match is None:
            raise ValueError(
                f"unsupported requires-python clause '{candidate}'; "
                "use numeric release clauses such as >=3.11,<3.13"
            )
        operator, version_text, wildcard_text = match.groups()
        wildcard = bool(wildcard_text)
        if wildcard and operator not in {"==", "!="}:
            raise ValueError("wildcard releases are supported only with == or !=")
        version = Version.parse(version_text)
        if wildcard and version.precision < 2:
            raise ValueError("wildcard release requires at least MAJOR.MINOR")
        clauses.append(Clause(operator, version, wildcard))
    return clauses


def satisfies(version: Version, clauses: list[Clause]) -> bool:
    return all(clause.contains(version) for clause in clauses)


def normalize_project_name(value: str) -> str:
    candidate = value.strip()
    if not PROJECT_NAME_PATTERN.fullmatch(candidate):
        raise ValueError("project name must contain letters, digits, dots, underscores or dashes")
    return candidate


def initialize_contract(
    path: Path,
    project_name: str,
    requires_python: str,
    selector: str,
) -> dict[str, Any]:
    root = path.expanduser().resolve()
    if not root.is_dir():
        raise ValueError(f"project directory does not exist: {root}")
    pyproject = root / "pyproject.toml"
    selector_path = root / ".python-version"
    if pyproject.exists() or selector_path.exists():
        raise ValueError(
            "pyproject.toml or .python-version already exists; "
            "edit the existing project contract deliberately"
        )

    project_name = normalize_project_name(project_name)
    clauses = parse_specifier(requires_python)
    selected = Version.parse(selector)
    if not satisfies(selected, clauses):
        raise ValueError(
            f"selected Python {selected} does not satisfy requires-python "
            f"'{requires_python}'"
        )

    pyproject.write_text(
        (
            "[project]\n"
            f'name = "{project_name}"\n'
            'version = "0.1.0"\n'
            f'requires-python = "{requires_python}"\n'
        ),
        encoding="utf-8",
    )
    selector_path.write_text(f"{selector}\n", encoding="utf-8")
    return {
        "root": str(root),
        "requires_python": requires_python,
        "selector": selector,
        "files": ["pyproject.toml", ".python-version"],
    }


def read_pyproject(root: Path) -> tuple[str | None, str | None]:
    path = root / "pyproject.toml"
    if not path.is_file():
        return None, "pyproject.toml is missing"
    try:
        data = tomllib.loads(path.read_text(encoding="utf-8"))
    except (OSError, tomllib.TOMLDecodeError) as error:
        return None, f"pyproject.toml is invalid: {error}"
    project = data.get("project")
    if not isinstance(project, dict):
        return None, "[project] table is missing"
    value = project.get("requires-python")
    if not isinstance(value, str) or not value.strip():
        return None, "[project].requires-python is missing"
    return value.strip(), None


def read_selector(root: Path) -> tuple[Version | None, str | None]:
    path = root / ".python-version"
    if not path.is_file():
        return None, ".python-version is missing"
    lines = [
        line.strip()
        for line in path.read_text(encoding="utf-8").splitlines()
        if line.strip() and not line.lstrip().startswith("#")
    ]
    if len(lines) != 1:
        return None, ".python-version must contain one Python release"
    try:
        return Version.parse(lines[0]), None
    except ValueError as error:
        return None, str(error)


def resolve_root(path: Path) -> Path:
    candidate = path.expanduser().resolve()
    if not candidate.is_dir():
        raise ValueError(f"project directory does not exist: {candidate}")
    return candidate


def runtime_version() -> Version:
    release = sys.version_info
    return Version((release.major, release.minor, release.micro), 3)


def evaluate_project(
    path: Path,
    candidates: list[str] | None = None,
    current_version: Version | None = None,
    executable: str | None = None,
) -> dict[str, Any]:
    root = resolve_root(path)
    requires_python, metadata_error = read_pyproject(root)
    selector, selector_error = read_selector(root)
    clauses: list[Clause] = []
    specifier_error = metadata_error
    if requires_python is not None:
        try:
            clauses = parse_specifier(requires_python)
        except ValueError as error:
            specifier_error = str(error)

    current = current_version or runtime_version()
    executable_path = executable if executable is not None else (sys.executable or "")
    current_supported = bool(clauses) and satisfies(current, clauses)
    selector_supported = (
        selector is not None and bool(clauses) and satisfies(selector, clauses)
    )
    selector_matches_runtime = (
        selector is not None and selector.matches_prefix(current)
    )

    matrix: list[dict[str, Any]] = []
    candidate_errors: list[str] = []
    for value in candidates or []:
        try:
            version = Version.parse(value)
        except ValueError as error:
            candidate_errors.append(str(error))
            continue
        matrix.append(
            {
                "version": str(version),
                "compatible": bool(clauses) and satisfies(version, clauses),
            }
        )

    checks = [
        {
            "id": "requires-python",
            "passed": specifier_error is None and bool(clauses),
            "message": (
                f"Project declares requires-python = '{requires_python}'."
                if specifier_error is None and clauses
                else specifier_error or "requires-python has no clauses"
            ),
        },
        {
            "id": "selector",
            "passed": selector_error is None and selector_supported,
            "message": (
                f".python-version selects {selector}, inside the supported range."
                if selector_error is None and selector_supported
                else selector_error
                or f"Selected Python {selector} is outside requires-python."
            ),
        },
        {
            "id": "runtime",
            "passed": bool(executable_path) and current_supported,
            "message": (
                f"Running Python {current} from {executable_path} is compatible."
                if executable_path and current_supported
                else (
                    f"Running Python {current} is outside requires-python."
                    if executable_path
                    else "sys.executable is unavailable."
                )
            ),
        },
        {
            "id": "selection-match",
            "passed": selector_error is None and selector_matches_runtime,
            "message": (
                f"Runtime {current} matches selector {selector}."
                if selector_error is None and selector_matches_runtime
                else selector_error
                or f"Runtime {current} does not match selector {selector}."
            ),
        },
        {
            "id": "candidate-input",
            "passed": not candidate_errors,
            "message": (
                "Candidate version inputs are valid."
                if not candidate_errors
                else "; ".join(candidate_errors)
            ),
        },
    ]
    return {
        "root": str(root),
        "ready": all(check["passed"] for check in checks),
        "requires_python": requires_python,
        "selector": str(selector) if selector else None,
        "runtime": {
            "version": str(current),
            "executable": executable_path,
            "implementation": platform.python_implementation(),
        },
        "checks": checks,
        "matrix": matrix,
    }


def render_markdown(report: dict[str, Any]) -> str:
    result = "compatible" if report["ready"] else "needs attention"
    lines = [
        "# Python version contract",
        "",
        f"- Project: `{report['root']}`",
        f"- Requires-Python: `{report['requires_python'] or 'missing'}`",
        f"- Selector: `{report['selector'] or 'missing'}`",
        f"- Runtime: `{report['runtime']['version']}`",
        f"- Executable: `{report['runtime']['executable'] or 'unknown'}`",
        f"- Result: **{result}**",
        "",
        "## Checks",
        "",
        "| Check | Result | Details |",
        "|---|---|---|",
    ]
    for check in report["checks"]:
        status = "PASS" if check["passed"] else "FAIL"
        lines.append(f"| `{check['id']}` | {status} | {check['message']} |")

    lines.extend(["", "## Candidate matrix", ""])
    if report["matrix"]:
        lines.extend(["| Python | Compatible |", "|---|---|"])
        for item in report["matrix"]:
            compatible = "yes" if item["compatible"] else "no"
            lines.append(f"| `{item['version']}` | {compatible} |")
    else:
        lines.append("_No candidate versions requested._")
    return "\n".join(lines) + "\n"


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        description="Create and check a Python version contract"
    )
    subparsers = parser.add_subparsers(dest="command", required=True)

    init_parser = subparsers.add_parser("init", help="Create a minimal version contract")
    init_parser.add_argument("path", type=Path)
    init_parser.add_argument("--name", required=True)
    init_parser.add_argument("--requires", required=True, dest="requires_python")
    init_parser.add_argument("--select", required=True, dest="selector")

    check_parser = subparsers.add_parser("check", help="Check the project and runtime")
    check_parser.add_argument("path", type=Path)
    check_parser.add_argument("--candidate", action="append", default=[])
    check_parser.add_argument("--format", choices=("markdown", "json"), default="markdown")
    check_parser.add_argument("--output", type=Path)
    return parser


def main() -> int:
    parser = build_parser()
    args = parser.parse_args()
    try:
        if args.command == "init":
            result = initialize_contract(
                args.path,
                project_name=args.name,
                requires_python=args.requires_python,
                selector=args.selector,
            )
            print(json.dumps(result, ensure_ascii=False, indent=2))
            return 0

        report = evaluate_project(args.path, candidates=args.candidate)
        rendered = (
            json.dumps(report, ensure_ascii=False, indent=2) + "\n"
            if args.format == "json"
            else render_markdown(report)
        )
        if args.output:
            args.output.write_text(rendered, encoding="utf-8")
        else:
            print(rendered, end="")
        return 0 if report["ready"] else 1
    except (OSError, ValueError) as error:
        parser.exit(2, f"python-version-check: {error}\n")


if __name__ == "__main__":
    raise SystemExit(main())
