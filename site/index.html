Self-paced workflow

Когда урок действительно пройден

Зелёный тест reference-кода подтверждает репозиторий, но не заменяет вашу практику.

01 / Attempt

Сначала своя модель

Воспроизведите формулу, таблицу или минимальный механизм в локальной папке work/.

02 / Break

Объяснить поломку

Запустите хотя бы один негативный сценарий и назовите нарушенное предположение.

03 / Check

Проверить перенос

Выполните упражнение и итоговый перемешанный квиз, затем только отмечайте урок.

Метод курса

Не коллекция библиотек

Студент сначала понимает механизм и цену ошибки, затем применяет рабочий инструмент, ломает решение на граничных случаях и поставляет результат.

01 / Problem

Начать с решения

Бизнес-вопрос, grain данных и критерий корректности появляются раньше API библиотеки.

02 / Build

Собрать механизм

Формула, реляционная операция или прозрачный baseline показывают, что именно скрывает инструмент.

03 / Ship

Оставить артефакт

Урок завершается результатом, который запускается, проверяется и используется вне учебного текста.
