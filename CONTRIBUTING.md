# Участие в разработке

## Источник правды

Фазы, зависимости и список уроков редактируются в `curriculum.json`. Файлы `ROADMAP.md` и
`phases/*/README.md` генерируются и не редактируются вручную.

После изменения программы:

```bash
python3 scripts/render_curriculum.py
python3 scripts/render_outputs.py
python3 scripts/render_site.py
python3 scripts/validate_course.py
python3 -m unittest discover -s tests
python3 scripts/run_lesson_tests.py
```

## Добавление урока

1. Добавьте урок в соответствующую фазу `curriculum.json`.
2. Перегенерируйте дорожную карту.
3. Создайте папку урока:

```bash
python3 scripts/scaffold_lesson.py 03-pandas 05-groupby "GroupBy и единица анализа"
```

4. Заполните документацию по `LESSON_TEMPLATE.md`.
5. Добавьте код, тесты, квиз и артефакт.
6. Добавьте 3-5 развивающих материалов по контракту из `LESSON_TEMPLATE.md`.
7. Переведите статус в `complete` только после удаления всех `TODO`.
8. Перегенерируйте индексы и запустите проверки из корня репозитория.

## Правила изменения программы

- Новый инструмент добавляется только вместе с задачей, которую он решает лучше уже
  присутствующих средств.
- Факультатив не становится обязательным пререквизитом.
- Урок должен иметь один главный результат и укладываться примерно в 45-120 минут.
- Большая библиотека делится по рабочим задачам, а не повторяет оглавление документации.
- Ссылки ведут преимущественно на официальную документацию и первичные источники.
- Каждая развивающая ссылка объясняет, что читать и какой навык она углубляет.
- Данные и ожидаемые ответы не должны раскрывать решение упражнения.
- `Build It` показывает механизм до production API; обзор методов библиотеки не считается
  ручной реализацией.
- Артефакт имеет имя, manifest, команду использования и работает вне контекста урока.
